# セーブロード関連問題

各仕様の調査で発見された、セーブ/ロードに起因する永続化の問題を集約する文書。
個別の仕様書では深追いせず、ここにリンクして逃がす。

根本的には **敵の個体状態（HP, broken, MentalHP等）の全般的な永続化** が未整備であり、
個別の問題は全てそこに帰結する。

---

## 1. 敵個体状態の永続化が未整備（根本問題）

**既知の記述**: `doc/友情コンビ登録システム仕様書.md` §7.5

> HP/broken状態のロード後未復元: GetRuntimeEnemies() でGUIDは復元されるが、
> HP・broken・MentalHPはDeepCopy初期値のまま。（中略）
> フリーエンカウント側でbroken敵がリセット状態で出現する可能性があり、
> これは**全体的な敵状態永続化の課題**（本システム固有ではない）

現状、敵個体の永続化は以下の状況：

| 項目 | 永続化 | 備考 |
|------|--------|------|
| GUID | ○ | `EnemyPersistenceData` + `RestoreGuid()` |
| コンボ登録 | ○ | `FriendshipComboSaveData` |
| コンボ敵のbroken | ○ | `FriendshipComboRegistry.MarkBroken()` |
| 非コンボ敵のbroken | **×** | `[NonSerialized]` のみ |
| HP / MentalHP | **×** | DeepCopy初期値に戻る |
| パッシブ | **×** | ランタイムのみ |
| 武器適応率 | **×** | `[NonSerialized]` 3フィールド |
| BattleMemory | **×** | `[NonSerialized]` 設計意図あり（§1.1参照） |

### 1.1 BattleMemoryについて

`doc/終了済み/敵AI未実装要素_多角的分析レポート.md` §3.1 で検討済み。
「BaseStatesに持たせるとゲームリスタート時に消える」問題を認識した上で、
NormalEnemyに `[NonSerialized]` で持たせる設計を選択している。
BattleMemoryは毎戦闘 `Clear()` でリセットされる戦術記憶であり、永続化の必要性は低い。

---

## 2. brokenフラグの永続化漏れ

**発見元**: `doc/終了済み/パッシブ致死構想.md` §3.3

§1の個別ケース。非コンボ敵のbrokenが永続化されない問題。

### 関連コード

| 箇所 | 内容 |
|------|------|
| `BaseStates.StatesState.cs` | `[NonSerialized] public bool broken = false;` |
| `BaseStates.BattleEvent.cs` OverKilledBrokenCalc | broken設定（戦闘中のみ） |
| `BattleManager.MarkBrokenEnemies()` | 戦闘終了時にComboRegistryへ反映（コンボ敵のみ） |
| `FilterEligibleEnemies` | broken敵をスキップ |
| `GetAliveRatio` | broken敵を非生存扱い |

---

## 3. 武器適応率の永続化漏れ

**発見元**: `doc/武器システム仕様書.md`

§1の個別ケース。規格切替後の適応率が永続化されない問題。

```
[NonSerialized] private float _adaptationRate = 1.0f;
[NonSerialized] private float _adaptationStartRate;
[NonSerialized] private int _battlesSinceProtocolSwitch;
```

ロード後に武器の適応ラグがリセットされ、初期値（1.0 = 完全適応）に戻る。
