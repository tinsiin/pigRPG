# スキル吸引構想書

機械種別（`CharacterType.Machine`）専用のターゲット吸引メカニクスの構想。

> 元メモ: 「機械系の敵のスキルにはそういうの関係なしに、攻撃を自分に吸い付ける物があると思う。」（挑発属性メモ）

---

## 1. 背景と動機

挑発属性（イラつきシステム）は有機キャラの感情を軸にした間接的なヘイト制御。機械はイラつきが付かない（`CharacterType.Machine` 免疫）ため、挑発属性の恩恵も脅威も受けない。

スキル吸引は、機械が「タンク役」を担うための独自手段であると同時に、機械にしかない戦術ギミックとして差別化する。

### 挑発属性との質的な違い

| | 挑発属性（有機） | スキル吸引（機械） |
|--|--|--|
| 仕組み | イラつき蓄積 → 確率で行動歪曲 | 直接ターゲット固定 |
| 効き方 | じわじわ、確率的 | 即座、確定的 |
| 制限範囲 | 攻撃系スキルのみ強制 | **全スキル系統**（攻撃・回復・付与等すべて） |
| 持続 | カウンター依存（減衰あり） | ターン数固定 |
| 副作用 | 暴走で火力UP（双方向リスク） | 未定（セクション6） |
| 解除 | 減衰 or 対象死亡 | ターン経過 or パッシブ解除 |
| テーマ | 感情の揺さぶり | 機械的・システム的制御 |

## 2. 基本仕様

### 2.1 概要

機械がスキルを使用してヒットした相手に対し、一定ターン数の間、その相手の攻撃ターゲットを自分（スキル使用者）に強制する。

```
機械M ──スキル（吸引属性あり）──→ 対象A, B, C（ZoneTraitに従う）
  → ヒットした各対象に対しステータス差判定
  → 通過した対象は設定ターン数、Mに攻撃が吸引される
```

### 2.2 対象範囲

スキルの `SkillZoneTrait` に従い、ヒットした全員に対して個別にステータス差判定を行う。判定を通過した対象のみ吸引状態になる。

- 単体スキル → 1体に吸引判定
- 全体スキル → 全員に吸引判定（それぞれ個別に成否あり）

### 2.3 ステータス差条件

挑発属性の十日能力補正（仕様書セクション5.4）と同じパターンを踏襲。

**被吸引者（吸引される側）から見て**、吸引者（機械）が3倍以上弱い場合、吸引が成立しない。

```
strengthRatio = machine.TenDayValuesSumForSkill() / target.TenDayValuesSumBase()

ratio > 0.33  → 吸引成立
ratio <= 0.33 → 吸引不成立（この機械ごときに制御されない）
```

挑発属性と異なり、弱くても最低1付与のような救済はない。**効くか効かないかの二値判定**。

### 2.4 実装方式: パッシブとして付与

吸引スキルがヒットすると、対象に**吸引パッシブ**が付与される。吸引の状態管理はパッシブシステムに委譲する。

```
機械M ──吸引スキル──→ 対象Aにヒット
  → ステータス差判定通過
  → 対象Aに「吸引パッシブ（吸引元=M）」を付与
  → パッシブのDurationTurnCounterで持続管理
```

#### なぜパッシブか（独自システムとの比較）

挑発属性は複数対象カウンター・確率的行動歪曲・暴走状態など複雑な状態を持つため、独自の `IrritationService` が必要だった。一方、吸引の状態は「誰に引きつけられているか（1体）+ 残りターン数」だけであり、パッシブの既存機能で十分に表現できる。

| 吸引の要件 | パッシブの既存機能 |
|-----------|------------------|
| 残りターン管理 | `DurationTurnCounter` |
| 上書き（最新優先） | 旧パッシブ除去 → 新パッシブ付与 |
| 吸引元の死亡時解除 | `OnRemove()` コールバック等 |
| 解除手段 | パッシブ解除スキルで外せる（対抗手段として機能） |

独自システムにするとターン管理・解除・表示を全て自前で書く必要があり、この程度のシンプルな状態に対しては過剰。

#### パッシブとしての追加要素

- 吸引パッシブは**吸引元（機械）への参照**をフィールドとして持つ
- 持続ターン数はスキルデータの値をパッシブの `DurationTurnCounter` に反映
- パッシブ枠を消費する → 枠コストが自然な制約になる
- パッシブ解除スキルで吸引を剥がせる → 対抗戦術として成立

### 2.5 吸引中の挙動

吸引パッシブが付いている対象は、**全スキル系統でターゲットが吸引元（機械）に強制される**。攻撃・回復・パッシブ付与等、スキル種別を問わず行動先が制限される。

- 挑発属性のイラつき攻撃は攻撃系のみ強制だが、吸引はより強力な機械的制御であり全行動を縛る

#### 範囲スキルとの関係

挑発属性と同じ設計判断を踏襲。`SkillZoneTrait` に関係なくターゲットが単体に強制される（`DirectedWill.One` に上書き）。範囲スキルが単体に絞られた場合、PowerSpreadは1人目の値（通常1.0）が適用されるため威力の損失はない。

## 3. 重ね掛けルール

**最新の吸引パッシブで上書き。** 旧パッシブが除去され、新パッシブが付与される。

```
敵機械E が味方Aに吸引をかける → Aに吸引パッシブ(元=E)付与 → Aの攻撃はEに向く
味方機械M がAに吸引をかける → 旧パッシブ(元=E)除去 → 新パッシブ(元=M)付与 → Aの攻撃はMに
```

この上書きルールにより、以下の戦術構造が生まれる:

- **敵機械がタンクとして吸引** → 味方は敵機械を殴り続けることを強いられる
- **味方機械が上書き吸引** → 味方の攻撃を自分（味方機械）に向けさせることで、敵機械の吸引を解除できる
- ただし味方機械に吸引された味方は、味方機械を殴る羽目になる → **リスクのある解除手段**
- **パッシブ解除スキル** → 吸引パッシブを直接剥がすことでも解除可能（機械がいなくても対抗できる）

## 4. 機械同士のロジック

### 方針: 統合

スキル吸引の仕組みに「対象が機械（`CharacterType.Machine`）ならステータス差チェックを省略」するルールを追加する。別枠の仕組みは作らない。

```
吸引スキルヒット時:
  対象が Machine → 無条件で吸引パッシブ付与
  対象が 非Machine → ステータス差判定（セクション2.3）→ 通過時のみ付与
```

### 統合を選んだ理由

- 吸引の効果自体（ターゲット強制・パッシブ付与・上書き）は機械同士でも変わらない
- 違いは「ステータス差チェックの有無」だけであり、分岐一つで済む
- 別枠にする場合、質的に異なる効果（ターゲット書き換え・相互ロック等）が必要だが、現時点でその必然性がない

## 6. バランス調整の方針

吸引には専用のリスク・デメリットを追加設計する必要は現時点でない。以下の既存制約がバランスの軸として機能する：

- **スキル枠の機会コスト**: 吸引スキルを撃つターン＝ダメージスキルを撃てないターン
- **パッシブ枠の消費**: 対象のパッシブ枠を圧迫する
- **パッシブ解除で対抗可能**: 吸引を剥がす手段が存在する
- **ステータス差で不成立**: 弱い機械には効かない
- **上書き構造のリスク**: 味方機械が解除に使うと味方を殴られる（セクション3）

これらに加え、スキルデータ側のパラメータ（持続ターン数、スキルの必要Pコスト、クールダウン等）でバランス調整が可能。追加リスクが必要かはプレイ感を見てから判断する。

## 7. スキルデータ上の位置付け（暫定）

挑発属性と同様に `SkillLevelData` のプロパティとして持たせる想定。吸引パッシブのテンプレート参照もスキルデータに含める。

| フィールド案 | 型 | 説明 |
|------------|-----|------|
| `HasAttractionAttribute` | `bool` | 吸引属性フラグ |
| `AttractionDurationTurns` | `int` | 付与する吸引パッシブの持続ターン数 |
