# 敵個体永続化 実装計画

全敵の個体状態（HP, broken, 武器適応率, 復活カウンター, 相性値変動等）をセーブ/ロードで永続化する。
現状のコンボ敵限定の永続化を全敵に拡張し、既存の復元不備（HP/broken未適用）も修正する。

---

## 前提決定事項

- **保存対象**: `encounterEnemies`キャッシュに存在する全敵（＝一度でもインスタンス化された敵）
- **ノード移動時**: 敵状態を破棄（option A）。キャッシュ＋セーブデータ上のその敵状態もクリア
- **復活カウンター**: セーブ/ロード後もカウント継続
- **敵パッシブ**: 明示的保存不要（毎エンカウント時にテンプレート→歩数シミュレーション→コンボ蓄積で再導出される）
- **相性値変動（`_bondDeltas`）**: 保存対象
- **`NowBattleProtocol`（現在の武器規格）**: 保存対象外。`ApplyWeapon(InitWeaponID)`で初期規格に戻る。適応率3フィールドは保存されるが、戦闘中のみの`SwitchProtocol`で変更された規格はリセットされる。実害は軽微（武器変更時の適応率は0.99f固定）

---

## 既存の問題（今回で同時修正）

`GetRuntimeEnemies`でコンボ敵のGUIDは復元されているが、**HP/MentalHP/brokenは復元されていない**。
`EnemyPersistenceData`に値は保存されているのにインスタンスへの適用コードが欠落。

---

## Step 1: EnemyPersistenceData のフィールド拡張

**ファイル**: `Assets/Script/Walk/Encounter/FriendshipComboData.cs`

既存フィールド（変更なし）:
- `EnemyGuid`, `EncounterId`, `TemplateIndex`, `IsBroken`, `HP`, `MentalHP`

追加フィールド:

| フィールド | 型 | 初期値 | 説明 |
|-----------|-----|--------|------|
| `LastEncounterProgress` | int | **-1** | 最後のエンカウント時歩数（-1=初回）。**0はレガシーセーブで危険（後述）** |
| `AdaptationRate` | float | **1.0f** | 武器適応率。**0.0fはレガシーセーブで攻撃力ゼロになるため危険** |
| `AdaptationStartRate` | float | **1.0f** | 規格切替時の初期適応率。同上 |
| `BattlesSinceProtocolSwitch` | int | 0 | 規格切替後の経過戦闘数 |
| `RebornRemainingSteps` | int | **-1** | 復活残り歩数。**0はレガシーセーブで即復活になるため危険** |
| `RebornLastProgress` | int | **-1** | 復活カウンター最終更新時の歩数 |
| `RebornState` | int | 0 | EnemyRebornState のint値（0=Idle） |
| `BondDeltas` | List\<BondDeltaEntry\> | null | 相性値変動（GUID→float辞書のシリアライズ形式） |

doc説明も「コンビメンバーの状態追跡用」→「敵個体の状態追跡用」に更新。

### レガシーセーブ互換の注意（レビュー指摘）

`JsonUtility`はJSON上に存在しないフィールドをC#デフォルト値（int=0, float=0.0f）にする。フィールド初期化子（`= -1`等）は無視される可能性がある。

**対策**: `RestoreState`内で追加のガード処理を入れる:
- `LastEncounterProgress == 0` → `-1`にフォールバック
- `AdaptationRate <= 0` → `1.0f`にフォールバック
- `AdaptationStartRate <= 0` → `1.0f`にフォールバック
- `RebornRemainingSteps == 0 && RebornState == 0(Idle)` → `-1`にフォールバック

## Step 3: 敵状態ストレージの整理

現状、`EnemyPersistenceData`は`FriendshipComboRegistry._enemyStates`に格納されている。
全敵の状態を扱うため、**GameContextレベルに敵状態ストレージを追加**する。

**ファイル**: `Assets/Script/Walk/GameContext.cs`

```
// 新規追加
private Dictionary<string, EnemyPersistenceData> _savedEnemyStates = new();
// key: "{EncounterId}:{TemplateIndex}"
```

メソッド追加:
- `ImportEnemyStates(List<EnemyPersistenceData> states)` — ロード時にセーブデータから辞書構築
- `GetSavedEnemyState(string encounterId, int templateIndex)` — GetRuntimeEnemies内から呼ぶ

### 二重管理の責務分離ルール（レビュー指摘）

`FriendshipComboRegistry._enemyStates` は友情コンボ固有のGUID追跡用としてそのまま残す。

**責務分離**:
- `_savedEnemyStates` (GameContext) = **ロード時の復元専用（read-only after import）**。ランタイム中に読み書きしない
- `ComboRegistry._enemyStates` = **コンボ管理専用**（GUID検索 + `MarkBroken`）
- ランタイム中の状態 = **`encounterEnemies`内のNormalEnemyインスタンス**が正
- エクスポート時 = ランタイムインスタンスから直接読む（`_savedEnemyStates`は参照しない）

この責務分離をコードコメントに明記すること。

## Step 4: GetRuntimeEnemies の復元ロジック強化

**ファイル**: `Assets/Script/Walk/GameContext.cs`

現状 `RestoreGuid()` のみ → 全フィールド復元に拡張。

```
var copy = enemy.DeepCopy();

// セーブデータから状態復元（コンボ・非コンボ問わず）
var saved = GetSavedEnemyState(encounter.Id, i);
if (saved != null)
{
    copy.RestoreGuid(saved.EnemyGuid);
    copy.RestoreState(saved);  // ← 新規メソッド

    // 復活状態の復元
    if (saved.RebornState > 0)
    {
        EnemyRebornManager.Instance.RestoreRebornState(
            copy, saved.RebornRemainingSteps,
            saved.RebornLastProgress, (EnemyRebornState)saved.RebornState);
    }
}
```

**ファイル**: `Assets/Script/Enemy/NormalEnemy.cs`

`RestoreState(EnemyPersistenceData saved)` を新規追加:
- HP, MentalHP 設定
- broken 設定（broken=trueかつHP>0の場合はHP=0を強制）
- `_lastEncounterProgress` 設定
- 武器適応率（`_adaptationRate`, `_adaptationStartRate`, `_battlesSinceProtocolSwitch`）設定
- `_bondDeltas` 復元
- レガシーセーブ互換のガード処理（Step 1参照）

**ファイル**: `Assets/Script/Battle/EnemyRebornManager.cs`

追加メソッド:
- `RestoreRebornState(NormalEnemy enemy, int remainingSteps, int lastProgress, EnemyRebornState state)` — ロード時に外部から復活カウンター状態を注入
- `ClearAll()` — 全エントリをクリア（ロード時・ノード移動時に使用）

**ファイル**: `Assets/Script/Battle/IEnemyRebornManager.cs`
- `IEnemyRebornManager` にも `RestoreRebornState`, `ClearAll` を追加

## テスト観点

1. **基本保存復元**: セーブ→ロード後に全敵のHP/broken/MentalHPが復元されること
2. **ReEncount**: ロード後の再エンカウントで`_lastEncounterProgress`が正しく、歩数シミュレーションが期待通りに動くこと
3. **復活カウンター**: broken敵がロード後も復活カウントを継続し、歩数到達で復活すること
4. **武器適応率**: ロード後に適応率が保持されること（1.0にリセットされないこと）
5. **相性値変動**: ロード後に`_bondDeltas`が復元されること
6. **ノード移動**: ノード移動後にキャッシュ・RebornManagerがクリアされ、次セーブに前ノードの敵が含まれないこと
7. **コンボ敵との互換**: 既存のコンボGUID復元が引き続き動作すること
8. **レガシーセーブ互換**: 新フィールドがないセーブデータをロードしたとき、ガード処理で安全に動作すること
9. **brokenガード**: broken=trueかつHP>0のデータがロードされた場合、HP=0に強制されること
10. **RebornManagerクリア**: ロード時・ノード移動時に旧エントリが残留しないこと
