# セーブロード関連問題

各仕様の調査で発見された、セーブ/ロードに起因する永続化の問題を集約する文書。
個別の仕様書では深追いせず、ここにリンクして逃がす。

### 対象外（意図的に後回し）

以下はセーブデータの中身ではなく「セーブ体験」に関わる項目であり、UI/ゲームフロー設計が固まるまで保留。

- セーブ/ロードのタイミング（自動セーブ、手動セーブのトリガー）
- セーブスロットUI・複数セーブ管理
- セーブデータのバージョン管理・互換性チェック
- タイムスタンプ・プレイ時間記録
- エラーリカバリ・破損セーブの復旧

## 1. 敵個体状態の永続化 → **解決済み**

`doc/敵個体永続化実装計画.md` に基づき実装完了。

`EnemyPersistenceData` を全敵対象に拡張し、`GetRuntimeEnemies` 内で `RestoreState` により復元。

| 項目 | 永続化 | 備考 |
|------|--------|------|
| GUID | ○ | `RestoreGuid()` |
| コンボ登録 | ○ | `FriendshipComboSaveData` |
| 全敵のbroken | ○ | `EnemyPersistenceData.IsBroken` + `RestoreState()` |
| HP / MentalHP | ○ | `EnemyPersistenceData` + `RestoreState()` |
| 武器適応率 | ○ | 3フィールド（AdaptationRate, AdaptationStartRate, BattlesSinceProtocolSwitch） |
| 復活カウンター | ○ | `EnemyRebornManager.RestoreRebornState()` |
| 相性値変動 | ○ | `BondDeltas`（List\<BondDeltaEntry\>） |
| エンカウント進行 | ○ | `LastEncounterProgress`（ReEncount歩数シミュレーション用） |
| パッシブ | 不要 | 毎エンカウント時にテンプレート→歩数シミュレーション→コンボ蓄積で再導出 |
| BattleMemory | 不要 | 設計意図あり（毎戦闘Clear） |

### ノード移動時の挙動

ノード移動時は敵状態を破棄する（`ClearEnemyCache()` + `EnemyRebornManager.ClearAll()`）。
戻ってきた場合はテンプレートから初期状態で再生成。

### レガシーセーブ互換

新フィールド未定義のセーブデータをロードした場合、`RestoreState` 内でガード処理:
- `LastEncounterProgress == 0` → `-1`（初回扱い）
- `AdaptationRate <= 0` → `1.0f`（完全適応）
- `AdaptationStartRate <= 0` → `1.0f`
- `broken == true && HP > 0` → `HP = 0`を強制
