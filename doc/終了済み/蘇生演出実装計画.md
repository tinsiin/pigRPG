# 蘇生演出 実装計画

蘇生（Angel）発生時の情報伝達を実装する。現状はHP代入のみで視覚フィードバックがゼロ。

関連仕様書:
- `doc/戦闘情報伝達/HP変動パス仕様書.md` — 蘇生節
- `doc/戦闘情報伝達/戦闘情報伝達の設計思想.md` — 4層の設計思想
- `doc/戦闘情報伝達/フロー数字仕様書.md` — 蘇生はフロー数字の対象外（劇的演出が重要なため）

## 現状の蘇生コード

```
BaseStates.ReactionSkill.cs
  └─ ReactionSkillOnBattle()
       └─ SkillType.DeathHeal ブランチ (L865-872)
            └─ ExecuteDeathHealFriendlyOnBattle(attacker, hitResult, ref isHeal)
                 ├─ Angel()                          ← HP代入のみ
                 ├─ isHeal = true
                 └─ PartyApplyConditionChangeOnCloseAllyAngel()  ← 仲間への波及
```

`Angel()` の処理:
- `m_BattleDeathProcessed = false`（死亡フラグリセット）
- `HP = float.Epsilon`（生存ライン復帰）
- パワーHighなら `HP = 30`

利用可能な変数:
- `attacker` — 蘇生術者（BaseStates）
- `this` — 蘇生対象（BaseStates）
- `this.BattleIcon` — 対象の BattleIconUI
- `attacker.NowUseSkill` — 使用スキル

備考: 現状、死亡アイコンに視覚変化がない（グレーアウト等なし）。HPバーが0になるだけ。

## 第4層: SchizoLog（実装方針確定）

`ExecuteDeathHealFriendlyOnBattle` 内、`Angel()` の直後に追加:

```csharp
AddBattleLog(attacker.CharacterName + "が" + CharacterName + "を蘇生した", important: true);
```

`important: true` で視認性を上げる（SchizoLog側の扱いに依存）。

## 第2層: エフェクト（採用: 光柱）

**コンセプト**: 下から光の柱が立ち上り、アイコンを包む。RPG王道の蘇生演出。

**KFXファイル**: `Assets/Resources/Effects/revival_pillar.json`

| 時間 | 演出 |
|---|---|
| 0.0〜1.2s | アイコン下部に白い光源（`base_glow`）が出現・膨張 |
| 0.05〜1.3s | 3本の光線（`pillar_left/center/right`）が時間差で下→上へ駆け上がる |
| 0.2〜1.0s | 白金リング（`ring_rise`）が下から上へ通過。アイコンを横切る |
| 0.28〜0.55s | リング通過時の瞬間フラッシュ（`flash`） |
| 0.25〜1.5s | 光の粒子（`rise_particles`）が上方向へ浮遊。角度250〜290°（ほぼ真上） |

**パラメータ**:
- canvas: 100 / fps: 30 / duration: 1.5s
- icon_rect: `{x:20, y:15, w:60, h:70}` — 光柱がアイコン外にはみ出す設計
- 全レイヤー加算ブレンド（光の表現）
- SE: 未設定（別途選定）

## コード変更箇所

エフェクト案が決まった後、以下を変更する:

### 1. KFXエフェクトファイル作成（完了）

`Assets/Resources/Effects/revival_pillar.json` — 光柱エフェクト

### 2. ExecuteDeathHealFriendlyOnBattle 修正

`BaseStates.ReactionSkill.cs` 内:

```csharp
void ExecuteDeathHealFriendlyOnBattle(BaseStates attacker, HitResult hitResult, ref bool isHeal)
{
    if (hitResult != HitResult.Hit) return;
    Angel();
    isHeal = true;

    // --- 追加ここから ---
    // 第2層: 蘇生エフェクト
    if (BattleIcon != null)
        EffectManager.Play("revival_pillar", BattleIcon);

    // 第4層: SchizoLog
    AddBattleLog(attacker.CharacterName + "が" + CharacterName + "を蘇生した", important: true);
    // --- 追加ここまで ---

    manager.MyGroup(this).PartyApplyConditionChangeOnCloseAllyAngel(this);
}
```

### 3. SE素材

蘇生用の効果音を `Assets/Resources/Audio/` に配置。KFXの `"se"` フィールドで参照。
