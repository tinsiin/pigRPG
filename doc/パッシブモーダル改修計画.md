# パッシブモーダル改修計画

## 現状の問題

1. **モーダルへの導線が分かりにくい**: CharaConfigのパッシブ表示で、最後のTMPフィールドに動的に`Button`コンポーネントをアタッチして省略ドット部分をタップ可能にしている。見た目にはボタンだと分からない
2. **モーダル内でもSmallPassiveNameが使われている**: モーダルは「全表示」の場面なので、正式名称（`PassiveName`）を使うべき

## 改修内容

### A. 専用ボタンの常時表示

CharaConfigのパッシブ表示エリアに、パッシブモーダルを開く**専用ボタン**を常時表示する。

**変更点**:
- CharaconfigController に新しい `[SerializeField] Button m_PassiveModalButton;` を追加
- `BindPassivesModalOpenButton()` の既存ロジック（最後のTMPにButton動的アタッチ）を削除
- 代わりに `m_PassiveModalButton.onClick` に `OpenPassivesModalForCurrent()` をバインド
- `UpdatePassiveButtonInteractable()` は `m_PassiveModalButton` の interactable を制御するよう変更
- ボタンのGameObject自体は常時表示（パッシブが0個の場合は interactable = false で無効化）

**Unity側の作業（手動）**:
- CharaConfigのパッシブ表示エリア付近にButtonを配置
- CharaconfigController の `m_PassiveModalButton` にワイヤリング

### B. モーダル内の表示を正式名称に変更

PassivesMordaleAreaController で使っている `PassiveTextUtils.BuildPassivesTokens()` を、`PassiveName` ベースの別メソッドに切り替える。

**変更点**:

1. `PassiveTextUtils` に `BuildPassivesFullNameTokens(BaseStates actor)` を追加:
   - `SmallPassiveName` の代わりに `PassiveName` を使用
   - フォールバック: `PassiveName` が空なら `SmallPassiveName` → それも空なら `fa{ID}`
   - ソート順は既存の `BuildPassivesTokens` と同じ（DurationTurn順）
   - トークン形式は `<正式名称>` のまま（`<>` リテラル表示の仕組みは変えない）

2. `PassivesMordaleAreaController.ShowFor()` の L66-68 を変更:
   ```csharp
   // Before
   string allTokens = m_DebugMode
       ? PassiveTextUtils.BuildDummyPassivesTokens(m_DebugCount, m_DebugPrefix)
       : PassiveTextUtils.BuildPassivesTokens(actor);

   // After
   string allTokens = m_DebugMode
       ? PassiveTextUtils.BuildDummyPassivesTokens(m_DebugCount, m_DebugPrefix)
       : PassiveTextUtils.BuildPassivesFullNameTokens(actor);
   ```

3. `CharaconfigController.OnPassiveButtonClicked()` 内の空チェックも同様に `BuildPassivesFullNameTokens` に合わせる（ただしパッシブの有無自体は変わらないので、実質的な動作差はない）

### C. 既存のSmallPassiveName表示はそのまま

CharaConfigの通常表示（`m_PassivesTexts[]`）は今まで通り `SmallPassiveName` のトークンで省略表示。変更なし。

## Unity側の手動作業

- CharaConfigシーン/プレハブにパッシブモーダル用ボタンを配置
- CharaconfigController の Inspector で `m_PassiveModalButton` をワイヤリング

## 実装順序

1. `PassiveTextUtils.BuildPassivesFullNameTokens()` 追加
2. `PassivesMordaleAreaController` のトークン生成を切り替え
3. `CharaconfigController` のボタン周りを改修
4. Unity側でボタン配置＋ワイヤリング（手動）
