# HP非成長型RPGとしてのpigRPG分析

> **ステータス**: 分析完了
> **関連**: `doc/HP仕様書.md` §1「HPは成長しない」

3つの専門視点（ゲームバランス / 戦闘メカニクス / プレイヤー体験）によるエージェントチーム議論の結果をまとめたもの。

---

## 1. 結論

**HP固定設計はpigRPGに合っている。**

単に「問題ない」レベルではなく、pigRPGの戦闘哲学（混戦、リスクリワード、レベルなし成長）を**構造的に支えている**設計判断である。

---

## 2. 全視点の一致点

### 2.1 差分型ダメージ式がHP固定の最大の味方

`(ATK-DEF) × SP + SP` という式構造は、DEFがATKに追いつけばSP無限成長の影響を `+SP` の線形保証のみに抑え込む。HP固定と最も相性の良い式。

- ATK=DEF時: ダメージ = SP（SPのみの線形成長）
- ATK<DEF時: 負の項がSPで増幅され、最低保証に収束
- ATK>DEF時: 差分にSPが乗算 → ここだけが崩壊リスク

### 2.2 耐久の多層化がHP成長の代替として機能

HP単一指標ではなく、7つの層で耐久を提供する設計:

| 層 | 機能 |
|---|---|
| DEF成長 | 十日能力経由の被ダメ減少 |
| VitalLayer | パッシブ由来の追加HP層 |
| パッシブ減衰 | ダメージ百分率削減 |
| HP割合制限 | MaxHP割合でHPをクランプ |
| 慣れ補正 | 同じ攻撃への耐性獲得 |
| 割り込みカウンター | 連続攻撃の中断 |
| AimStyle防御切替 | 適切な防ぎ方でDEFを最大化 |

→ HP固定はこの多層構造を「全ての層が常に意味を持つ」状態に保つ。

### 2.3 リスク感覚の恒常性

HP固定により、序盤から終盤まで「被弾の痛み」が一定。以下のシステムが一貫した重みを持ち続ける:

- ランダムターンの「運悪く連続で殴られた」場面のダメージ感覚
- 前のめりの回避半減リスク
- 0.7クランプ（不適切な防ぎ方）のペナルティ感
- 割り込みカウンターの価値

### 2.4 閾値設計の安定性

HP固定だからこそ、以下の閾値が**常に一定の意味を持ち続ける**:

| 閾値 | 内容 |
|---|---|
| 互角一撃 | MaxHP×34~66%のダメージ帯で生存判定 |
| TLOA減衰 | HP<38%でダメージ0.7倍 |
| 殺せない制限 | 3.4%止め（パッシブ5で10%止め） |
| 互角一撃復帰 | MaxHP×7%に復帰 |

HP成長型ではこれらの閾値設計が複雑化する。

## 4. 要注意ポイント

### 4.1 ATK-DEF差の崩壊点

差分型ダメージ式の特性上、DEF成長がATKに追いつかなくなる局面で、SP無限成長が固定HPを一瞬で消し飛ばす「崩壊」が起こりうる。この崩壊は**段階的ではなく突然起こる**。

- ATK成長はスキル使用で促進（攻撃行動自体がATK成長を促す）
- DEF成長は受動的で間接的な傾向
- 長期的にはATK成長がDEF成長をやや上回る可能性がある

→ これはHP固定設計自体の問題ではなく、**敵データの設計品質**の問題。ゲーム進行の想定期間内でATK-DEF差が暴走しないことの確認が必要。

### 4.2 互角一撃とHP固定の関係

互角一撃の発動条件であるMaxHP×34~66%ダメージ帯はHP固定なので絶対値として不変。

議論の過程で「SP成長で一撃ダメージが66%を超えると互角一撃が発動しなくなるのでは」という懸念が出たが、以下の理由で実質的な問題にはならない:

1. **DEFも成長する**: ATKとDEFは同じ十日能力プールから算出される。ATKだけが一方的に伸びてDEFが置き去りになるシナリオは、同格の敵同士では起きにくい
2. **十日能力合計1.6倍条件**: 互角一撃はそもそも「攻撃者の十日能力合計 ≤ 被害者の十日能力合計 × 1.6」の同格以下の敵にしか発動しない。同格ならATK/DEFバランスも近く、ダメージが66%を大幅に超え続ける状況自体が不自然
3. **ATK-DEF差が広がったらダメージが増える**: これはHP固定に限らずどんなRPGでも当たり前の話であり、HP固定固有の問題ではない

→ むしろHP固定により、互角一撃の発動圏（MaxHP×34~66%の絶対値）がプレイヤーにとって**常に予測可能**である点がポジティブ。「このくらいのダメージなら互角一撃が出るかも」と直感的に判断できる。

### 4.3 オーバーキルBroken閾値

MaxHP固定 → Broken閾値（MaxHP × OverKillBrokenRate）も固定。ATK成長で余剰ダメージが増えるとBroken頻度が上がる可能性がある。

**緩和要因**: 種別判定率（機械33%/生物93%）、攻撃者条件（生物→サイコパスのみ等）、余剰ダメージ最大値の制限（`GetOverkillOverflowMax()`）。

### 4.4 魔法の最低保証ダメージ

魔法ダメージ式の `SP × ATK × 0.09` 項はDEFに依存しない最低保証。SP成長で無限に伸びるため、長期的に魔法が固定HPに対して過剰になるリスクがある。

→ 係数0.09は小さいため致命的ではなく「削り」として機能する設計意図が見えるが、超長期では検証が必要。

## 5. プレイヤー体験の分析

### 5.1 感情カーブ: U字型の緊張感

| フェーズ | 体験 | HP固定の効果 |
|---|---|---|
| 序盤 | 全部痛い。緊張と集中 | MaxHP固定で被弾の重みが際立つ |
| 中盤 | DEF成長で耐えられるようになる | 「硬くなった」成長実感 |
| 終盤 | 火力インフレ vs 固定HP | 緊張の再燃。「先に殴ったほうが勝つ」=前のめりテーマの完成 |
| やりこみ | 駆け引きの深さを楽しむ | 「常に同じ土俵」の保証 |

### 5.2 成長実感の代替

HP数字が増えないことの代わりに:

- 37種の十日能力の個別成長
- SkillPowerの無限成長（数字として明確に見える）
- スキル解放
- 武器の取得・切替
- DEF上昇による「被ダメが減った」体感（HP増加より体感しやすい）
- VitalLayerによる視覚的な「バリア増加」

→ HP成長を加えると、既存の成長軸の認知的価値が薄まるリスクがある。

### 5.3 認知負荷の軽減

- MaxHP固定 → プレイヤーは「自分のHPは常にX」と覚えておける
- 互角一撃の発動圏（34~66%の絶対値）が不変 → 直感的判断が可能
- 回復量の計算が安定（勝利時15%、逃走時30%のMaxHP比が固定）

### 5.4 HP固定RPGの先例との比較

| ゲーム | HP設計 | pigRPGとの関係 |
|---|---|---|
| モンスターハンター | HP固定（アイテムで一時増加） | 類似：ビルドで乗り越える設計。pigRPGは「戦略的選択」が該当 |
| ダークソウル | HP成長するが敵も強化 | 実質相対的HP固定。pigRPGは文字通りの固定で設計意図を明確化 |
| ゼルダBotW | HP成長するが一撃死頻発 | 「成長しても油断できない」をよりエレガントに実現 |

pigRPG固有の文脈: ターン制RPGでHP固定は珍しいが、ランダムターンの不確実性がHP固定を「どの瞬間でも致命的になりうる」体験に昇華させている。

---

## 6. 設計上の意味

HP固定はpigRPGにおいて以下の設計的役割を果たしている:

1. **「HPを上げれば全て解決」という万能解の排除** → DEF・引き締め値・パワー管理・慣れ活用という質的な成長をプレイヤーに求める
2. **レベルなし哲学との一貫性** → レベルアップが存在せず、十日能力がスキル使用で個別成長する設計の中で、HPが特別扱いされない
3. **閾値設計の簡素化** → 互角一撃、TLOA減衰、殺せない制限のバランス調整が一度で済む
4. **敵設計の容易化** → 敵のATK設計時にプレイヤーのHP成長を考慮する必要がない
