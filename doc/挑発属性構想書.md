# 挑発属性構想書

元メモ: obsidian `豚岬フォルダ/挑発属性.md`
ステータス: 構想段階（仕様詰め中）

---

## 1. 概要

「挑発属性」はスキルの性質の一つ。挑発属性を持つスキルで攻撃すると、被攻撃者に「イラつき」状態異常を付与する。イラつきは攻撃対象の行動を歪め、冷静さを失わせる。

```
A（挑発者）──挑発属性スキルで攻撃──→ B（被挑発者）
                                       │
                                       ▼
                                 B に「イラつき→A」が蓄積
```

---

## 2. イラつきの基本仕様

### 2.1 蓄積構造

- イラつきは **「誰に対してどれだけイラついているか」のカウンター**
- 1キャラが複数の相手に対するイラつきを同時に持てる
- 挑発属性スキルで攻撃されるたびに、該当キャラへのイラつきカウントが+1

```
例: Bの状態
  イラつき→A: 3
  イラつき→C: 1
  合計イラつき: 4
  UI表示: <イラつき→A>（最大の相手を表示）
```

### 2.2 付与制限

| 条件 | 結果 |
|------|------|
| 対象が機械種別（`CharacterType.Machine`） | イラつきは付かない |
| 対象の精神属性による制限 | 効き方・付与確率が異なる（詳細は後述、未確定） |

### 2.3 減衰

- 毎ターン、精神属性に応じた減衰率でイラつきカウントが減少する
- 精神属性別の減衰テーブルは未確定（後で設計）

---

## 3. イラつきの効果

### 3.1 イラつき攻撃の発生条件

イラつきを持つキャラのターン開始時に**確率判定**が行われる。この判定は `TargetProbability` とは独立した専用ロジック。

#### パターン1: 確率ヒット → ターゲット強制 + スキル制限

```
ターン開始 → イラつき確率判定 → ヒット！
  → ターゲットがイラつき最大の相手に強制固定
  → スキル選択は可能だが、攻撃系スキルのみに制限（回復・補助は選べない）
  → 強制前のめり + 命中補正なし
```

プレイヤーも敵も関係なく、確率にヒットしたら**殴る相手は選べないが、何で殴るかは選べる**。ただし怒りで殴りに行ってるので攻撃系のみ。

#### パターン2: 確率ミス → 通常行動だが、イラつき対象を選んだ場合

```
ターン開始 → イラつき確率判定 → ミス
  → 通常のコマンド選択（UI選択 or 敵AI）
  → もしイラつき対象を攻撃先に選んだ場合
    → 「イラつき攻撃」として処理される（強制前のめり + 命中補正なし）
```

自分の意志で選んだとしても、イラつき対象への攻撃はイラつきに引きずられた攻撃として扱う。

#### まとめ

| 状況 | 対象選択 | スキル選択 | 前のめり | 命中補正 |
|------|---------|----------|---------|---------|
| 確率ヒット | イラつき最大の相手（強制） | 攻撃系のみに制限 | 強制 | なし |
| 確率ミス + イラつき対象を自分で選択 | 自分で選択 | 制限なし | 強制 | なし |
| 確率ミス + イラつき対象以外を選択 | 自分で選択 | 制限なし | 通常 | 通常 |

**確率の具体値:** 未確定。イラつき合計量や精神属性で変動する想定。

### 3.2 ターゲット偏向の実装方針

- 既存の `TargetProbability` は「自分が狙われる確率」を制御するもので、「特定の相手を狙いに行く」制御には不向き
- イラつきパッシブ側で **ターゲット選択を上書きする専用処理** が必要
- 敵AI: `Plan()` の前にイラつき確率判定 → ヒットならPlanをスキップして強制ターゲット
- プレイヤー: 確率ヒット時はUI選択をスキップして自動攻撃。ミス時は通常選択だが、イラつき対象を選んだ場合はイラつき攻撃扱い

### 3.3 イラつき暴走（合計が閾値超過）

**イラつきの合計カウントが一定数を超えると、対象に関係なく全体的に暴走する。**

| 効果 | 詳細 |
|------|------|
| **火力上昇** | 全攻撃のATKに1.25〜1.75倍の補正（イラつき合計量に応じて段階的？） |
| **全攻撃で強制前のめり** | イラつき対象以外を狙う場合も前のめりになる |

暴走は「ブチギレ状態」であり、火力は上がるが防御面のリスク（前のめりによる回避率低下等）を負う。暴走中の強制前のめりは3.1のイラつき攻撃とは独立して常時発生する。

---

## 4. 挑発者（A）側の効果

挑発属性スキルの使用者自身にもリスクがある。

> 「挑発属性の攻撃では自分が一定ターン数数割の確率で攻撃されやすくなる」（元メモ）

これはA自身に何らかの「ヘイト上昇」効果が付くことを意味する。ただし `TargetProbability` を使うとBだけでなく全敵からAが狙われてしまうため、**イラつきパッシブのターゲット偏向に内包される形の方が自然**。

つまり:
- Aに別途パッシブを付けるのではなく、Bの「イラつき→A」自体がAへの攻撃偏向を生む
- A側への独立した効果は不要かもしれない → **要検討**

---

## 5. 機械系の特殊スキル（別枠）

> 「機械系の敵のスキルにはそういうの関係なしに、攻撃を自分に吸い付ける物がある」（元メモ）

これはイラつきとは別の仕組み。機械はイラつきが付かない代わりに、スキル効果として強制ターゲット吸引を持つ。挑発属性の実装とは独立して扱う。

## 6. 既存システムとの接続点

### 使えるもの

| 既存システム | 挑発での用途 |
|-------------|-------------|
| パッシブ基盤（`BasePassive`） | イラつきパッシブの親クラス |
| `OkType`（種別フィルタ） | 機械に付かない制御 |
| `OkImpression`（精神属性フィルタ） | 精神属性ごとの効き方 |
| 前のめり制御 | 強制前のめりの実行 |
| `DurationTurn` / 減衰 | イラつきの持続管理 |

### 新規実装が必要なもの

| 要素 | 理由 |
|------|------|
| イラつきカウンター（対象別） | 既存パッシブに「誰に対して」の概念がない |
| ターゲット偏向の専用ロジック | `TargetProbability`では「特定の相手を狙う」制御ができない |
| イラつき暴走判定 | 合計閾値による火力補正・全面前のめり |
| 精神属性別減衰テーブル | 既存の減衰は全精神属性で共通 |
