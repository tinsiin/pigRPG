## 1. 概要

「挑発属性」はスキルの性質の一つ。挑発属性を持つスキルで攻撃すると、被攻撃者に「イラつき」状態異常を付与する。イラつきは攻撃対象の行動を歪め、冷静さを失わせる。

```
A（挑発者）──挑発属性スキルで攻撃──→ B（被挑発者）
                                       │
                                       ▼
                                 B に「イラつき→A」が蓄積
```

### イラつき付与の2つの経路

| 経路 | 仕組み | 特徴 |
|------|--------|------|
| **直接付与（スキル属性）** | 挑発属性を持つスキルが対象に効いたら（完全回避以外）イラつき加算 | どのSkillTypeでも機能する（攻撃/回復/パッシブ付与等） |
| **持続付与（パッシブ経由）** | 挑発パッシブを付与するスキル → パッシブのOnNextTurn()で毎ターンイラつき加算 | じわじわ溜まる持続型の挑発 |

#### 直接付与の実装方針

全SkillTypeで共通の `HitResult` が返るため、パイプラインの違いを意識する必要はない。スキル実行の共通後処理として：

```
スキル実行完了（どのSkillTypeでも）
  ↓
挑発属性チェック: スキルに挑発属性がある？
  ↓ YES
HitResultチェック: CompleteEvade以外？（かすりでも付く）
  ↓ YES
対象にイラつきカウント加算（付与量はスキルごとに設定）
```

- 攻撃系: IsReactHIT() → HitResult で判定
- 回復系/善意攻撃: SkillHitCalc() → HitResult で判定
- パッシブ付与: hitResult == Hit で判定
- **全て共通の `HitResult` を返すため、完全回避以外なら一律で挑発が付く**

## 2. イラつきの基本仕様

### 2.1 蓄積構造

- イラつきは **「誰に対してどれだけイラついているか」の整数カウンター**
- 1キャラが複数の相手に対するイラつきを同時に持てる
- 挑発属性スキルで攻撃されるたびに、該当キャラへのイラつきカウントが加算される
- **付与量はスキルごとに設定可能**（整数値、スキルデータに持たせる）

```
例: Bの状態（スキルAの付与量=2、スキルCの付与量=1の場合）
  イラつき→A: 4
  イラつき→C: 1
  合計イラつき: 5
  UI表示: <イラつき→A>（最大の相手を表示）
```

### 2.2 付与制限

| 条件 | 結果 |
|------|------|
| 対象が機械種別（`CharacterType.Machine`） | イラつきは付かない |
| 対象の精神属性による制限 | 付与量が精神属性ごとに補正される（セクション8.2） |

### 2.3 減衰

- **3ターンごとに判定**。判定時にイラつきカウントが減少する
- 基本減衰量: **1**（3ターンごとに各対象へのイラつきが1減る）
- 精神属性別の減衰テーブルで減衰量が変わる場合も、判定タイミングは3ターンごとで統一
- 0以下になったイラつきは消滅する

## 3. イラつきの効果

### 3.1 イラつき攻撃の発生条件

イラつきを持つキャラのターン開始時に**確率判定**が行われる。この判定は `TargetProbability` とは独立した専用ロジック。

#### パターン1: 確率ヒット → ターゲット強制 + スキル制限

```
ターン開始 → イラつき確率判定 → ヒット！
  → ターゲットがイラつき最大の相手に強制固定
  → スキル選択は可能だが、攻撃系スキルのみに制限（回復・補助は選べない）
  → 強制前のめり + 命中補正なし
```

プレイヤーも敵も関係なく、確率にヒットしたら**殴る相手は選べないが、何で殴るかは選べる**。ただし怒りで殴りに行ってるので攻撃系のみ。

#### パターン2: 確率ミス → 通常行動だが、攻撃系スキルでイラつき対象を選んだ場合

```
ターン開始 → イラつき確率判定 → ミス
  → 通常のコマンド選択（UI選択 or 敵AI）
  → もし攻撃系スキルでイラつき対象を攻撃先に選んだ場合
    → 「イラつき攻撃」として処理される（強制前のめり + 命中補正なし）
  → 非攻撃系スキル（回復・補助等）でイラつき対象を選んだ場合
    → イラつき攻撃にはならない（通常処理）
```

自分の意志で選んだとしても、攻撃系スキルでイラつき対象を狙えばイラつきに引きずられた攻撃として扱う。ただし回復や補助でイラつき対象を選んでも、それは「殴りに行く」行為ではないのでイラつき攻撃にはならない。

#### まとめ

| 状況 | 対象選択 | スキル選択 | 前のめり | 命中低下免除 |
|------|---------|----------|---------|---------|
| 確率ヒット | イラつき最大の相手（強制） | 攻撃系のみに制限 | 強制 | 免除されない |
| 確率ミス + 攻撃系スキルでイラつき対象を選択 | 自分で選択 | 攻撃系（条件） | 強制 | 免除されない |
| 確率ミス + 非攻撃系スキルでイラつき対象を選択 | 自分で選択 | 制限なし | 通常 | 通常 |
| 確率ミス + イラつき対象以外を選択 | 自分で選択 | 制限なし | 通常 | 通常 |

#### 命中低下免除について

通常の前のめりでは「前に出たから当たりやすい」ため、後衛ペナルティ（攻撃相手のAGI×0.2分の命中低下）が免除される。しかしイラつき攻撃の強制前のめりは怒りで雑に突っ込んでいるだけなので、**この命中低下免除が適用されない**（前のめり状態なのに命中低下ペナルティが発生する）。

なお、リカバリーターン2倍速は前のめりの物理的な位置関係から生まれる効果なので、イラつき攻撃でもキャンセルされない。

#### 精神属性による例外

| 精神属性 | 例外内容 |
|---------|---------|
| Psycho（サイコ） | イラつき攻撃時でも**命中低下免除が通常通り適用される**（冷静に狙える） |

**発動確率の計算式:**

```
発動値 = 最大相手のカウント + (合計 - 最大相手のカウント) / 4
発動確率 = 発動値 × Demeanor別係数%（デフォルト6%、テーブルはセクション8.3）
```

| 最大相手 | 残り | 発動値 | 確率 |
|---------|------|--------|------|
| 1 | 0 | 1.0 | 6% |
| 3 | 0 | 3.0 | 18% |
| 5 | 2 | 5.5 | 33% |
| 7 | 3 | 7.75 | 46.5% |
| 8 | 0 | 8.0 | 48% |

※ やり場のないイラつきは発動確率計算には含めない（対象不在のため）。

### 3.2 実装方針

#### イラつき確率判定の実行位置

イラつき確率判定は**ターン管理層**（`SkillActRun()` より上流）で、敵・プレイヤー統一で行う。

```
ターン管理層（敵もプレイヤーも共通）
  │
  ├─ イラつき確率判定
  │    ├ ヒット → 「イラつき攻撃フラグ」ON + ターゲット確定
  │    └ ミス  → フラグOFF、通常フロー
  │
  ▼
敵: SkillActRun() → Plan()
  └ フラグON時: 攻撃系スキルのみに候補を絞ってPlan()、ターゲットは確定済み
  └ フラグOFF時: 通常のPlan()

プレイヤー: UI選択フロー
  └ フラグON時: ターゲット選択UIをスキップ、スキル選択は攻撃系のみ表示
  └ フラグOFF時: 通常のUI選択
```

**統一する理由:** プレイヤーと敵で同じ「イラつき」を同じロジックで処理できる。AI層の外（ターン管理層）にフラグを立てて、下流の各システム（AI/UI）がそのフラグを参照する形。

#### TargetProbabilityとの関係

- 既存の `TargetProbability` は「自分が狙われる確率」を制御するもので、「特定の相手を狙いに行く」制御には不向き
- イラつきのターゲット偏向は `TargetProbability` とは独立した専用処理

#### 賢い敵の場合

イラつき確率判定がミスした場合、思慮レベルの高い敵AIは `Plan()` 内で「自分がイラついている」ことを認識し、あえてイラつき対象を避ける戦術も書ける。ただし確率ヒット時は思慮レベルに関係なく強制（頭でわかっていても体が動く）。

### 3.3 イラつき暴走（合計が閾値以上）

**イラつきの合計カウントが暴走閾値以上になると、対象に関係なく全体的に暴走する。**

- **暴走閾値はキャラごとに設定可能**（整数値、デフォルト: **8**）
- 短気なキャラは閾値を低く、冷静なキャラは高く設定できる

| 効果 | 詳細 |
|------|------|
| **火力上昇** | 全攻撃のATKに倍率補正（下記テーブル参照） |
| **全攻撃で強制前のめり** | イラつき対象以外を狙う場合も前のめりになる |

#### 火力倍率

暴走閾値を超えた超過量に応じて段階的に上昇:

| 超過量 | 倍率 | 例（閾値8の場合） |
|--------|------|------------------|
| 0（ちょうど閾値） | 1.25倍 | 合計8 |
| 1〜2 | 1.40倍 | 合計9〜10 |
| 3〜4 | 1.55倍 | 合計11〜12 |
| 5以上 | 1.75倍 | 合計13以上 |

※ この段階テーブルは仮値。実際のプレイ感で調整する。

暴走は「ブチギレ状態」であり、火力は上がるが防御面のリスク（前のめりによる回避率低下等）を負う。暴走中の強制前のめりは3.1のイラつき攻撃とは独立して常時発生する。

**暴走の合計値には「やり場のないイラつき」も加算される。**（3.4参照）

#### 暴走とイラつき攻撃の差別化

暴走（イライラ状態）はイラつき攻撃（3.1）とは意図的に効果が異なる:

| | イラつき攻撃（3.1） | 暴走/イライラ状態（3.3） |
|--|--|--|
| 前のめり | 強制 | 強制 |
| 命中低下免除なし | あり（雑に突っ込む） | なし（命中ペナは通常通り免除） |
| 火力上昇 | なし | あり（超過量で段階的） |
| ターゲット強制 | 最大イラつきの相手に固定 | なし（誰を殴ってもいい） |

暴走では命中低下免除が通常通り適用され、ターゲットも自由。イラつき攻撃の「制御不能感」に対して、暴走は「強くなったが前に出ざるを得ない」というリスクリワード。

#### 設計意図: 挑発の双方向リスク

挑発属性の狙いは「イラつかせた側も一方的に有利にはならない」こと。

- **挑発された側**: イラつき攻撃でターゲットや行動を強制される不自由さがある。暴走まで達すると火力は上がるが常時前のめりで被弾リスクを背負う。制御不能にされる感がある
- **挑発した側**: 相手をイラつかせすぎて暴走させると、火力上昇という形で自分たちが被害を受ける。挑発属性で追い込めば追い込むほど相手が強くなるデメリットがある
- **結果**: 挑発は「タンクが敵の攻撃を引きつける」一方的な盾ではなく、お互いにリスクを抱える駆け引きになる。イラつかせすぎないよう加減するか、暴走覚悟で一気に畳みかけるかの判断が生まれる

### 3.4 イラつき対象の死亡時処理

#### 基本: 対象が死んだらそのイラつきは消える

```
B: イラつき→A: 5, イラつき→C: 2
Aが死亡 → イラつき→A: 消滅
B: イラつき→C: 2
```

殺したのが誰かに関係なく、対象の死亡でそのカウンターは消える。

#### 例外: 「やり場のないイラつき」への変換

以下の条件を**全て**満たす場合、通常の消滅ではなく「やり場のないイラつき」に変換される:

1. その対象(A)へのイラつき値が **暴走閾値以上**（対象単体の値で判定、合計ではない）
2. **B自身の攻撃またはパッシブで倒したのではない**（他の味方が倒した等）

```
例: 暴走閾値 = 8
B: イラつき→A: 9

ケース①: 味方DがAを倒した
  → Aへのイラつき9 ≧ 閾値8、かつB自身が倒してない
  → やり場のないイラつき→(A): 9 に変換

ケース②: B自身の攻撃でAを倒した
  → B自身が倒した → 変換なし、すっきり消滅

ケース③: Bのパッシブ効果でAが死亡した
  → B自身のパッシブで倒した → 変換なし、すっきり消滅

ケース④: Aへのイラつきが6（閾値8未満）で味方DがAを倒した
  → 閾値未満 → 変換なし、普通に消滅
```

#### やり場のないイラつきの性質

| 性質 | 詳細 |
|------|------|
| **暴走の合計値に加算** | 通常イラつきと合算して暴走閾値判定に影響する |
| **イラつき攻撃は発生しない** | 対象が不在なので発動確率計算には含めない |
| **対象情報は保持** | 復活等で再登場した時のために記録する |
| **通常の減衰は適用されない** | 対象不在のため3ターンごとの減衰対象外 |

#### やり場のないイラつきの解消条件

**条件①: 元の対象に攻撃でダメージを与える**
- 復活等で再登場した対象に、**B自身の攻撃（パッシブは対象外）で最大HPの3%以上のダメージ**を一度に与えたら、やり場のないイラつきは全消滅

**条件②: 同じ対象から再度挑発される**
- やり場のないイラつきが通常のイラつきカウントに**即座に全加算**されて消滅

```
例: B: やり場のないイラつき→(A): 6
Aが復活して挑発属性スキル（付与量2）でBを攻撃 →
  通常付与: イラつき→A: 2
  やり場のないイラつき全加算: +6
  結果: イラつき→A: 8
  やり場のないイラつき→(A): 消滅
```

## 4. 挑発者（A）側の効果 → **不要（確定）**

元メモの「自分が攻撃されやすくなる」は、Bのイラつきによって**結果的にAが狙われる**ことを指しており、A自身に別途効果を付ける必要はない。Bの「イラつき→A」が攻撃偏向を生むことで元メモの意図は満たされている。

---

## 5. 機械系の特殊スキル（別枠）

> 「機械系の敵のスキルにはそういうの関係なしに、攻撃を自分に吸い付ける物がある」（元メモ）

これはイラつきとは別の仕組み。機械はイラつきが付かない代わりに、スキル効果として強制ターゲット吸引を持つ。挑発属性の実装とは独立して扱う。

## 6. 既存システムとの接続点

### 使えるもの

| 既存システム | 挑発での用途 |
|-------------|-------------|
| パッシブ基盤（`BasePassive`） | イラつきパッシブの親クラス |
| `OkType`（種別フィルタ） | 機械に付かない制御 |
| `OkImpression`（精神属性フィルタ） | 精神属性ごとの効き方 |
| 前のめり制御 | 強制前のめりの実行 |
| `DurationTurn` / 減衰 | イラつきの持続管理 |

### 新規実装が必要なもの

| 要素 | 理由 |
|------|------|
| イラつきカウンター（対象別） | 既存パッシブに「誰に対して」の概念がない |
| ターゲット偏向の専用ロジック | `TargetProbability`では「特定の相手を狙う」制御ができない |
| イラつき暴走判定 | 合計閾値による火力補正・全面前のめり |
| 精神属性別減衰テーブル | 既存の減衰は全精神属性で共通 |

基本式の6%係数をDemeanorごとに差し替える。`最終確率 = 発動値 × 係数%`（デフォルト: 6%）。

## 9. 実装計画

### 9.0 設計方針

**IrritationSystemはパッシブ体系から完全に独立した専用システムとして実装する。**

```
IrritationSystem（完全独立）
  データ: BaseStatesにフィールド追加（Dictionary等）
  ロジック: IrritationService（戦闘レベルサービス）
  表示: 専用UIコンポーネント（パッシブ表示とは無関係）

パッシブ体系との接点はIrritationCausePassiveの1点のみ。イラつきの全データ・全ロジック・全表示はパッシブから独立。

### 9.2 実装詳細の決定事項

#### 9.2.1 イラつき攻撃フラグの管理

イラつき攻撃フラグは2つのタイミングでセットされる:

```
パターン1（確率ヒット）:
  ターン管理層でIrritationService.JudgeTrigger() → ヒット
  → フラグON + ターゲット確定
  → 以降のスキル選択・実行がフラグを参照

パターン2（確率ミス + 自発的にイラつき対象を攻撃）:
  ターン管理層でJudgeTrigger() → ミス → フラグOFF
  → 通常のスキル選択・ターゲット選択
  → スキル実行直前: 攻撃系スキル かつ ターゲットがイラつき対象か判定
  → 該当すればフラグを後付けON
```

フラグの居場所: **BaseStatesにターン一時フィールドとして保持**。ターン開始時にリセット。

```
bool IsIrritationAttack;         // イラつき攻撃フラグ
BaseStates IrritationForcedTarget; // 確率ヒット時の強制ターゲット（null=確率ミス）
```

- `IrritationForcedTarget != null` → パターン1（確率ヒット、ターゲット強制）
- `IrritationForcedTarget == null && IsIrritationAttack` → パターン2（自発選択だがイラつき攻撃扱い）
- `!IsIrritationAttack` → 通常攻撃

#### 9.2.2 減衰カウンターの管理

「3ターンごと」の判定用カウンターは**BaseStatesにフィールドとして保持**。

```
int _irritationDecayTurnCounter;  // 0→1→2→0（3で判定してリセット）
```

- 毎ターン `OnNextTurn` 相当のタイミングでインクリメント
- 3に達したら `IrritationService.Decay()` を呼び出してリセット
- イラつきを一切持っていないキャラはカウント不要（持った時点で0開始）

#### 9.2.3 暴走の火力倍率の適用箇所

暴走の火力倍率は **`GetSpecialPercentModifier(StatModifier.Atk)` 経由ではなく、スキルダメージ計算の最終段で専用乗算** とする。

理由:
- `GetSpecialPercentModifier` はバフ/デバフ用の汎用枠で、イラつき暴走を混ぜると他のATK補正と区別がつかなくなる
- 暴走倍率はイラつき合計の超過量で動的に変わるため、固定値を入れるSpecialModifierの仕組みと合わない
- スキルダメージ計算時に `IrritationService.GetRageMultiplier(chara)` を呼んで乗算する方が明示的

```
最終ダメージ = 通常のダメージ計算結果 × IrritationService.GetRageMultiplier(attacker)
```

`GetRageMultiplier()` はイラつき合計が暴走閾値未満なら `1.0` を返す。

  └→ フェーズ6（表示）
```

フェーズ1が全ての土台。フェーズ2・5・6は独立して並行実装可能。フェーズ3→4は順序依存あり。
