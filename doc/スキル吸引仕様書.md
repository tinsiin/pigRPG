# スキル吸引仕様書

## 1. 概要

「スキル吸引」は機械種別（`CharacterType.Machine`）専用のターゲット制御メカニクス。吸引属性を持つスキルでヒットした相手に吸引パッシブを付与し、一定ターン数の間、その相手の**全スキル系統**のターゲットを自分（吸引元）に強制する。

```
機械M ──吸引属性スキルで攻撃──→ 対象A
                                  │
                                  ▼
                            Aに吸引パッシブ(元=M)付与
                            → Aの全行動がMに向く
```

### 挑発属性との質的な違い

| | 挑発属性（有機） | スキル吸引（機械） |
|--|--|--|
| 仕組み | イラつき蓄積 → 確率で行動歪曲 | 直接ターゲット固定 |
| 効き方 | じわじわ、確率的 | 即座、確定的 |
| 制限範囲 | 攻撃系スキルのみ強制 | **全スキル系統**（攻撃・回復・付与等すべて） |
| 持続 | カウンター依存（減衰あり） | ターン数固定 |
| 副作用 | 暴走で火力UP（双方向リスク） | なし（パラメータ調整で対応） |
| 解除 | 減衰 or 対象死亡 | ターン経過 or パッシブ解除 or 吸引元死亡 |
| テーマ | 感情の揺さぶり | 機械的・システム的制御 |

## 2. 付与条件

### 2.1 発動条件

吸引属性を持つスキルが対象にヒット（CompleteEvade以外）すると、対象ごとにステータス差判定を行う。

### 2.2 対象範囲

スキルの `SkillZoneTrait` に従い、ヒットした全員に対して個別に判定を行う。

- 単体スキル → 1体に判定
- 全体スキル → 全員に判定（それぞれ個別に成否あり）

### 2.3 ステータス差条件

| 対象の種別 | 条件 |
|-----------|------|
| 非Machine（有機キャラ） | `source.TenDayValuesSumForSkill() / target.TenDayValuesSumBase() > 0.33` で成立 |
| Machine（機械） | **無条件で成立**（通信制御的ニュアンス） |

挑発属性と異なり、弱くても最低1付与のような救済はない。**効くか効かないかの二値判定**。

```
strengthRatio = machine.TenDayValuesSumForSkill() / target.TenDayValuesSumBase()

ratio > 0.33  → 吸引成立
ratio <= 0.33 → 吸引不成立
```

ゼロ除算防止: 被吸引者の十日能力総量が0以下の場合、無条件で成立。

### 2.4 付与制限

| 条件 | 結果 |
|------|------|
| 使用者が非Machine（有機キャラ） | 付与されない（機械専用） |
| 自分自身への吸引 | 付与されない |
| 対象が3倍以上強い（非Machine） | 付与されない |
| CompleteEvade | 付与されない |

使用者の種別制限はスキルデータ側ではなく `AttractionService.TryApply` 内のランタイムチェックで担保する。`SkillLevelData` は全種別共通のフラットなデータ構造であり、挑発属性の「機械には効かない」が `IrritationService.Add` 内のtargetチェックで担保されているのと同じパターン。

## 3. 吸引パッシブの仕様

### 3.1 実装方式

吸引はパッシブシステムに委譲する。`AttractionPassive`（`BasePassive` サブクラス）が対象に付与され、パッシブの存在自体が吸引状態を示す。

- **PassiveManager登録**: ID `26` でテンプレート登録（`AttractionService` 内の定数 `AttractionPassiveID` と一致）
- 付与時は `PassiveManager.GetAtID(26)` → `DeepCopy()` → スキルデータの持続ターン数で `DurationTurn`/`DurationTurnCounter` を上書き → `ApplyPassive`
- `_grantor` = 吸引元（機械）
- `DurationTurnCounter` = 残りターン数
- `IsBad = true`（デバフ扱い）
- `OkType = Everything`、`OkImpression = Everything`（機械制限はsource側で判定するためtarget側は無制限）

### 3.2 吸引中の挙動

吸引パッシブが付いている対象は、**全スキル系統でターゲットが吸引元に強制される**。

- 攻撃・回復・パッシブ付与等、スキル種別を問わず行動先が制限される
- `SkillZoneTrait.SelfSkill`（自己対象スキル）は吸引の影響を受けない
- 吸引はイラつき攻撃より優先される

#### 範囲スキルとの関係

`SkillZoneTrait` に関係なくターゲットが単体に強制される（`DirectedWill.One` に上書き）。PowerSpreadは1人目の値（通常1.0）が適用されるため威力の損失はない。

### 3.3 解除条件

| 条件 | 詳細 |
|------|------|
| ターン経過 | `DurationTurnCounter` が0になると自動解除 |
| 吸引元の死亡 | 吸引元が死亡すると全キャラから該当パッシブを除去 |
| パッシブ解除スキル | デバフ扱いのため、パッシブ除去系スキルで剥がせる |

## 4. 重ね掛けルール

**最新の吸引パッシブで上書き。** 旧パッシブが除去され、新パッシブが付与される。

```
敵機械E が味方Aに吸引をかける → Aに吸引パッシブ(元=E)付与 → Aの全行動はEに向く
味方機械M がAに吸引をかける → 旧パッシブ(元=E)除去 → 新パッシブ(元=M)付与 → Aの全行動はMに
```

戦術構造:

- **敵機械がタンクとして吸引** → 味方は敵機械に全行動を強いられる
- **味方機械が上書き吸引** → 敵機械の吸引を解除できるが、味方が味方機械を殴る羽目になる（リスクのある解除手段）
- **パッシブ解除スキル** → 吸引パッシブを直接剥がすことでも解除可能（機械がいなくても対抗できる）

## 5. バランス調整の方針

吸引には専用のリスク・デメリットを追加設計する必要は現時点でない。以下の既存制約がバランスの軸として機能する:

- **スキル枠の機会コスト**: 吸引スキルを撃つターン＝ダメージスキルを撃てないターン
- **パッシブ枠の消費**: 対象のパッシブ枠を圧迫する
- **パッシブ解除で対抗可能**: 吸引を剥がす手段が存在する
- **ステータス差で不成立**: 弱い機械には効かない
- **上書き構造のリスク**: 味方機械が解除に使うと味方を殴られる（セクション4）

これらに加え、スキルデータ側のパラメータ（持続ターン数、スキルの必要Pコスト、クールダウン等）でバランス調整が可能。追加リスクが必要かはプレイ感を見てから判断する。
