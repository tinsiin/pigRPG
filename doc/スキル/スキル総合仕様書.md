# スキル総合仕様書

スキルシステム全体の設計方針と、各詳細仕様書へのリンクをまとめた文書。

---

## 1. 設計原則：スキル入手はイベント駆動

**スキルはポイント交換や撃破報酬で入手するものではない。これは意図的な設計判断である。**

pigRPGにおけるスキル入手は、歩行システムによる場所の探索と、
その場所で発生するイベント（シナリオ進行・サブイベント・レベルデザイン上の配置）を通じて行われる。

スキル入手の連鎖は以下の要素に依存する：

| 要素 | 内容 |
|------|------|
| **歩行システム** | 訪れる場所を体系的に決定する |
| **シナリオ状態** | 物語の進行に応じて発生するイベントが変わる |
| **所持パッシブ・状態** | プレイヤーの現在の性質が入手条件に影響する |
| **レベルデザイン** | 場所ごとに明示的に設定されたイベント配置 |

→ スキル入手は「縁」の連鎖であり、ポイントを貯めて交換する経済システムではない。

### 推理ゲームとの結合

pigRPGは推理ゲームでもあり、スキルの有無がシナリオ分岐に直接影響する：

- あるスキルを持っていれば証拠として使える、議論のアプローチが変わる
- 持っていなければその推理ルートの難易度が上がる、論破の方法が変わる

スキル入手がイベント駆動であることで、「どのスキルを持っているか」がプレイヤーごとの探索経路の結果として自然に分岐する。
スキルポイント制ではこの分岐が「最適解を買うだけ」に退化し、ステージ単位のレベルデザインで入手を制御することとも相性が悪い。

### スキルの成長手段

スキルの入手後の成長は以下で行われる：

| 要素 | 内容 |
|------|------|
| **使用回数** | スキルレベルの算出基盤（→ `スキルレベル仕様書.md`） |
| **戦闘勝利** | 敵スキル解放進行（WinGrowthStrategy） |
| **再エンカウント** | 敵スキル解放（ReEncountGrowthStrategy） |
