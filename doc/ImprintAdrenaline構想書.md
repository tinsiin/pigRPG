# Imprint Adrenaline 構想書

> **ステータス**: 内部実装済み（UI・エフェクト未実装）
> **由来**: 混沌ルーシーの属性コンボシステムをpigRPG向けに再設計

---

## §1 コンセプト

同じスキル印象（SkillImpression）のスキルを連続使用することでコンボが蓄積し、
汎用クリティカル率が段階的に上昇するシステム。
カウンタが上限に達するとバースト効果が発動し、カウンタが0にリセットされる。

### §1.1 混沌ルーシーとの差異

| 項目 | 混沌ルーシー | pigRPG（本構想） |
|------|-------------|-----------------|
| スキル選択 | ランダム（デッキ） | 固定（才能） |
| コンボ難易度 | ランダムゆえに高い | 意図的に繋げられるため低い |
| 解禁条件 | 汎用システム | **専用パッシブ所持者のみ** |
| 位置付け | コアメカニクス | 多様な強化要素のひとつ |

スキル固定のため、汎用システムにすると戦略性が単調になる。
パッシブ（または時限パッシブを付与するスキル）による解禁制にすることで、
**個性のひとつ**として成立させる。

---

## §2 基本メカニクス

### §2.1 コンボカウンタ

- 同じスキル印象のスキルを連続使用するとカウンタが+1
- **異なるスキル印象を使用した時点で0にリセット**
- カウンタ上限: **5**（5到達の次の使用でバースト発動 → 0リセット）
- 分類軸: スキル印象（SkillImpression enum）。今後印象の種類は増やす想定

### §2.2 コンボ段階と汎用クリティカル率

刃物クリティカル（BladeCriticalCalculation）とは**別系統**の汎用クリティカル判定。
現時点でこのパッシブ専用の枠とする（将来的に他パッシブが参入する余地はあるが、当面はここだけ）。

| コンボ数 | クリティカル率 |
|---------|--------------|
| 1（初回） | 0% |
| 2 | 3% |
| 3 | 6% |
| 4 | 8% |
| 5 | 12% |
| バースト（6回目） | 14% + バースト効果 |

---

## §3 バースト効果

カウンタ5到達後の次の使用（6回目）で発動。**2つの効果が同時に発生する。**

### §3.1 瞬間効果: 確定威力1.4倍

バースト発動時の攻撃に確定で1.4倍の威力補正。
ルーシーのバースト爆発感をエフェクト・SEとともにここで回収する。

連続行動条件（§4）によりバースト到達自体が不確実なため、
スキル固定ゲームでも過剰にはならない。

### §3.2 持続報酬: BM内恒常クリティカル率UP

バースト到達後、**その戦闘中ずっと**汎用クリティカル率が **+2%** 加算される。
重ねがけ可能（2回バーストで+4%、3回で+6%…）。

- 1回あたり+2%は控えめなので暴走しない
- 戦闘が長引くほど重ねた意味が出る「到達した者へのご褒美」
- コンボ段階のクリティカル率（§2.2）はリセットごとに消えるが、こちらは戦闘中永続

## §4 連続行動条件

ルーシーではスキルがランダムに出るためコンボ自体に不確実性があった。
pigRPGではスキル固定のため、**連続行動条件**で不確実性を補う。

### §4.1 連続行動の定義

コンボカウントが進むのは、同じスキル印象のスキルを使い、かつ
**自分のターンが連続で回ってきた場合のみ**。

判定基準: **グローバルTurnCountの差分が1であること**。

```
前回行動時のTurnCount == 現在のTurnCount - 1 → 連続行動 ✅
それ以外（間に他キャラが挟まった）             → コンボ途切れ ✗
```

### §4.2 連続行動に含まれるケースと含まれないケース

| ケース | TurnCount | コンボ加算 | 理由 |
|--------|-----------|-----------|------|
| 通常のランダム選出で連続で選ばれた | 進む（+1） | ✅ | ランダムに選ばれた＝不確実性あり |
| 凍結持ち越し（FreezeConsecutive） | 進む（+1） | ✅ | `NextTurn(true)`で新TurnCount。ランダム選出の余地がある |
| 先約リストからの行動 | 進む（+1） | ✅ | 割り込みカウンター・反撃等。正当な連続行動 |
| 同一ターン内マルチヒット（SameTurnConsecutive） | **進まない** | ❌ | `NextTurn(false)`。1スキルの複数ヒットはコンボではない |

→ 既存のTurnCount進行ルール（`NextTurn(true/false)`）で**自然に区別できる**。

### §4.3 実装上の必要事項

既存の `_tmp_EncountTurn`（`BaseStates.ACTUtility.cs`）はリカバリー判定のたびに
更新されるため、「前回**行動した**TurnCount」としては使えない。

**追加が必要なもの: 1フィールドのみ**

```csharp
// BaseStates に追加
private int _lastActedTurnCount;  // 前回行動完了時のグローバルTurnCount
```

行動開始時（`RecovelyWaitStart()` 等）に現在の `TurnCount` を記録。
次の行動時に `TurnCount - _lastActedTurnCount == 1` で連続行動を判定。

### §4.4 設計意図

ランダムターン制では「次に誰が選ばれるか」は不確実。
連続行動条件を加えることで、コンボの成立に不確実性が生まれる:

- **前のめり（Vanguard）**: リカバリー2倍速で選ばれやすい → コンボが繋がりやすい（リスクとリターン）
- **通常状態**: リカバリーが普通 → 間に他キャラが挟まりやすい → コンボが途切れやすい
- **リカバリーターンが長いキャラ**: コンボがほぼ繋がらない → このパッシブとの相性が悪い

→ 「狙って繋げられるが、確実ではない」状況が生まれる。
→ 威力1.4倍バースト（§3.1）が許容される根拠でもある。

### §4.5 前のめり（Vanguard）との相乗効果

前のめり状態ではリカバリーターンが2倍速で経過するため、
連続ターンを獲得しやすい → コンボが繋がりやすい。
ただし回避率が半減するため被弾リスクが上がる。

**「コンボを繋げたければ前に出ろ」** — アドレナリンの名に相応しいリスク・リターン構造が、
既存の前のめりシステムとの組み合わせだけで自然に生まれる。

| 状態 | コンボ繋がりやすさ | リスク |
|------|------------------|--------|
| 前のめり | 高（リカバリー2倍速） | 回避半減、テラーズヒット対象 |
| 通常 | 低（間に他キャラが挟まりやすい） | 安全 |

## §5 提供形態

両形態が共存する。実装上はパッシブ本体は共通で、付与方法と持続時間が異なるだけ。

### §5.1 プレイヤー側: スキル → 時限パッシブ付与

スキルを1ターン使って「Imprint Adrenalineモード」に入る。
パッシブには持続ターンがあり、期限内にバーストを目指す。

**プレイヤーの判断:**
1. **いつ起動するか** — 起動に1ターン消費するコスト。序盤か終盤か
2. **前のめりするか** — コンボを繋げやすくするが被弾リスク増
3. **制限時間内にバーストまで届くか** — ランダムターン制ゆえの不確実性

「いつ入るか」のタイミング判断が、スキル固定ゲームにおけるランダム性の代替になる。

### §5.2 敵側: 永続パッシブ

ボスが常時所持。プレイヤーから見ると：
「あいつ同じ系統の攻撃を重ねてきてる…バーストが来る…！」
という**脅威の可視化**。コンボ蓄積が見えること自体がプレッシャーになる。

## §6 既存システムとの関係

### §6.1 補正チェーン上の位置

ダメージ計算上に「クリティカル」と名の付く仕組みが既に2つあるが、いずれも本システムとは別物：

| 既存のクリティカル | 判定タイミング | 効果 | 発生源 |
|------------------|-------------|------|--------|
| 命中クリティカル（HitResult.Critical） | 命中回避計算時 | ダメージ1.5倍 | EYE/AGI/ケレン行動 |
| 刃物即死クリティカル（BladeCriticalCalculation） | ダメージ計算後半 | HP→0（即死） | 十日能力「刃物」 |

**汎用クリティカル（1.9倍）** はこれらとは独立した新枠として新設する。
現時点ではImprint Adrenalineのコンボ専用だが、将来的に他パッシブが参入する余地は残す。

### §6.2 汎用クリティカルのスキルタイプ別適用

汎用クリティカルは命中クリティカルと異なり、**スキルタイプに関係なく発動する**設計思想。
コンボさえ繋がっていれば、攻撃でも回復でもパッシブ付与でも恩恵がある。
これにより攻撃タイプのスキルでは命中クリティカル（1.5倍）と重なり、攻撃の価値がより高まる。

基本方針: **「アドレナリンの非制御感」— 何も考えず適用。** 数値があれば1.9倍、なければ対象外。

#### 数値系（1.9倍を直接適用）— 決定済み

| スキルタイプ | 効果 | 備考 |
|------------|------|------|
| **Attack（攻撃）** | ダメージ1.9倍 | 命中クリティカルと乗算で最大 1.5×1.9=2.85倍 |
| **善意攻撃** | ダメージ1.9倍 | 非制御。味方を助けたいならコンボを切る判断が生まれる |
| **Heal（回復）** | 回復量1.9倍 | |
| **MentalHeal（精神回復）** | 回復量1.9倍 | |
| **DeathHealOnBattle（復活）** | 復活時HP1.9倍 | |
| **AddVitalLayer（追加HP付与）** | VitalLayerの耐久値1.9倍 | バリアが分厚くなる |

#### 持続系（DurationTurn × 1.9倍）— 決定済み

| スキルタイプ | 効果 | 備考 |
|------------|------|------|
| **AddPassive（良いパッシブ付与）** | DurationTurn × 1.9倍（端数切り捨てor四捨五入は要決定） | 付与者視点で「長く続くと嬉しい」 |
| **悪いパッシブ付与（敵対）** | DurationTurn × 1.9倍 | 敵にデバフが長く残る。攻撃側の恩恵として自然 |

> **検討済み**: スレーム（ID12/17）のようなカスケード型パッシブ（OnDamageで連鎖的に他パッシブを付与する複雑なもの）を検証した結果、
> 汎用クリティカルの1.9倍はスキルの `addPassive` で**直接付与されるパッシブのDurationTurnのみ**に適用され、
> パッシブのコールバック（OnDamage等）から連鎖付与されるパッシブには波及しないため、問題なしと判断。
> 今後問題が見つかった場合はパッシブ側に「クリティカル延長免除」フラグを設けて個別対応する。

#### 適用が困難（対象外）— 決定済み

| スキルタイプ | 理由 |
|------------|------|
| **RemovePassive（パッシブ除去）** | 「1.9倍」の概念がない。除去は除去 |
| **RemoveVitalLayer（追加HP除去）** | 同上 |

#### スキルパッシブ付与（addSkillPassive）— 決定済み

スキルパッシブ（BaseSkillPassive）はキャラクターではなく**スキル自体に付くパッシブ**。
主なプロパティは `SkillPowerRate`（威力倍率）、`IsLock`（スキル封印）、`DurationTurn`（持続ターン）。

| プロパティ | 汎用クリティカル時の扱い | 理由 |
|-----------|----------------------|------|
| SkillPowerRate | **適用しない** | スキル威力への影響は攻撃時の1.9倍で既にカバー。二重乗算は不要 |
| DurationTurn | **× 1.9倍** | 持続系パッシブと同じルール |
| IsLock | **適用しない** | 二値（ON/OFF）なので1.9倍の概念がない |

### §6.3 慣れ補正（AdaptToSkill）との関係

慣れ補正は「**防御側が**同じ印象の攻撃を受け続けると慣れる」システム。
Imprint Adrenalineは「**攻撃側が**同じ印象を使い続けると冴える」システム。
方向が逆であり、棲み分けの問題はない。むしろ攻防両面からスキル印象に意味を持たせる形になる。

### §6.4 スキル印象（SkillImpression）

現在4種（TLOA_PHANTOM, HalfBreak_TLOA, Assault_Machine, SubAssult_Machine）。
今後増やす想定であり、種類が増えるほどコンボの選択肢と駆け引きが広がる。
