# 割り込みカウンター仕様書

## 1. 概要

敵の**連続攻撃中**に割り込み、その攻撃を無効化し、次のターンで反撃する仕組み。パッシブID=1。

割り込みカウンターは3段階の効果を持ち、キャラクターの種別と精神属性によって到達できる段階が異なる。

---

## 2. 発生条件

**呼び出し元**: `BaseStates.ReactionSkill.cs` L882 — 攻撃を受けた側のリアクションとして判定

**前提条件**:
1. 攻撃者が**連続攻撃の2回目以降**（`NowConsecutiveATKFromTheSecondTimeOnward()`）
2. 連続攻撃が `FreezeConsecutive` でない（ターンをまたぐ連続攻撃は割り込めない）
3. `IsInterruptCounterActive == true`（UIラジオボタンでON/OFF可能）
4. カウンター側のパワーが `PowerLevel.Medium` 以上

**確率判定**（2段階）:

### 第1段階: 防御無視率ベース

```
成功確率 = skill.DEFATK / 3 + plusAtkChance * 0.01
  ※ plusAtkChance = max(0, 自分のVond - 攻撃者のVond)
```

攻撃者のスキルの防御無視率が高いほど成功しやすい。ヴォンドの差で微加算。

**スキル専属十日能力補正（IsSkillEffect）**: 割り込みカウンターの確率判定では、**攻撃者側（カウンターされる側）のみ**にスキル専属補正がかかる。カウンター側は素の十日能力値（Base）を使用する。

### 第2段階: 能力値ベース + 50%乱数

```
counterValue = (myVond + myPersonaDivergence / (myTentVoid - ExplosionVoidValue)) * 0.9
attackerValue = max(0, eneSort - eneRain/3) + eneCold

判定: random(counterValue + attackerValue) < counterValue AND random(1) < 0.5
```

両方の乱数を通過して初めて割り込みカウンター成立。

---

## 3. 効果の3段階

割り込みカウンター成立時、以下の3段階が**上から順に**評価される。各段階は累積（上の段階が成立すれば下も成立する）。

### 第1段階: 攻撃無効化（全キャラ共通）

```
attacker.DeleteConsecutiveATK()       // 連続攻撃を打ち切り
attacker.IsActiveCancelInSkillACT = true  // 今回の攻撃を無効化
```

キャラクター種別・精神属性に関わらず、割り込みカウンターが成立すれば**全員が到達**する。

### 第2段階: 割り込みカウンターパッシブ付与（適合条件あり）

パッシブID=1（`InterruptCounterPassive`）を自身に付与。

- 適合条件は `CanApplyPassive()` で判定（パッシブの `OkType` / `OkImpression` による）
- 適合する精神属性: Pillar, Sacrif, Cquie, Psyco, God（メモより）
- 攻撃者が既にID=1パッシブを持っていた場合、`PassivePower + 1` で生成（カウンター合戦でレベルアップ）

**パッシブ効果**（`InterruptCounterPassive.cs`）:
- ATK乗算: `PassivePower × 0.26` 上乗せ（攻撃力強化）
- EYE固定値: `PassivePower × 12` 上乗せ（命中率強化）
- `OnAfterAttack` で効果半減（連続攻撃対応: 攻撃するたびに減衰）

### 第3段階: 先約リストへの反撃予約（Life種別のみ）

```csharp
if (HasCharacterType(CharacterType.Life))
{
    manager.Acts.Add(this, faction, "割り込みカウンター", null, isfreeze, attacker, CounterDEFATK);
}
```

- **`CharacterType.Life` のみ**が反撃ターンを獲得する
- Machine等は攻撃無効化とパッシブ付与まではできるが、反撃ターンは入らない

**反撃ターンの詳細**:
- `ExCounterDEFATK`: 攻撃者のスキルの防御無視率。反撃スキルの防御無視率より大きければこちらが使われる
- `Freeze`: カウンター側が中断不可能な連続攻撃中 or 発動カウント中なら `true` → 武器スキルで返す
- `SingleTarget`: `attacker`（カウンター対象）を指定。対象が死亡していればVoidTurnでターン消滅

## 4. 関連システム

### 4.1 食らわせパッシブ（ID=2）

割り込みカウンターパッシブ所持者が攻撃成功時、攻撃対象が「さっき自分を割り込んだ相手」なら、対象にID=2パッシブを付与。

**判定条件**（`BaseStates.Damage.cs` L664-679）:
- `RecentACTSkillData.IsDone == false`（まだ行動中の攻撃記録がある）
- `RecentACTSkillData.Target == 攻撃者`（割り込みの起点だった相手と一致）
- 持続ターン数 = そのスキルの `SKillDidWaitCount`（追加硬直値）

**効果**: 攻撃力と命中に低下（割り込まれたのに行動できてしまっている分のペナルティ）

### 4.2 割り込みカウンターのON/OFF

- `BaseStates.IsInterruptCounterActive`: デフォルト `true`
- `AllyClass`: UIラジオボタンで切り替え可能（CharaConfig画面）
- 敵AI: `NormalEnemy` でオーバーライド可能（現在は未実装）

### 4.3 パッシブ由来スキル発動率との関係

通常のパッシブ由来スキル発動率は `AttackChara`（攻撃キャラ単位）で判定されるが、割り込みカウンターの判定は `ReactionSkill`（対象者ごと）で行われる。つまり連続攻撃の各ヒット対象それぞれが独立して割り込みカウンターを判定する。

### 4.4 カウンター合戦

割り込みカウンターの連続攻撃を**さらに割り込みカウンター**した場合:
- 攻撃者の `PassivePower + 1` のパッシブが付与される
- CanOprate/SaneTurnの連続攻撃なら複数キャラに割り込みカウンターが飛び火する可能性がある

## 5. SingleTarget 接続（修正済み）

`Acts.Add` の `SingleTarget` 引数に `attacker`（カウンター対象）を渡すよう修正済み。

これにより以下が機能する:
1. **カウンター対象が死亡 → VoidTurn**: `TurnExecutor.CharacterAddFromListOrRandom` で検出、ターン消滅
2. **スキル範囲に関わらず対象を単体強制**: `SkillExecutor` の排他分岐で `DirectedWill.One` に上書き
3. **UI制限**: `BattleOrchestrator` で単体攻撃系スキルのみに制限 + ターゲット選択スキップ

### ターゲット強制の優先順位との関係

`ターゲット強制優先順位仕様書.md` に基づく優先順位:

```
吸引 > SingleTarget（カウンター対象固定） > イラつき > 通常
```

吸引中にカウンターが発生した場合、反撃先は吸引元に向く（吸引の「機械的に逆らえない」性質）。
