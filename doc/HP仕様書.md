# HP仕様書

## 1. 設計原則：HPは成長しない

**MaxHPは固定値であり、成長しない。これは意図的な設計判断である。**

pigRPGにおいて、四大ステ（ATK/DEF/AGI/EYE）は十日能力を通じて成長するが、
HPはこの成長体系に含まれない。
MaxHPはキャラクターのテンプレート定義値がそのままランタイムに使われ、
戦闘経験やスキル使用によって増加する仕組みは存在しない。

これは「HPが成長するタイプのRPG」ではないというゲーム性の根幹に関わる設計方針であり、
HP経験値システム等を導入しない理由でもある。

### 耐久力の変化手段

HPそのものは成長しないが、耐久力に影響する要素は存在する：

| 要素 | 内容 |
|------|------|
| **DEF成長** | 十日能力経由でDEFが成長し、被ダメージが減る |
| **VitalLayer** | MaxHPとは独立した追加HP層。パッシブ等で付与可能 |
| **HP回復** | 戦闘勝利時（15%）、敵逃走時（30%）、再遭遇時の自然回復 |

→ 「HPの数値を上げる」のではなく、「被ダメを減らす・追加層で守る」方向で耐久力が変化する。

## 2. HP非成長型RPGとしての方向性

この設計方針が戦闘システム全体と整合するかを、ゲームバランス・戦闘メカニクス・プレイヤー体験の3視点から多角的に分析した結果、**HP固定はpigRPGに合っている**と結論づけた。

詳細は `doc/HP非成長型RPGとしてのpigRPG分析.md` を参照。
