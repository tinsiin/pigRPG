# 挑発耐性パッシブ構想書

元メモ: セッション内の設計議論より
ステータス: 実装完了

---

## 1. 概要

BasePassiveに挑発属性への耐性プロパティを3種追加する。パッシブを通じて「イラつきへの備え」を戦術的に組み込めるようにする。

---

## 2. 設計意図

### 2.1 なぜパッシブなのか — 「補完要素」としての位置づけ

挑発耐性は**キャラ自身の固有ステータスには作らない**。これは意図的な設計判断。

| 層 | 既存のもの | 性質 |
|----|-----------|------|
| 種別 | 機械免疫（`CharacterType.Machine`） | 完全免疫。有機キャラには存在しない |
| 性質 | 精神属性補正（付与量/減衰量） | キャラの内面に固定。操作不可 |
| ステータス | 十日能力の強さ補正 | 力の差に応じた自然な耐性。意図的な操作は困難 |
| **戦術** | **← ここが空白だった** | **プレイヤーが意図的に選べる対策手段** |

機械は完全免疫を持つ。有機キャラは**イラつきから完全に逃れることはできない**。これは挑発属性の設計原則。

キャラ自身にイラつき耐性ステータスを持たせてしまうと、この原則が曖昧になる — 「実質的に免疫に近いキャラ」が生まれ、機械免疫の独自性（「感情がないから」）を侵食する。

パッシブは**枠を消費するコスト**がある。挑発対策に枠を使えば、他のことに使えない。このトレードオフが「完全な無効化はありえないが、備えることはできる」を担保する。

### 2.2 現在のバランスの穴

挑発属性の既存レバーはすべて**受動的**で、プレイヤーが意図的にコントロールできるものがない。

- 同格の相手に対して、防ぐ手段がない
- 減衰を待つしかない（3ターンごとに1、精神属性によっては0）
- 暴走を受け入れるか、挑発者を倒すかの二択

パッシブ耐性は**「挑発が来ると分かっていれば備えられる」戦術レイヤー**を提供する。状態異常耐性アクセサリと同じ位置づけ。

### 2.3 双方向リスクへの影響

挑発属性の核心設計は「追い込むほど相手が強くなる」双方向リスク。パッシブ耐性がこれを壊さないか。

| プロパティ | 双方向リスクへの影響 |
|-----------|-------------------|
| 付与量軽減 | 暴走到達が遅れる → 挑発側の暴走リスクが減る → 双方向性がやや薄まる。ただしパッシブ枠コストで相殺 |
| 減衰量加算 | 付与量軽減と同様。持続付与（パッシブ経路）に特に有効 |
| 発動確率軽減 | **双方向性を最も壊さない**。イラつきカウントは通常通り溜まり、暴走もする。行動強制だけが減る。「頭では怒ってても行動は冷静」 |

→ 3種全部入れるが、**発動確率軽減が中核**であることを意識する。

## 3. プロパティ仕様

BasePassiveに以下3つの `[SerializeField]` フィールドを追加する。

### 3.1 付与量軽減率（`_irritationApplicationResistRate`）

| 項目 | 内容 |
|------|------|
| 型 | `float` |
| デフォルト | `1.0`（軽減なし） |
| 意味 | 最終付与量に乗算される係数。0.5で半減 |
| 適用箇所 | `IrritationService.Add()` 内、精神属性補正・強さ補正の後 |
| 集約方法 | 全パッシブの値を乗算（`Π(rate)`）。複数パッシブで重ねがけ可能 |
| 下限 | `Mathf.Max(1, ...)` が既にあるため、完全に0にはならない |
| フレーバー | 挑発を受け流す精神的余裕 |

```
最終付与量 = スキル付与量 × 精神属性補正 × 強さ補正 × Π(パッシブ付与量軽減率)
           → Mathf.Max(1, Mathf.RoundToInt(...))
```

### 3.2 減衰量加算（`_irritationDecayBonus`）

| 項目 | 内容 |
|------|------|
| 型 | `int` |
| デフォルト | `0`（加算なし） |
| 意味 | 3ターンごとの減衰量に加算される整数値 |
| 適用箇所 | `IrritationService.Decay()` 内、精神属性減衰量の取得後 |
| 集約方法 | 全パッシブの値を合算（`Σ(bonus)`） |
| 特性 | 持続付与（パッシブ経路のじわじわ型）に特に効果的 |
| **制約** | **精神属性の減衰量が0の場合（BaleDrival等）、パッシブ加算は適用されない**（下記参照） |
| フレーバー | 怒りの冷め方が早い |

```
baseDecay = 精神属性別減衰量
if baseDecay <= 0: return（パッシブでも覆せない、不動）
最終減衰量 = baseDecay + Σ(パッシブ減衰量加算)
```

#### 精神属性の減衰0は不動（設計判断）

BaleDrivalのように精神属性の減衰量が0のキャラは、パッシブで減衰ボーナスを積んでも減衰しない。これは「流れのままに行くタイプだから減衰しない」という**キャラの本質的性質**であり、外的なパッシブで覆すべきではないという設計判断。減衰0は機械免疫と同じく**固有の不動の性質**として扱う。

### 3.3 発動確率軽減率（`_irritationTriggerResistRate`）

| 項目 | 内容 |
|------|------|
| 型 | `float` |
| デフォルト | `1.0`（軽減なし） |
| 意味 | Demeanor別発動確率係数に乗算される係数。0.5で発動確率半減 |
| 適用箇所 | `IrritationService.JudgeTrigger()` 内、probability計算後 |
| 集約方法 | 全パッシブの値を乗算（`Π(rate)`） |
| 特性 | イラつきカウント自体は溜まる。暴走もする。行動強制のみ軽減 |
| フレーバー | イラついていても行動に出さない自制心 |

```
発動確率 = 発動値 × Demeanor別係数% × Π(パッシブ発動確率軽減率)
```

## 4. 実装方針

### 4.1 BasePassive への追加

```csharp
// --- 挑発耐性プロパティ ---
[SerializeField] float _irritationApplicationResistRate = 1f;
[SerializeField] int   _irritationDecayBonus = 0;
[SerializeField] float _irritationTriggerResistRate = 1f;
```

getter / 集約メソッドも追加:

```csharp
public float IrritationApplicationResistRate => _irritationApplicationResistRate;
public int   IrritationDecayBonus => _irritationDecayBonus;
public float IrritationTriggerResistRate => _irritationTriggerResistRate;
```

### 4.2 BaseStates.HasPassives への集約メソッド追加

既存パターン（`PassivesHealEffectRate` 等）を踏襲:

```csharp
/// <summary>
/// 全パッシブの付与量軽減率を乗算集約。
/// </summary>
public float PassivesIrritationApplicationResistRate()
{
    float result = 1f;
    foreach (var pas in _passiveList)
        result *= pas.IrritationApplicationResistRate;
    return result;
}

/// <summary>
/// 全パッシブの減衰量加算を合算集約。
/// </summary>
public int PassivesIrritationDecayBonus()
{
    int total = 0;
    foreach (var pas in _passiveList)
        total += pas.IrritationDecayBonus;
    return total;
}

/// <summary>
/// 全パッシブの発動確率軽減率を乗算集約。
/// </summary>
public float PassivesIrritationTriggerResistRate()
{
    float result = 1f;
    foreach (var pas in _passiveList)
        result *= pas.IrritationTriggerResistRate;
    return result;
}
```

### 4.3 IrritationService への適用

**Add()** — 付与量軽減:

```csharp
float modifier = GetApplicationModifier(target.MyImpression);
float strengthMod = GetStrengthModifier(source, target);
float passiveResist = target.PassivesIrritationApplicationResistRate();
int finalAmount = Mathf.Max(1, Mathf.RoundToInt(rawAmount * modifier * strengthMod * passiveResist));
```

**Decay()** — 減衰量加算:

```csharp
int decayAmount = GetDecayAmount(chara.MyImpression) + chara.PassivesIrritationDecayBonus();
```

**JudgeTrigger()** — 発動確率軽減:

```csharp
float probability = triggerValue * coefficient * chara.PassivesIrritationTriggerResistRate();
```

```

  rawAmount（スキル設定値）
    × GetApplicationModifier（精神属性補正 5.2）

発動確率:
  triggerValue × GetTriggerCoefficient（Demeanor別係数 5.3）

```
