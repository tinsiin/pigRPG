## 1. フキダシスタイル（立ち絵モード専用）

### 概要

立ち絵モードのテキストボックスをフキダシとして扱い、**しっぽ（トゲ）の向き**で話者の左右を表現する。
テキストボックスの位置自体は固定。しっぽの向きだけが切り替わる。

ディノイドモードはアイコン内包型のため、フキダシの概念は適用しない。

### 話者判定と左右選択

話者の位置に応じて、左用/右用のフキダシデータセットから自動選択される。

| 話者の位置 | 使われるデータセット |
|-----------|-------------------|
| 左立ち絵が話者 | 左用データセット |
| 右立ち絵が話者 | 右用データセット |

**判定方法**: `DialogueStep.speaker` と左右の `PortraitState.characterId` を照合。
一致した側のデータセットからランダムに1枚選ばれる。

### フキダシ画像の性質

フキダシは**枠・しっぽ込みの1枚画像**。枠としっぽを分離しない。
線を引くたびにディテールが変わるため、左用・右用それぞれに複数枚のバリエーションを登録し、
セリフごとにランダムで切り替わることで「毎回微妙に形が違う」感を出す。

### データ構造

```
BubbleStyleSet（ScriptableObject）
├── leftVariants[]: Sprite[]       <- 左用フキダシ画像群（話者が左の時に使う）
├── rightVariants[]: Sprite[]      <- 右用フキダシ画像群（話者が右の時に使う）
└── name: string                   <- セット名（デバッグ/指定用）

TextBoxPresenter（または上位）
└── defaultBubbleStyleSet: BubbleStyleSet   <- デフォルトで参照されるセット

DialogueStep
└── bubbleStyleSetOverride: BubbleStyleSet  <- null=デフォルトを使用、指定時=このセットを使用
```

**選択フロー:**
1. `DialogueStep.bubbleStyleSetOverride` があればそれを使う、なければデフォルト
2. 話者の位置を判定 → 左用 or 右用の配列を選ぶ
3. 配列からランダムに1枚選んで表示

## 2. テキストボックスの登場演出

### 現状（実装済み）

両モード共通: ダンロン風（右下+50px/-50px、5°回転、OutBack、0.3秒）

### 改修内容

ディノイドモードと立ち絵モードで**別々の登場演出**にする。

### ディノイドモード: 上下に開く

```
Phase 1: 水平線が定位置に瞬時出現（alpha即座に1、scaleY ≈ 0.02）
         ┌━━━━━━━━━━━━━━━━━━━━━━━━┐  <- 細い水平線がパッと出る
         └━━━━━━━━━━━━━━━━━━━━━━━━┘    alphaアニメなし（即時表示）

Phase 2: 中央pivotから上下対称に広がる（scaleY: 0.02→1）
         ┌──────────────────────────┐
         │                          │  <- テキストボックス完成
         │                          │
         └──────────────────────────┘
         pivot = center（上下対称に展開）
         イージング・時間: 裁量（目安 0.25秒程度、OutQuad）

Phase 3: テキスト表示開始
```

退場（後述の「ぎゅっと閉じる」）との対称性を持たせる。

### 立ち絵モード: 右下から回転（現状維持）

現状のダンロン風をそのまま使用。
- 開始位置: 定位置から右+50px、下-50px
- 開始回転: 5°
- フェードイン + 位置補間 + 回転補間（0.3秒、OutBack）

## 3. テキストボックスの退場演出

### 現状（実装済み）

両モード共通: フェードアウトのみ（alpha: 1→0、0.15秒、InQuad）

### 改修内容

ディノイドモードと立ち絵モードで**別々の退場演出**にする。

### ディノイドモード: テキスト消去 → 枠線出現・点滅 → ぎゅっと閉じる

```
Phase 1: テキスト・画像が一瞬で消える（alphaアニメなし、即時非表示）
         ┌──────────────────────────┐
         │                          │  <- 中身が空になる
         └──────────────────────────┘

Phase 2: 退場用の枠線が出現し、点滅する
         ┏━━━━━━━━━━━━━━━━━━━━━━━━┓
         ┃                          ┃  <- 普段は非表示の枠線が登場
         ┗━━━━━━━━━━━━━━━━━━━━━━━━┛    alpha振動で点滅（3-4回、約0.2-0.3秒）
         ※枠線は退場演出専用のImage。通常のテキストボックスとは別オブジェクト

Phase 3: 枠線が中央pivotに向かって縮小（scaleY: 1→0）
         ┌━━━━━━━━━━━━━━━━━━━━━━━━┐  <- 水平線に潰れて消える
         └━━━━━━━━━━━━━━━━━━━━━━━━┘
         点滅回数・縮小時間: 裁量（目安: 点滅0.25秒 + 縮小0.2秒）

Phase 4: 完全に消える（枠線を非表示に戻す）
```

登場の「上下に開く」と対称。テレビのオン/オフの対比。
**退場用枠線はコード描画（画像納品不要）**:
退場用枠線は画像素材ではなく、4辺を個別のImageで構成してコード側で生成する。
これにより線の太さ・色・点滅振幅をコードで自由に制御できる。

```
DinoidExitFrame (RectTransform: テキストボックスと同サイズ、通常時非表示)
├── TopLine    (Image: 白矩形、height=線の太さ、stretch横)
├── BottomLine (Image: 同上)
├── LeftLine   (Image: 同上、width=線の太さ、stretch縦)
└── RightLine  (Image: 同上)
```

- 太さ: 各LineのsizeDeltaで制御
- 色: Imageのcolorで制御（tint可）
- 点滅: 4辺まとめて親のCanvasGroupでalpha振動
- 閉じる: 親のscaleYを縮小すれば4辺ごと潰れる

### 退場演出の基本形とモード切替用アレンジ

退場演出には**基本形（会話終了時）**と、それを簡略化した**切替用アレンジ（モード切替時）**がある。

**基本形 = 会話終了時のフル演出。** これが退場演出の本体。
**切替用アレンジ = 基本形から装飾を削ぎ落としたテンポ重視版。**

#### ディノイドモードの場合

| | 基本形（会話終了時） | 切替用アレンジ（Dinoid→Portrait切替時） |
|---|---|---|
| テキスト消去 | 即消去 → 退場用枠線に差替え | なし（見た目そのまま） |
| 点滅 | あり（枠線が点滅） | **なし** |
| 閉じる | 枠線がscaleY縮小 | テキストボックスごとscaleY縮小 |
| 目的 | 会話終了の区切りを演出する | すぐに立ち絵登場 → Portrait登場へ繋げる |

#### 立ち絵モードの場合

| | 基本形（会話終了時） | 切替用アレンジ（Portrait→Dinoid切替時） |
|---|---|---|
| 内容 | 点滅しながら下スライド | **将来実装予定**（現状はフェードアウト） |

### 立ち絵モード: 点滅しながら下にスライドダウン

```
Phase 1: テキストボックス全体が点滅しながら下方向へスライド開始（同時進行）
         ┌──────────────────────────┐
         │  テキスト                 │  ↓ 下方向へ移動
         └──────────────────────────┘    alpha振動で点滅（一定周期）

         ※立ち絵のExit()（左右スライドアウト）と同時開始

Phase 2: EyeArea下端より下まで移動して見えなくなる
         ────────────────────────────── <- EyeArea下端
         ┌──────────────────────────┐
         │  （画面外で見えない）       │
         └──────────────────────────┘
```

**スライド距離**: EyeAreaのRectTransformの下端を基準に算出（コード側でEyeAreaの位置から計算）。
テキストボックスがEyeArea外に完全に出る距離まで移動する。

**フェードアウトしない**: alphaを0に向かって下げる処理はしない。
点滅（alpha振動）しながら画面外に出ることで視覚的に消える。

**点滅パターン**: 一定周期のalpha振動。下降の時間経過に合わせた工夫は不要。
点滅速度・振幅: 裁量（目安: 周期0.08秒程度、alpha 0.3〜1.0の振動）

立ち絵の退場（左右スライドアウト）と同時進行。
テキストボックスは下へ、立ち絵は左右へ、それぞれ捌ける。

## 4. 立ち絵の背面装飾レイヤー

### 概要

立ち絵の周囲に装飾画像を追加する。**2つの別画像**で構成される。

### 2つのパーツ

| パーツ | 位置 | 内容 | 話者状態で変化するか |
|---|---|---|---|
| **前方の線** | キャラの前方に飛び出す | 装飾線 | **しない**（固定） |
| **後方の矢印** | キャラの背中側 | 矢印形状 | **する**（Talk / Hold on） |

### 後方矢印の話者状態

後方の矢印は、そのターンで話しているかどうかで見た目が切り替わる。

| 状態 | 見た目 | 条件 |
|---|---|---|
| **Talk** | 優勢な矢印（相手の方に向かって伸びる） | そのターンで話してる側 |
| **Hold on** | 縮こまった矢印（相手の方を向かない） | 話してない側 |

**判定方法**: フキダシの話者判定と同じ。`DialogueStep.speaker` と `PortraitState.characterId` の照合。

**やめたこと**: 話してない側の立ち絵全体をグレーアウト+サイズ縮小する案。
高速タップ時に左右が一々縮んだり大きくなったりしてうるさいため、矢印の切替だけで表現する。

### レイヤー構造

```
LeftPortrait
├── LeftBackArrow (Image)        <- 新規: 後方矢印（Talk/Hold on切替）
├── LeftCharacterImage (Image)   <- 既存: 立ち絵本体
└── LeftFrontLine (Image)        <- 新規: 前方の線

RightPortrait
├── RightBackArrow (Image)       <- 新規: 後方矢印（Talk/Hold on切替）
├── RightCharacterImage (Image)  <- 既存: 立ち絵本体
└── RightFrontLine (Image)       <- 新規: 前方の線
```

- 後方矢印は立ち絵の**後ろ**（描画順で後ろ）
- 前方の線は立ち絵の**前**（描画順で前）

### データの持ち方

```
PortraitDecorationSet（装飾データ）
├── leftFrontLine: Sprite          <- 前方の線（左用、固定）
├── rightFrontLine: Sprite         <- 前方の線（右用、固定）
├── leftBackArrowTalk: Sprite      <- 後方矢印+パターン: 左用Talk
├── leftBackArrowHoldOn: Sprite    <- 後方矢印+パターン: 左用Hold on
├── rightBackArrowTalk: Sprite     <- 後方矢印+パターン: 右用Talk
└── rightBackArrowHoldOn: Sprite   <- 後方矢印+パターン: 右用Hold on

PortraitPresenter
└── decorationSet: PortraitDecorationSet  <- 装飾データ
```

### UI特性

**この装飾は完全に不変のUI。** フキダシのようなセリフごとのランダム選択やオーバーライドはない。
変化するのは後方矢印のTalk/Hold on切替のみで、これは話者判定による自動切替。

### 後方矢印の画像構成

後方矢印の画像には、矢印本体だけでなく**矢印から前方の線までの間のパターン描画**も含める。
これにより矢印切替時にパターン部分も一緒に差し替わる。
前方の線は別レイヤーで固定なので、どの矢印パターンでもキャラの裏に背景が透けて見える。

```
描画順（奥→手前）:
  後方矢印+パターン（Talk or Hold on） ← 矢印切替で画像ごと差し替え
  └── キャラ立ち絵本体
      └── 前方の線（固定）
```

### 装飾のアニメーション

立ち絵本体のトランジション（Rockman/Slide等）に追従。
立ち絵と同じタイミングで登場/退場する（別個のアニメーションは不要）。
後方矢印のTalk/Hold on切替はスプライト差し替えのみ（アニメーションなし）。

## 実装順序（素材納品後）

1. **立ち絵背面装飾レイヤー追加** — PortraitPresenterにImage追加、PortraitDecorationSetデータ構造作成
2. **フキダシスタイル** — BubbleStyleSetデータ構造、話者判定による左右選択、ランダム選択、DialogueStepへのoverride追加
3. **テキストボックス退場演出** — ディノイド: 点滅+縮小 / 立ち絵: 点滅+下スライド
4. **テキストボックス登場演出** — ディノイド: 上下開き（退場と対称）
