# 戦闘規格・狙い流れ仕様書

## この仕様書の主題

このゲームの戦闘ダメージ計算には「排他ステ」という仕組みがある。
**攻撃力と防御力の一部が、戦闘スタイルに応じて異なる十日能力から算出される。**

攻撃側と防御側の排他ステは**別の軸**で分岐する。この非対称性が戦闘の駆け引きの根幹。

| ステータス | 分類軸 | 概要 |
|-----------|--------|------|
| **排他ATK** | **戦闘規格 (BattleProtocol)** | 自分の武器の規格で決まる。自分自身で選ぶ |
| **排他DEF** | **狙い流れ (AimStyle)** | 相手の攻撃パターンに対してどう防ぐかで決まる。攻撃者の行動に応じて受動的に切り替わる |

→ 攻撃力は「自分が何者か」で決まり、防御力は「相手が何をしてくるか」で決まる。

設計メモソース:
- `十日印象能力.md` — 排他ステ係数テーブル、複数規格武器、適応ラグ
- `狙い流れ.md` — AimStyleバリエーション、物理属性適性、引き締め値との連携
- `none防ぎ方の考察について.md` — none AimStyleの特殊挙動
- `連続攻撃.md` — 連続攻撃とAimStyleの関係
- `引き締め値.md` — 防御切替速度の補正
- `割り込みカウンター.md` — 防御無視率とカウンターの関係
- `戦闘規格のラグ.md` — 規格切替時の適応率ペナルティ

## 1. 戦闘規格 (BattleProtocol)

武器によって決定される「攻撃の流儀」。排他ATKの分類軸。

`BaseStates.WeaponManager.cs:79-91`

| 値 | 日本語名 | 概要 | デフォルト狙い流れ |
|----|---------|------|------------------|
| `LowKey` | 地味 | 堅実型 | アクロバマイナ(6割) / ダブレット |
| `Tricky` | トライキー | 変則型 | ダスター(9割) / 四弾差し込み |
| `Showey` | 派手 | 攻撃型 | 中天一弾(8割) / ダブレット |
| `none` | なし | フリーハンド | 狙い流れなし |

- 武器の `protocols` フィールドで設定。規格は武器は複数持てるが、基本は一つ
- フリーハンドは全規格（`[LowKey, Tricky, Showey, none]`）を持ち切替可能
- 規格の切替には適応ラグが発生する（詳細は `doc/武器システム仕様書.md` 参照）

## 3. 排他ATK（攻撃力の規格依存部分）

### ATK計算の全体構造

`BaseStates.StatesNumber.cs` の `b_ATK()` メソッド

```
ATK = b_b_atk（基礎値）
    + Σ( 共通ATK係数[能力] × 十日能力値[能力] )    ← 全規格共通（22種）
    + Σ( 排他ATK係数[能力] × 十日能力値[能力] × 適応率 )    ← 規格ごとに異なる ★
    + 武器ボーナス（スキル特判別に応じたTenDayBonusData）
    + 特別補正（SetSpecialModifier経由）
```

排他ATK係数には**適応率 (`_adaptationRate`)** が乗算される。規格切替直後は77〜100%で、6戦かけて100%に回復。

## 4. 排他DEF（防御力の狙い流れ依存部分）

### DEF計算の全体構造

```
DEF = b_b_def（基礎値）
    + Σ( 共通DEF係数[能力] × 十日能力値[能力] )    ← 全AimStyle共通（18種）
    + Σ( 排他DEF係数[能力] × 十日能力値[能力] )    ← AimStyleごとに異なる ★
```

**重要**: DEF排他の分類軸は BattleProtocol ではなく **AimStyle**。
BattleProtocol はデフォルトの AimStyle を決めるだけで、実際の DEF は AimStyle 単位で計算される。

## 5. 戦闘規格 → 狙い流れ の対応（攻撃パターンの決定）

### デフォルト防御パターン

`BaseStates.DictionaryData.cs` の `DefaultDefensePatternPerProtocol` 辞書

各戦闘規格には a/b の2つの狙い流れパターンがある。攻撃時に確率で決まり、スキルのムーブセット（a=0, b=1）も連動する。

| 戦闘規格 | a パターン | a の確率 | b パターン | b の確率 |
|---------|-----------|---------|-----------|---------|
| LowKey（地味） | アクロバマイナ体術 | **60%** | ダブレット | 40% |
| Tricky（トライキー） | ダスター | **90%** | 四弾差し込み | 10% |
| Showey（派手） | 中天一弾 | **80%** | ダブレット | 20% |
| none | なし | — | なし | — |

**注目**: ダブレットは LowKey の b と Showey の b の両方に含まれる。ダブレットの排他DEFを高めれば、地味と派手の両方に対して有利。

### スキルのムーブセット

スキルの連続攻撃は a/b いずれかのムーブセットに沿って進む。
ムーブセットが a か b かによって、攻撃の AimStyle が変わる。
これが「戦闘規格により含まれる狙い流れのバリエーションは決まり、実際のセットアップの移り変わりはスキルで決まる」というメモの意味。

## 6. 防御切替メカニズム（SwitchDefenceStyle）

`BaseStates.Damage.cs:240-301`

攻撃を受けるたびに呼ばれ、防御側の `NowDeffenceStyle` を更新する。

### Case 1: 攻撃者が none 規格の場合

```
攻撃者.NowBattleProtocol == none → 防御側.NowDeffenceStyle = AimStyle.none
```

フリーハンドの「掴んで投げる」を食らうと、**強制的に AimStyle.none になる**。
排他DEFがゼロになり、次の攻撃に対して無防備な一瞬が生まれる。

### Case 2: 初回攻撃（単体・連続1回目）

1. 攻撃者の規格から `DefaultDefensePatternPerProtocol` でパターンを取得
2. 確率で a/b が決まり、攻撃の AimStyle が決定
3. **引き締め値の補正段階 ≥ 2** なら 75% の確率で正しい防ぎ方に切り替わる
4. 補正段階 < 2 なら初回での対応は**ほぼ確実に失敗**（ランダムな防ぎ方になる）

```
引き締め段階 ≥ 2 → 75%で正しく防ぐ / 25%でランダム
引き締め段階 < 2 → 0%で正しく防ぐ / 100%ランダム
```

### Case 3: 連続攻撃（2回目以降）

1. 攻撃の AimStyle と現在の防ぎ方が一致していれば何もしない
2. 一致していなければ、**AimStyleメモリ**を更新
3. メモリのカウントが閾値に達したら防ぎ方を切り替える
4. FreezeConsecutive（ターンまたぎ連続攻撃）の場合、切替成功時でも**30%の確率で切り替わらない**

→ 連続攻撃が多いほど、防御側は徐々に適応していく。だが引き締め値が低ければ対応が間に合わない。

## 7. 引き締め値（防御切替速度の補正）

`BaseStates.Damage.cs:344-432`

十日能力 `NightInkKnight` の値から算出される**防御の適応速度**。

### 補正段階の計算

```
補正段階 = floor(NightInkKnight / 10)
パワーが high の場合、50%の確率で +1
```

| NightInkKnight | 補正段階 | 効果 |
|----------------|---------|------|
| 0〜9 | 0 | 初回対応不可、連続攻撃での切替も遅い |
| 10〜19 | 1 | 初回対応不可、切替やや遅い |
| 20〜29 | 2 | **初回75%対応可能**、切替速度向上 |
| 30〜39 | 3 | 初回75%対応、切替さらに速い |
| 50+ | 5+ | 切替非常に速い、カウント最大削減確率80% |

### 連続攻撃の切替に必要なカウント

AimStyle の組み合わせごとに基本カウント（`DefenseTransformationThresholds`）が定義されている。
引き締め値が高いほど、この最大カウントが削減され、1回あたりの進捗も増える。

#### CalcTransformCountMax — 最大カウントの決定

```
基本カウント = DefenseTransformationThresholds[(攻撃AimStyle, 現防御AimStyle)]
段階 ≥ 2: ~31%確率 + NightInkKnight/100 の確率で -1
段階 ≥ 5: 80%確率で さらに-1
最小値: 2
```

#### CalcTransformCountIncrement — 1回あたりの進捗

```
段階 < 2: 常に 1
段階 ≥ 2: 1 + random(0 〜 段階)   ※パワーがmedium未満なら上限-1
段階 > 5: 1 + random(段階/6 〜 段階)
```

### AimStyle間の切替難易度テーブル（DefenseTransformationThresholds）

数字が大きいほど、その防ぎ方から目的の防ぎ方への切替に時間がかかる。

| → 攻撃AimStyle / ↓ 現在の防ぎ方 | アクロバ | ダブレット | 四弾差込 | ダスター | ポタヌ | 中天一弾 |
|-------------------------------|--------|----------|--------|--------|------|--------|
| **アクロバマイナ** | — | 2 | 6 | 4 | 7 | 4 |
| **ダブレット** | 3 | — | 6 | 8 | 4 | 7 |
| **四弾差し込み** | 4 | 2 | — | 5 | 6 | 4 |
| **ダスター** | 3 | 8 | 4 | — | 7 | 5 |
| **ポタヌヴォルフ** | 2 | 3 | 5 | 4 | — | 5 |
| **中天一弾** | 4 | 3 | 6 | 8 | 2 | — |

**読み方**: 行 = 攻撃されている AimStyle、列 = 今の防ぎ方。数値 = 切替に必要な基本カウント。
例: ダスターで攻撃されている時、ダブレットで防いでいると → **8カウント**必要（非常に遅い）。

## 8. 0.7クランプ（不適切な防ぎ方のペナルティ）

`BaseStates.Damage.cs:308-334` — `ClampDefenseByAimStyle()`

攻撃の AimStyle と防御の AimStyle が**一致しない場合**、防御力にペナルティが発生する。

### ルール

```
適切な防御力 = その AimStyle に正しく対応した場合の DEF
クランプ値 = 適切な防御力 × 0.7
```

| パワー | 挙動 |
|--------|------|
| **high超** | 現在のDEFが「適切な防御力」を**超えている場合のみ**クランプ。超すまではゴリ押し可能 |
| **medium以下** | **常にクランプ**。不適切な防ぎ方での防御力は、適切な防御力の0.7が上限 |

**設計意図**: パワーが高いキャラは「間違った防ぎ方でも力押しで耐える」余地がある。パワーが低いと、防ぎ方を間違えた時点で0.7上限が強制される。

### noneとクランプの特殊な関係

- `none` で攻撃された → 防御側は `AimStyle.none` が強制 → 排他DEFがなくなる
- 次のターンで `none` 以外の攻撃を受けた場合:
  - 切替に失敗すると `none` のまま → **0.7クランプが必ず作動**（noneと他のAimStyleは絶対に一致しない）
  - ただし `none` には排他DEFがないため、そもそも防御力が低い → クランプされても実質影響は限定的
- **RandomAimStyle にnoneは含まれない** → 切替失敗時のランダム防ぎ方にnoneは出ない

## 9. 物理属性適性ボーナス（連続攻撃3回目以降）

`BaseStates.Damage.cs:200-220` — `CheckPhysicsConsecutiveAimBoost()`

武器の物理属性と AimStyle の物理属性適性が一致し、かつ連続攻撃の**3回目以降**で **ATK 1.3倍** ブースト。

| AimStyle | 物理属性適性 | 一致する武器物理属性 |
|----------|------------|-------------------|
| ダブレット | ヴォル転 | `WeaponPhysicalProperty.volten` |
| ポタヌヴォルフ | ヴォル転 | `WeaponPhysicalProperty.volten` |
| ダスター | 床ずれ | `WeaponPhysicalProperty.dishSmack` |

- `ATKCountUP >= 2`（0-indexed、つまり3回目以降）で発動
- `SetSpecialModifier` 経由で ATK に 1.3 倍を乗算
- **連続攻撃限定**。単発スキルでは発動しない

## 10. 防御無視率と割り込みカウンター

### 防御無視率 (DEFATK)

スキルのムーブセットで設定され、a/b パターンごとに異なる値を持てる。
スキルの DEFATKプロパティで乗算される。

### 割り込みカウンター

連続攻撃中に防御側が割り込んで反撃するシステム。

- **防御無視率が高いほど、割り込みカウンターが発生しやすい**（攻撃が露骨→隙が大きい）
- 防御無視率 ÷ 3 を百分率判定の基準に使用
- ヴォンドとペルソナ乖離が攻守のカウンター判定に影響
- **パワーが普通以上の場合のみ発動**

割り込みカウンターが成功すると:
1. 敵の連続攻撃を中断
2. 次ターンに ATK 2倍 + 命中100% の確定反撃（連続するたびに補正半減）
3. 攻撃された相手に「食らわせ」パッシブ付与（ATK・命中低下）

詳細: `BaseStates.Damage.cs` のカウンター判定処理

## 11. none戦闘規格の特殊性

フリーハンドの「掴んで投げる」スキル限定の特殊な AimStyle。

### 攻撃側がnoneの場合

- 排他ATK係数がない → 攻撃力が低い
- 防御側は `SwitchDefenceStyle` で `AimStyle.none` を強制される → 排他DEFがなくなる
- **掴んで投げられた後の無防備状態**をロールプレイとして再現

### 防御側がnoneにされた後の展開

1. 次のターンで別のキャラに攻撃される
2. `SwitchDefenceStyle` で新しい AimStyle への切替を試みる
3. 引き締め値が低いと切替に失敗 → `none` のまま
4. `none` のまま → **0.7クランプが必ず発動**（none ≠ 他の全AimStyle）
5. ただし `RandomAimStyle` に `none` は含まれないため、切替失敗時は none 以外のランダムな防ぎ方になる

→ 掴んで投げるの**真の脅威は即座のダメージではなく、直後の無防備時間**。

## 12. 適応ラグ（概要）

戦闘規格を切り替えた際の排他ATK反映率ペナルティ。詳細は `doc/武器システム仕様書.md` 参照。

| 場面 | 適応率スタート | 回復 |
|------|--------------|------|
| 規格切替（戦場） | 77〜100% ランダム | 線形6戦 |
| フリーハンド→規格（戦場） | 90〜100% ランダム | 線形6戦 |
| 武器変更（戦場） | 99% 固定（名目上） | 1戦 |
| 休憩地点 | 100% | ペナルティなし |
| 武器主六面の魂 | 常に100% | 免除 |

適応率は排他ATK係数にのみ乗算される。共通ATK・DEF・AGI・EYE には影響しない。

## 13. ダメージ計算における処理順序

`BaseStates.Damage.cs` の `DamageOnBattle()` 内での排他ステ関連の処理順:

```
1. SwitchDefenceStyle()        ← 防御側の AimStyle を決定
2. CheckPhysicsConsecutiveAimBoost() ← 物理属性一致で1.3倍ATK
3. ATK計算（共通 + 排他 × 適応率）
4. DEF計算（共通 + 排他）
5. ClampDefenseByAimStyle()    ← 不一致なら0.7クランプ
6. 防御無視率の適用
7. 物理耐性の適用
8. 精神補正の適用
9. 最終ダメージ → HP減算
```
