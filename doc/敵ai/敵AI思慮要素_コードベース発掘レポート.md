## 凡例

## A. ダメージシミュレーション精度系

SimulateDamage / SimulateHitRate に**未反映**の要素群。AIのダメージ予測と実際の戦闘結果が乖離する原因になりうる。ヘルパー化（シミュレーション精度向上）の余地があるが、計算コストや複雑性との兼ね合いで意図的に省略されているものも含む。

### A-1. 精神分散防御

- **ファイル**: `BaseStates.Damage.cs:42-75`, `Config/MentalDispersalConfig.cs:46-64`
- **概要**: 人間状況（高揚/混乱/辛い）かつ特定精神属性の組み合わせで発動。物理ダメージの一部が精神HPに転化する
- **分散率計算**: `min(1.0, (Smiler*2.0 + Taraiton*1.4 + ColdHeartedCalm*1.0) / 162)`
- **精神変換係数**: `max(0, Random(0.95, 1.0) - TentVoid * 0.00675)`
- **シミュレート反映**: 未反映。AIは物理ダメージのみを予測し、精神HPへの転化を考慮できない
- **影響度**: 中〜高。人間状況が変化する長期戦で乖離が大きくなる

### A-2. 刃物即死クリティカル

- **ファイル**: `BaseStates.Damage.cs:80-103`
- **概要**: 攻撃者のBlades（刃物）十日能力値に基づく即死判定。HP残存時のみ発動
- **発動確率**: Blades / 150 でしきい値発生 → Blades / 被害者総十日能力でクリティカル発動
- **シミュレート反映**: 未反映。即死確率をAIが事前推定する関数がない
- **影響度**: 中。Blades値が高いキャラのスキル選択に影響

### A-3. 互角一撃生存判定

- **ファイル**: `BaseStates.BattleEvent.cs:393-450`
- **概要**: 致死ダメージ範囲（MaxHP*34〜66%）で「奇跡的に生き残る」確率判定
- **発動条件**: HP >= MaxHP*20% AND 攻撃者総十日能力 <= 被害者総十日能力*1.6
- **シミュレート反映**: 未反映。ダメージシミュレーションはダメージ値のみ返却し、致死判定を含まない
- **影響度**: 低〜中。致死圏のダメージ判断のみに影響

### A-4. かすりダメージ倍率の精密化

- **ファイル**: `BaseStates.Damage.cs:1266-1290`
- **概要**: かすり時のダメージ倍率はAGI差で0.02〜0.35倍の非線形減衰
- **シミュレート反映**: 部分的。命中率計算で「かすり化近似」は実装済みだが、かすり時のダメージ減衰率自体は未反映
- **影響度**: 中。AGIが高い敵へのかすり多発スキルで乖離

### A-5. AimStyle不一致クランプ

- **ファイル**: `BaseStates.Damage.cs:351-373`
- **概要**: 防御側のAimStyleが攻撃側と不一致の場合、DEFが0.7倍にクランプ
- **シミュレート反映**: 未反映。SimulateDamageでは防御側のAimStyle選択を予測しない
- **影響度**: 中。防御スタイルがランダムに変わる敵では予測困難

### A-6. がむしゃら補正

- **ファイル**: `BaseStates.Damage.cs:135-155`
- **概要**: 連続攻撃の2回目以降、被害者のEYE/AGI平均が攻撃者EYEを超える場合にダメージ増幅
- **増幅率**: `1.0 + floor((敵(EYE+AGI)/2 - 攻撃者EYE) / 5) * 0.01`（回数分乗算）
- **シミュレート反映**: 未反映
- **影響度**: 低〜中。連続攻撃スキル選択時のみ

### A-7. 連続攻撃時の物理属性ブースト

- **ファイル**: `BaseStates.Damage.cs:240-260`
- **概要**: 連続3回目以降、スキルのAimStyleと武器の物理属性が一致すると攻撃力1.3倍
- **シミュレート反映**: 未反映
- **影響度**: 低。武器属性一致時のみ

### A-8. TLOA威力減衰

- **ファイル**: `BaseStates.Damage.cs:228-238`
- **概要**: 敵HPが38%以下の場合、TLOAスキルの威力が0.7倍に減衰
- **シミュレート反映**: 未反映（HP条件自体はSimulate時点で判定可能）
- **影響度**: 低。HP極限時のTLOAスキル選択のみ

### A-9. 慣れ補正（AdaptToSkill）

- **ファイル**: `BaseStates.AdaptToSkill.cs` 全般, `doc/総合戦闘仕様書.md:119-152`
- **概要**: 同じ「スキル印象」を複数回受けると、その印象への慣れ補正が発生しダメージが段階的に減衰。精神属性ごとに異なるグルーピング方式
- **シミュレート反映**: 未反映
- **影響度**: 高。長期戦では累積効果が大きい。同スキル連発戦略の評価に直結

### A-10. 戸惑い

- **ファイル**: `BaseStates.AdaptToSkill.cs:782-805`
- **概要**: 同じスキル印象でも、異なる攻撃者から初めて受けると「戸惑い」が発生し慣れ補正が無効化される
- **戸惑わない精神属性**: Doremis, GodTier, Kindergarten, Cquiest
- **シミュレート反映**: 未反映
- **影響度**: 低。初対面グループのみ有効

### A-11. パッシブによるダメージ軽減

- **ファイル**: `BasePassive.cs:1042`（DamageReductionRateOnDamage）, `BasePassive.cs:500,537,590`（PassivesOnBeforeDamage等）
- **概要**: 敵パッシブによる全ダメージ減衰率（0〜1.0）。DEF増加パッシブも未反映
- **シミュレート反映**: 未反映。思慮レベル2+でパッシブ読み可能だが、SimulateDamage計算には未統合
- **影響度**: 中〜高。防御パッシブ持ちの敵へのダメージ過大評価

### A-12. パッシブの最大HPダメージ制限

- **ファイル**: `BasePassive.cs:969-1037`
- **概要**: `DontDamageHpMinRatio` — 「最大HPの30%以上のダメージを受けない」等のキャップ
- **シミュレート反映**: 未反映
- **影響度**: 中。大ダメージスキルが実は効かない場面での判断ミス

## B. 敵・ターゲットの追加情報

現行11.2の「敵情報」テーブルに含まれていない、AIが参照可能な敵の状態情報。全てpublicアクセス可能で新ユーティリティ不要。判断ロジックはPlan()手書き。

### B-1. 敵の人間状況（Demeanor）

- **ファイル**: `BaseStates.StatesState.cs:597+`
- **8状態**: Normal, Painful, Optimistic, Elated, Resolved, Angry, Doubtful, Confused
- **AI活用**: 精神分散防御のトリガー判定、敵の攻撃傾向変化の予測
- **情報取得**: `target.NowCondition` はpublic
- **判断ロジック**: Plan()手書き（「高揚状態の敵には精神攻撃が有効」等）

### B-2. 敵の精神乖離状態

- **ファイル**: `BaseStates.BattleEvent.cs:259-355`
- **概要**: 精神HPと実HPの乖離度合い。乖離発動ターン数 = `(SpringNap - TentVoid/2) / NightDarkness`
- **AI活用**: 精神的耐性の指標。値が大きいほど精神ダメージに強い
- **情報取得**: 乖離関連プロパティはpublic
- **判断ロジック**: Plan()手書き

### B-3. 敵のバリア層詳細

- **ファイル**: `BaseVitalLayer.cs` 全般
- **プロパティ**: HeavyResistance, voltenResistance, DishSmackRsistance, MentalPenetrateRatio, BarrierResistanceMode（A/B/C/C_MaxHP）
- **AI活用**: 「この敵のバリアにはheavy攻撃が有効」等の属性別戦略。SimulateDamageのSimlateVitalLayerPenetrationオプションで自動計算されるが、事前の属性判断は手書き
- **情報取得**: VitalLayersリストはpublic
- **判断ロジック**: Plan()手書き

### B-4. 敵の落ち着きカウント

- **ファイル**: `BaseStates.EvasionCalc.cs:48-63`, `BaseStates.ATKCalc.cs:31-46`
- **概要**: CalmDownCount（0〜CalmDownCountMax）。スキル使用直後は回避率・攻撃力が変動し、ターン経過で平準化
- **AI活用**: 敵がスキル使用直後なら回避率が一時的に変動していることを知り、攻撃タイミングを判断
- **情報取得**: public
- **判断ロジック**: Plan()手書き

### B-5. 敵の武器適応率

- **ファイル**: `BaseStates.WeaponManager.cs:43`
- **概要**: AdaptationRate（0.77〜1.0）。武器切替直後に低下、戦闘継続で回復
- **AI活用**: 武器切替直後の敵はダメージが下がっている→攻撃のタイミング判断
- **情報取得**: public
- **判断ロジック**: Plan()手書き

### B-6. 敵の属性ポイント（AttrP）

- **ファイル**: `BaseStates.P.cs:99-150`
- **概要**: 精神属性ごとのポイント管理。強力スキルの準備状況を示す
- **AI活用**: 「敵の属性ポイントが溜まっている→強力スキルが来る」予測
- **情報取得**: `target.GetAttrP(属性)` はpublic
- **判断ロジック**: Plan()手書き（思慮レベル3+想定）

### B-7. 敵の思えの値（ResonanceValue）

- **ファイル**: `BaseStates.StatesNumber.cs:1262-1326`
- **概要**: 基本値 = TenDayValuesSum * 0.56 * スケーリング + 馬鹿値 * 1.3。知能の逆数的指標
- **AI活用**: 将来的な感情判断AIの基盤
- **情報取得**: public
- **判断ロジック**: Plan()手書き

- **AI活用**: 敵のモチベーション上昇を察知→より強い攻撃が来ると予測
- **情報取得**: CharaCompatibility + NowConditionで追跡可能
- **判断ロジック**: Plan()手書き

## F. 特殊メカニクス

特定条件でのみ発動する特殊システム。AIが直接制御するものではないが、戦術判断の背景知識として重要。

### F-1. 慣れフラットロゼ（追撃）

- **ファイル**: `FlatRozeEffect.cs`, `doc/総合戦闘仕様書.md:77-116`
- **概要**: 前のめり + パッシブID0 + 攻撃スキル + 使用回数20回超 + 命中率50-60付近で追撃判定（27%）
- **追撃ATK倍率**: 1.6 + 泉水*0.005
- **AI活用**: 敵が追撃条件を満たしている場合のダメージ上乗せ予測
- **情報取得**: DoCount, SkillHitPer等はpublic
- **判断ロジック**: Plan()手書き

### F-2. 乖離スキル精神消耗

- **ファイル**: `Config/MentalExhaustionConfig.cs`
- **概要**: 精神属性と異なるスキル（乖離度8以上）を連発すると精神HP消耗が加速。蓄積値3.0超で精神DEF 0.5倍、6.0超で0.25倍
- **AI活用**: 「乖離スキルを使い続けると精神が崩壊する」自制判断、または敵がこの状態に陥っているかの推測
- **情報取得**: 使いこなし度（mastery）はCalcCradleSkillLevelで計算
- **判断ロジック**: Plan()手書き

### F-3. パワーレベル遷移確率

- **ファイル**: `BaseStates.StatesState.cs:225-591`
- **概要**: 歩行時にVeryLow/Low/Medium/Highが遷移。精神属性ごとに異なる確率テーブル
- **例**: Doremis=不安定（High→Medium 35%）、Pillar=安定（High→Medium 2.23%）
- **AI活用**: 再戦時の敵パワー予測（戦闘内ではなく戦闘間の変動）
- **判断ロジック**: 戦闘外の情報

### F-4. パワーアップチャンス（ハイネスチャンス）

- **ファイル**: `BaseStates.StatesState.cs:203-220`
- **概要**: 敵を倒した時、倒した敵のTLOAスキルが自分の精神属性と一致するとパワーアップ確率上昇
- **基礎確率**: Kindergarten=40%, LiminalWhiteTile=30%, その他=20%。一致スキル数*5%加算
- **AI活用**: 戦闘後のパワーアップ予測
- **判断ロジック**: 戦闘外の情報

## 設計思想メモに基づく3層フレームワーク分析

仕様書11.6の設計思想（情報取得/判断ロジック/操作の3層フレームワーク + ヘルパー化判断基準）を全52要素に適用し、AI内での扱い方を決定する。

### 判断の前提（11.6より）

- **シミュレートの設計意図**: 「厳密にBM上での実行計算を再現する必要はない。簡易シミュレートにすることで『予想と違った！』感が出る」
- **ヘルパー化の判断**: キャラ共通の基礎判断→ヘルパー、キャラ固有の戦略判断→手書き
- **引数発散パターン**: 汎用化すると引数が発散し、結局全派生でoverrideするため関数として成立しない→手書き
- **情報取得は前提層**: 判断がヘルパーでも手書きでも、その要素を扱うなら常に必要

---

### カテゴリA判定: シミュレーション精度系（12要素）

**設計思想との関係**: 「簡易シミュレートで『予想と違った！』感を出す」設計意図に照らすと、A群の大半は**意図的に省略されている**。全て組み込むとシミュレートが実戦闘と一致してしまい、AIの「学習・適応」物語が失われる。

**方針**: 基本的に**SimulateDamageには組み込まない**。ただし、特定の要素は「手書きAIが意識的に考慮するための情報」として扱う。

| 要素 | 情報取得 | 判断ロジック | 操作 | 方針 |
|---|---|---|---|---|
| A-1 精神分散防御 | target.NowCondition + MyImpression（public） | **手書き**: 「高揚/混乱の敵には精神攻撃が追加で効く」判断はキャラ固有 | 既存 | **シミュレート不組込**。Demeanor確認はB-1と合わせて手書き材料 |
| A-2 刃物即死クリティカル | user.TenDayValues[Blades]（public） | **手書き**: 「Blades値が高いなら刃物スキル優先」はキャラ固有の強み認識 | 既存 | **シミュレート不組込**。「予想外の即死」が物語性を生む |

| A-9 慣れ補正 | **情報取得が困難**: 慣れ蓄積はprivateで外部から読めない | — | — | **無視してよい**。設計意図通り「予想と違った！」を生む要素。BattleMemoryの被害記録から間接的に学習する方が自然 |

### カテゴリE判定: パッシブ詳細（10要素）

| E-3 ステータス補正値 | ダメージ予測の補正 | **手書き**: パッシブ読み+ダメージ補正の組み合わせはキャラ固有の知性表現 |
| E-4 攻撃者限定補正 | 「自分への攻撃は回避されやすい」 | **手書き**: ターゲット切替判断はキャラ固有 |

| E-6 生存条件 | 「バリア破壊でパッシブも消える」 | **手書き**: 同上 |
| E-7 スキル発動率 | 「発動率低下→安全に攻撃できる」 | **手書き**: リスク許容度はキャラ固有 |
| E-8 ターゲット確率 | 「囮パッシブ認識」 | **手書き**: 囮を無視するか狙うかはキャラ固有 |
| E-9 リカバリーターン補正 | 「敵の行動が遅い→チャンス」 | **手書き**: タイミング判断 |
| E-10 回復効率補正 | 「回復効きにくい→回復スキル避ける」 | **手書き**: 回復戦略はキャラ固有 |

**E群の結論**: 全10要素が**手書き材料**。情報取得はReadTargetPassives()（既存）で統一的に対応済み。パッシブの内容判断は本質的にキャラ固有の知性表現であり、ヘルパー化は引数発散パターンに陥る。

**E群の設計パターン**: Plan()内で以下の形で使用:
```
var passives = ReadTargetPassives(target);
foreach (var p in passives) {
    if (p.DamageReductionRateOnDamage > 0.3f) { /* 高軽減→避ける */ }
    if (p.DurationTurn > 0 && p.DurationTurn <= 2) { /* あと2ターンで消える→耐える */ }
    // ... キャラ固有の判断
}
```

### カテゴリF判定: 特殊メカニクス（4要素）

| 要素 | 判断ロジック | 方針 |
|---|---|---|
| F-1 慣れフラットロゼ | 追撃条件を満たしている味方のダメージ上乗せ予測 | **手書き材料**。ただし条件が複雑（前のめり+パッシブID0+20回超+命中率50-60）で、気にするキャラは稀 |
| F-2 乖離スキル精神消耗 | 自制判断 | **手書き材料**。「乖離度が高いスキルを連発しない」はキャラ固有の自制表現 |
| F-3 パワーレベル遷移 | 戦闘外の情報 | **AI思考の範囲外**。戦闘内AIの判断材料ではない |
| F-4 パワーアップチャンス | 戦闘後の情報 | **AI思考の範囲外** |

**F群の結論**: F-1,F-2は手書き材料（ただし使用頻度低）。F-3,F-4は戦闘内AIの思慮範囲外。

### 全体結論

#### ヘルパー化判定の結果

| 判定 | 要素数 | 内容 |
|------|-------|------|
| **ヘルパー化を要検討** | 1 | D-1 連続攻撃期待回数（AnalyzeBestDamageへの統合候補） |
| **情報取得ユーティリティ候補** | 0 | 全要素がpublic1行アクセスまたは既存ユーティリティで対応済み |
| **手書き材料** | 39 | Plan()内でキャラ固有の戦略として記述 |
| **SimulateDamage組込** | 0 | 設計意図「予想と違った！感」に合致。精度向上は意図的に行わない |
| **無視してよい** | 8 | A-3,6,7,9,10（AI制御外/自動処理）, D-7（既存フィルタ済み）, F-3,4（戦闘外） |
| **保留** | 2 | B-2 乖離計算ユーティリティ, C-6 精神属性別逃走率のShouldEscape統合 |

#### 設計思想との整合性

52要素の分析結果は11.6の設計思想と完全に整合する:

1. **「シミュレートは簡易でよい」** → A群12要素中0個をSimulateDamageに組み込む判断
2. **「キャラ固有の戦略は手書き」** → B-E群の大半が手書き材料と判定
3. **「引数発散パターンはヘルパー化しない」** → ストック溜め/ポイント温存と同構造の要素を正しく手書き判定
4. **「情報取得は前提層として常に整備」** → 全要素がpublicアクセスまたは既存ユーティリティで対応済みであり、新規整備不要

#### 唯一のアクション候補: D-1 連続攻撃期待回数

AnalyzeBestDamageが「1回分のダメージ」で比較する現状では、RandomConsecutivePerが高いスキル（例: 60%で連続 → 期待2.5回）の実効ダメージが過小評価される。`EstimateExpectedDamage(target, skill)` 内で `expectedDmg *= EstimateHitCount(skill)` のように期待攻撃回数を乗算する拡張が考えられる。

ただしこれは「キャラ共通の基礎判断」に該当するため、実装するなら SkillAnalysisPolicy にフラグ（`considerConsecutiveExpectedCount: bool`）を追加し、Inspector調整可能にする方式が適切。
