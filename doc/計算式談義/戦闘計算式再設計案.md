## 0. 簡易まとめ — 4式がどう変わるか

### 式I: 通常・本体

```
旧: (ATK - DEF) × SP + SP
新: Floor固定値 + SP × max(0, ATK - DEF)
```

SPが「倍率」と「最低保証」を兼ねていたのを分離。壁を越えたら線形に報われる、越えなければ小さな固定値だけ。シンプル。

### 式II: 魔法・本体

```
旧: MagicBlend(ATK,DEF) × SP×0.5 + SP × ATK × 0.09
新: Floor固定値 + SP × k × ATK²/(ATK²+DEF²) + λ × SP × ATK_raw^r/(ATK_raw^r+DEF^r)
```

シグモイドブレンド+if分岐+漏れ項の魔窟が、きれいな1式に。if分岐なし、負値なし、ゼロ除算なし。漏れ項もPower Ratio形式で暴走しない。

### 式III: 通常・精神

```
旧: (ATK × boost - MentalDEF) × SPm + SPm    ← 本体ATKを流用、余裕ブースト暴走リスク
新: Floor固定値 + SPm × max(0, 精神ATK - 精神DEF) × (1 + β × tanh(α × 余裕差))
```

式Iのコピペだったのが独自の入力+余裕の刃を持つ。精神ATK/DEFが十日能力の別テーブルから来る。余裕補正はtanhで±50%に飽和。7300万ダメージは二度と出ない。

### 式IV: 魔法・精神

```
旧: ATK × boost / MentalDEF × SPm×0.6 + SPm×0.7    ← ゼロ除算リスク、他式と構造が全く違う
新: Floor固定値 + SPm × k × 精神ATK²/(精神ATK²+精神DEF²) × W(余裕差) + 漏れ項
```

4式の中で一番壊れていたのが、フレームワークに統合。式IIの構造 × 式IIIの余裕補正。2軸直交から自然に導出される。

### 要するに

4式バラバラだったものが「**Floor + SP × Scale × Modifier + Leak**」の1つの骨格から、軸を変えるだけで全部出てくるようになる。バグの温床（if分岐、ゼロ除算、暴走ブースト、コピペ不統一）が構造的に消える。

## 2. 再設計の核：十日能力の重みテーブル拡張による精神ステ分離

### 2.1 現行の仕組み

現行システムでは、37種の十日能力がProtocol（攻撃規格）・AimStyle（狙い方）の係数テーブルで重み付き合成され、1つのATK/DEFに畳み込まれている。

```
ATK = Σ(十日能力i × 共通係数i) + Σ(十日能力j × 排他係数j[Protocol])
DEF = Σ(十日能力i × 共通係数i) + Σ(十日能力j × 排他係数j[AimStyle])
```

この設計の面白さ：「同じATK=50でも中身が違う」。キャラクターの個性が数値の内訳に宿る。

### 2.2 提案：精神用重みテーブルの追加

同じ十日能力に対して、本体用と精神用で**異なる重みテーブル**を用意する。

```
本体ATK = Σ(十日能力i × 本体用係数i) + Σ(十日能力j × 本体用排他係数j[Protocol])
精神ATK = Σ(十日能力i × 精神用係数i) + Σ(十日能力j × 精神用排他係数j[Protocol])

本体DEF = Σ(十日能力i × 本体用係数i) + Σ(十日能力j × 本体用排他係数j[AimStyle])
精神DEF = Σ(十日能力i × 精神用係数i) + Σ(十日能力j × 精神用排他係数j[AimStyle])
```

### 2.3 精神用重みの特徴

精神用係数テーブルでは、以下の十日能力に重みが偏る（例）：

- **Leisure（余裕）** — 心の安定から来る精神的な圧
- **Smiler（微笑み）** — 表情による心理的揺さぶり
- **Calm（落ち着き）** — 精神的な安定感
- その他、心理・感情系の十日能力

一方、Strength（力）やSpeed（速度）などの肉体系は精神用係数が低い。

### 2.4 この案の利点

| 観点 | 利点 |
|------|------|
| **プレイヤー負荷** | プレイヤーに見えるのは今まで通りATK/DEFのみ。内部処理が変わるだけ |
| **キャラ個性の深化** | 「物理は強いが精神が弱い」「精神攻めが得意」などのビルド多様性が自然に生まれる |
| **既存設計との整合** | 十日能力 × 係数テーブルの仕組みをそのまま活用。新しいシステムを足さない |
| **設計哲学の拡張** | A-8（非対称性）の自然な拡張。「同じ人間が状況で違う顔を見せる」が本体/精神の軸にも適用される |
| **余裕(Leisure)の位置づけ** | A-5を自然に包含。余裕は精神用係数テーブルで高い重みを持つことで、精神ATKに大きく寄与する |

### 2.5 現行の余裕ブーストとの関係

現行では余裕(Leisure)の差分が精神ダメージに別枠でブーストとして乗算されている。再設計案では：

- 余裕の影響は**精神ATKの重みテーブル内に吸収**される
- 「相手より余裕がある」→ 精神ATKが高い → 精神ダメージが大きい、という流れは保存
- 別枠ブーストの暴走リスク（B-12, B-13）が構造的に解消される

ただし、余裕の「差分」による影響をどこまで重みテーブルに吸収するか、あるいは小さな補正項として残すかは要検討。

## 3. 再設計フレームワーク：3層分離 × 2軸直交

### 3.1 3層分離（C-1回答より）

現行のSPが「倍率」と「最低保証」を兼ねている問題を解消する。

```
ダメージ = Floor + SP × Scale(ATK, DEF) × Modifier

Floor    : 技を振った以上の最低保証（A-1「行為の意味の保証」）
SP       : スキル倍率専任
Scale    : 攻防関数（攻撃モデルごとに異なる）
Modifier : 精神属性相性、余裕補正などの後段補正
```

- Floorの値はSPとは独立に設計可能（SPが上がってもFloorは変わらない）
- これによりDEFの存在意義が構造的に保護される

### 3.2 2軸直交フレームワーク（C-5回答より）

4式を「攻撃モデル軸」と「ダメージ領域軸」の直交で統一する。

```
              本体ダメージ           精神ダメージ
           ┌────────────────┐    ┌────────────────┐
  通常攻撃 │ Floor_N        │    │ Floor_N        │
           │ + SP ×         │    │ + SP ×         │
           │   差分型Scale  │    │   差分型Scale  │
           │                │    │ × 余裕補正     │
           ├────────────────┤    ├────────────────┤
  魔法攻撃 │ Floor_M        │    │ Floor_M        │
           │ + SP ×         │    │ + SP ×         │
           │   比率型Scale  │    │   比率型Scale  │
           │ + 漏れ項       │    │ × 余裕補正     │
           └────────────────┘    └────────────────┘
```

**攻撃モデル軸が決めるもの（行方向）：**
- 通常 → 差分型Scale（DEF壁あり、ハイリスク・ハイリターン）
- 魔法 → 比率型Scale + 漏れ項（DEFに負けにくい、安定型）

**ダメージ領域軸が決めるもの（列方向）：**
- 本体 → 本体ATK/DEFを使用
- 精神 → 精神ATK/DEFを使用 + 余裕補正（Modifier層）

各セルの計算は「行の攻撃モデル」と「列のダメージ領域」の組み合わせで一意に決まる。

### 3.3 Scale関数（確定）

#### 通常（差分型） — 線形のまま保持

```
Scale_Normal(atk, def) = max(0, atk - def)
```

ATK < DEF → Scale = 0 → Floorのみ。これが「通常のDEF壁」（A-2）。

**線形発散の対策は式の外で行う（確定）。** 理由：
- 式の中でバウンドすると `max(0, ATK-DEF)` が本質的に比率型に化け、3.6「÷ vs -が原初の設計核」が死ぬ
- HPは十日能力と無関係な独立した成長軸。ATK-DEF差とHPの比率はレベルデザインで制御可能
- A-2「ハイリターン」を保存。壁を越えた先の報酬が頭打ちにならない

外部対策の優先順位：
1. 十日能力の成長曲線にソフトキャップ（既存の成長距離15制限）
2. ATK係数テーブルの正規化（Raincoat=22.0の支配問題の解消）
3. 敵のDEF設計をプレイヤーATKに追随させる
4. 補正チェーン内のソフトキャップ（最終手段）

#### 魔法（比率型） — Power Ratio型

```
Scale_Magic(atk, def) = k × atk^q / (atk^q + def^q)

q = 2（推奨。全エージェント一致）
k = 要調整（ScaleRatioの上限値）
```

- ATK < DEF でもゼロにならない（A-4: 魔法の最低保証性）
- ATK >> DEF で k に漸近（天井あり、暴走しない）
- ATK = DEF で k/2（互角時に安定した中ダメージ）
- q=2 で通常とのクロスポイントが ATK/DEF ≈ 1.3 付近に来る
- if分岐不要（旧シグモイドブレンドのif矛盾を構造解消）

ゼロ除算ガード:
```
float denom = atkQ + defQ;
float scale = denom < 0.0001f ? k/2 : k * atkQ / denom;
```

#### 漏れ項（魔法専用） — Power Ratio方式（確定）

```
Leak(sp, atk_raw, def) = λ × sp × atk_raw^r / (atk_raw^r + def^r)

λ   = 0.05〜0.15（漏れ強度）
r ≤ q（メイン項を超えない構造的制約）
```

- A-9「漏れ項概念」を独立した項として保存
- 漏れ項が[0, λ×SP)に有界。旧式の0.09問題（LeakがScale_Magicを圧倒）が構造的に再発しない
- r ≤ q の制約で、漏れ項がメイン項を超えることが原理的に不可能
- atk_raw（素の自分）を使用。「根源的な魔力の滲み出しは興奮に左右されない」

精神ダメージ側にも漏れ項あり（2軸直交の自然な帰結）:
```
Leak_mental(spm, am_raw, dm) = λ_m × spm × am_raw^r / (am_raw^r + dm^r)
```

### 3.4 Floor層（確定） — SP完全独立

```
Floor = f（スキルランクごとの固定値）

基本スキル: f = 2〜4
上位スキル: f = 5〜8
TLOA:      f = 8〜12
```

- C-1回答「SPの二重人格は偶然」に最も忠実な設計。SPとFloorを完全に切る
- SPが育ってもFloorは変わらない → DEFの存在意義が構造的に保護される
- ゲーム進行でFloorの相対的価値は自然に低下（HPの成長で）
- 通常/魔法でFloor値を変えることも可能（f_m > f_n で魔法の安定性をFloorレベルでも保証）

目安: FloorはHP比1-3%。「ゼロじゃなかった」の安心感を確保しつつ、Scale層の意味を殺さない。

### 3.5 余裕補正（精神ダメージ専用Modifier）

```
W(L_atk, L_def) = 1.0 + β × tanh(α × (L_atk - L_def))

α = 0.03〜0.06（感度。最大Leisure差100想定で飽和手前に）
β = 0.5（推奨。最大±50%の精神ダメージ変動）
```

- tanhにより値域 [1-β, 1+β] で暴走が原理的に不可能（B-12の7300万ダメージが再発しない）
- 劣勢時でもW=0.5（β=0.5時）。精神攻撃が死に手にならない（C-2回答を反映）
- β < 1.0 を厳守（β≧1.0でダメージが負になる）
- 本体ダメージではModifier=1.0固定。余裕は精神ダメージ固有（A-5）

## 4. 確定した4式

### 記号定義

```
A       = 本体ATK(SkillAttackModifier)  — 興奮込みの自分
A_raw   = 本体ATK()                     — 素の自分
D       = 本体DEF
Am      = 精神ATK                       — 精神用重みテーブルから算出
Am_raw  = 精神ATK（素の自分）
Dm      = 精神DEF                       — 精神用重みテーブルから算出
SP      = SkillPower（本体向け）
SPm     = SkillPowerForMental（精神向け）
f_n     = 通常Floor（スキルランク固定値）
f_m     = 魔法Floor（f_m ≥ f_n）
k       = ScaleRatioの上限値
q       = 2（ScaleRatioの曲線パラメータ）
λ, λ_m  = 漏れ項係数（0.05〜0.15）
r       = 漏れ項の曲線パラメータ（r ≤ q）
W       = 1.0 + β × tanh(α × (L_atk - L_def))
α       = 余裕感度（0.03〜0.06）
β       = 余裕最大補正幅（0.5）
```

### 式I: 通常攻撃・本体ダメージ

```
Damage_NB = f_n + SP × max(0, A - D)
```

体験: 力と力の正面衝突。壁を越えれば線形に報われる。越えなければFloorのみ。

### 式II: 魔法攻撃・本体ダメージ

```
Damage_MB = f_m + SP × k × A^q / (A^q + D^q) + λ × SP × A_raw^r / (A_raw^r + D^r)
```

体験: 壁を壊すのではなく透過する。常に安定したダメージ。漏れ項は素の自分の魔力の滲み。

### 式III: 通常攻撃・精神ダメージ

```
Damage_NM = f_n + SPm × max(0, Am - Dm) × W(L_atk, L_def)
```

体験: 精神の壁との勝負 + 余裕という第3の刃。落ち着いている者が心を攻める。

### 式IV: 魔法攻撃・精神ダメージ

```
Damage_MM = f_m + SPm × k × Am^q / (Am^q + Dm^q) × W(L_atk, L_def) + λ_m × SPm × Am_raw^r / (Am_raw^r + Dm^r)
```

体験: 安定した浸透で精神を蝕む + 余裕の刃。最も安定した精神攻撃手段。

### 統一テンプレート

4式すべてが同一の骨格から生成される:

```
ダメージ = Floor(model) + SP × Scale(ATK, DEF, model) × Modifier(domain) + Leak(SP, ATK_raw, model)

攻撃モデル軸(model)が決めるもの:
  Normal → ScaleDiff, Leak=0, Floor=f_n
  Magic  → ScaleRatio, Leak=λ×SP×..., Floor=f_m

ダメージ領域軸(domain)が決めるもの:
  Body   → 本体ATK/DEF, Modifier=1.0
  Mental → 精神ATK/DEF, Modifier=W(Leisure差)
```

## 5. まとめ：決定事項と残る調整

### 確定した設計判断

| 判断事項 | 決定 | 根拠 |
|---------|------|------|
| 3層分離 | Floor / Scale / Modifier 独立 | C-1回答 + 全エージェント合意 |
| 差分型Scaleの発散対策 | **式の外で対処** | 式の中だと「引く→割る」に化け3.6が死ぬ。HPが十日能力外で外部制御可能 |
| 魔法Scale | Power Ratio型、q=2 | 全エージェント合意 |
| Floor | SP完全独立のスキルランク固定値 | C-1回答に最も忠実 |
| 漏れ項 | Power Ratio方式 | A-9保存 + 0.09問題の構造的防止 |
| 余裕補正 | tanh飽和、β=0.5 | C-2回答 + 暴走原理封殺 |
| 精神ATK/DEF | 十日能力の精神用重みテーブル | C-4回答 + A-7拡張 |
| 4式の統一 | 2軸直交の離散enum切替 | C-5回答 |

### 残る調整パラメータ（ゲーム数値帯確定後に決定）

| パラメータ | 推奨検討レンジ | 決定手順 |
|-----------|--------------|---------|
| k（ScaleRatio上限値） | 要シミュレーション | ATK-DEF差の典型値とScale値のバランスで調整 |
| λ, λ_m（漏れ強度） | 0.05〜0.15 | Scale=k/2（互角時）に対し漏れが10-20%程度 |
| r（漏れの曲線パラメータ） | r ≤ q | 漏れ項がメイン項を超えない構造的制約 |
| α（余裕感度） | 0.03〜0.06 | Leisure差の最大想定値で飽和手前に |
| f_n, f_m（Floor値） | HP比1-3%目安 | 序盤HPから逆算 |
| 精神用重みテーブルの係数 | — | Leisure, Smiler等に高重み。1種が支配的にならないよう注意 |

### 決定の正しい手順

```
① ゲーム全体の十日能力成長レンジを確定
② ATK/DEF/精神ATK/精神DEFの典型値帯を導出
③ k を値帯に合わせて仮設定
④ α, β をLeisureの典型レンジに合わせて仮設定
⑤ 序盤・中盤・終盤のシナリオでシミュレーション
⑥ テストプレイで微調整
```
