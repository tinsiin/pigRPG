# 戦闘計算式エッセンス抽出 — 再構成用

本書は、4つのダメージ計算式の再設計に先立ち、式から抽出した「保存すべき本質」「捨てていい偶然」「制作者判断が必要な保留事項」をまとめたもの。

関連:
- `doc/戦闘計算式評価レポート.md` — 評価・批判・専門家議論の記録
- `doc/スキル計算仕様書.md` — 計算式の正式定義

---

## 1. このゲームの戦闘が目指す1つの感情

> **「力と力のぶつかり合いの中で、相手の体だけでなく心の揺れまで読みながら、自分自身も揺れ動く — その不安定さの中で最善手を掴み取る、ヒリヒリした駆け引きの快感」**

## 2. 攻撃種別ごとの体験キーワード

| 攻撃種別 | キーワード | 体験の核 |
|----------|-----------|----------|
| **物理** | 賭け・突破・力の証明 | 相手の壁に正面からぶつかる。通るか通らないかのドキドキ |
| **魔法** | 確実・浸食・じわりとした支配 | 壁を壊すのではなく透過する。派手さはないが着実に削る |
| **精神** | 見透かす・格の差・残酷な優位 | 心の立ち位置で圧倒する。余裕がある者が余裕のない者を蹂躙する |

## 3. 式に刻まれた暗黙の設計哲学

多角的議論から発掘された、制作者の無意識の設計判断:

### 3.1 「行為の意味の保証」

4式すべてに `+SP系` のフロアがある。技を振った以上、何かが起こらなければならない。ゼロダメージ体験への拒否。

### 3.2 「にじみ」の感性

if分岐ではなくシグモイドで2つのモデルを滑らかにブレンドした。世界を「分類」ではなく「グラデーション」として捉える思想。十日能力の係数テーブル、引き締め値の段階的適応、落ち着きカウントの線形補間にも同じ感覚が通底する。

### 3.3 「素の自分」と「興奮した自分」

`ATK(SkillAttackModifier)` = 興奮込みの自分。`ATK()` = 素の自分。

- 本体ダメージの主項 → 興奮込み（技の勢いが乗る）
- 魔法の漏れ項 → 素の自分（根源的な魔力の滲み出し）
- 精神ダメージ全般 → 素の自分（心への攻撃は興奮に左右されない）

「落ち着き」という名前が語る通り、スキル使用直後は興奮し、時間とともに素の自分に戻る。

### 3.4 「余裕 = 最も深い意味での強さ」

精神攻撃力がINTでもWISでもなく「余裕(Leisure)」。しかも絶対値ではなく相手との「差」。相手より余裕がある者が、相手の心を攻められる。

光と影の両面: 余裕は追い詰められた敵を救う慈悲にもなりうるが、弱者を踏みにじる暴力にもなる。

### 3.5 全体テーゼ

> **「戦いとは数値の大小ではなく、人間と人間の関係性である」**

同じ力が戦い方(Protocol)と防ぎ方(AimStyle)で異なる面を見せる。25+の十日能力が1つのATK/DEFに畳み込まれ、状況で違う顔が現れる。「人間は一人の人間でありながら、状況によって違う顔を見せる」という人間観が数式に焼き込まれている。

### 3.6 「÷ vs -」が原初の設計核

初期メモ（`魔法スキル.md`）から判明: シグモイドブレンドより前の段階で、制作者は「通常は引く(-)、魔法は割る(÷)」という**算術演算レベルでの差別化**を意図していた。A-2（ハイリスク vs 安定）はこの算術的差異から自然に生まれた結果であり、逆ではない。現行のシグモイドブレンドは、この原初の÷思想を2モデル間の滑らかな遷移として発展させたもの。

### 3.7 インフレ耐性のトレードオフは自覚的

初期メモに「インフレしない = 突き詰めると弱い？」と記録されている。÷計算が終盤のステータスインフレに対して伸び悩む問題を、制作者は**設計初期から認識していた**。これは偶然の欠陥ではなく、未解決のまま保留されたトレードオフ。再設計時にはこの問題への明示的な回答（許容するか、対策を入れるか）が必要。

## 4. 保存すべきエッセンス（A群: 再設計でも必ず残す9項目）

### A-1. ATK/DEF共有、式の構造で物理/魔法を差別化

物理と魔法で別ステ(ATK/MAG, DEF/RES)を持たない。同じATK/DEFに対して計算式の構造だけが異なる。

**保存理由**: これが消えたら37種の十日能力 x 係数テーブルでキャラの個性を出す設計が半壊する。ビルドの多様性が「物理型/魔法型」の二択に退化する。

### A-2. 通常=ハイリスク・ハイリターン、魔法=ローリスク・ミドルリターン

DEF差に敏感で勝てば爆発・負ければ最低保証のみ（通常）vs DEFに負けにくい安定型だが爆発力は控えめ（魔法）。

**保存理由**: 「硬い敵には魔法、柔らかい敵には物理」という戦術判断の軸。これがなければスキル選択が意味をなさない。

### A-3. SP × f(ATK, DEF) の乗算関係

SkillPowerが攻防関数に乗算される構造。スキルレベル成長(SP)とステータス成長(ATK)の両方がダメージに意味を持つ。

**保存理由**: 二軸成長構造の根幹。どちらかが加算だけになると成長軸の片方が死ぬ。

### A-4. 魔法がDEFに完全に負けない最低保証性

ATK < DEF でも魔法はゼロにならない。手段（除算型、ブレンド、対数等）は問わない。

**保存理由**: 通常式にはこの性質がない（DEF壁でフロアに落ちる）からこそ差別化になっている。魔法の存在意義そのもの。

### A-5. 余裕(Leisure)が精神ダメージ固有の攻撃軸

精神攻撃だけに、十日能力「余裕」の差分がダメージに影響する。本体ダメージには影響しない。

**保存理由**: 精神ダメージが「パラメータを差し替えた本体ダメージ」ではなく、独自の戦術領域として成立する条件。

### A-6. 精神補正（精神属性の相性）がSPに乗算

ComputeSkillPowers内で精神属性ごとの相性が適用率20%/40%でSPに反映。

**保存理由**: 10種の精神属性の意味が戦闘中に体感される経路。これが消えると精神属性が飾りになる。

### A-7. ATK/DEFが十日能力の重み付き合成

37種の十日能力 x 共通係数 + 排他係数 = 1つのATK/DEF。StatesPowerBreakdownで内訳追跡。

**保存理由**: 「同じATK=50でも中身が違う」キャラクターの個性。内訳追跡→思えダメージの連鎖はこのゲーム固有の設計。

### A-8. 排他ATK=自分の規格、排他DEF=相手の攻撃パターンの非対称性

攻撃力は「自分が何者か(Protocol)」で決まり、防御力は「相手が何をしてくるか(AimStyle)」で決まる。

**保存理由**: 格闘ゲーム的な読み合いをRPGに持ち込んだ独創的設計。連続攻撃中の防御適応（引き締め値）の意味の源泉。

### A-9. 魔法の「漏れ項」概念

DEFを完全には遮断できない浸透成分が魔法に存在する。具体的な式（SP x ATK x 0.09）は変えてよいが、概念は保存。

**保存理由**: 「どれだけ硬い相手にも一定量は通る」というファンタジー的直感。通常式にはない魔法固有の味。

## 5. 捨てていい偶然の産物（B群: 再設計で自由に変更・除去してよい13項目）

### フィーリング定数群

| # | 項目 | 理由 |
|---|------|------|
| B-1 | k=0.02, t=140 | ゲームの数値帯未確定のままフィーリングで設定。数値帯確定後に導出すべき |
| B-2 | 除算モデルの+8 | 「思った以上に効果ないから+10とかしとくわ」の産物。設計的根拠なし |
| B-5 | 漏れ項の係数0.09 | ATK≈DEFの通常戦闘で漏れ項がメイン項の2倍。MagicBlendを形骸化 |
| B-7 | 魔法のSP×0.5 | スキルレベルアップ体感を鈍化。通常との差は計算モデルの構造で出すべき |

### 設計矛盾・バグ

| # | 項目 | 理由 |
|---|------|------|
| B-3 | 除算モデルの diff/1.6項 | ATK < DEFでratio計算が負値を返す。最低保証の設計意図と論理矛盾 |
| B-4 | MagicBlend内のif分岐 | シグモイドの滑らかな遷移と矛盾。ATK=DEF境界で微分不連続 |
| B-8 | 式④のゼロ除算ガード欠如 | 式②にはepsilonガードがあるのに30行差の式④にない。実装漏れ |
| B-12 | 余裕ブーストの暴走 | L差55×MentalHP300で7300万ダメージ。バグ |

### 不整合

| # | 項目 | 理由 |
|---|------|------|
| B-6 | 漏れ項がATK()(raw)を使う点 | 意図的な可能性はあるが設計コメントなし。判別不能 |
| B-9 | 式④が純除算でブレンドを使わない | 式②の設計哲学と不一致。4式に統一思想がない象徴 |
| B-10 | 式④のフロアがSPm×0.7 | 他式は×1.0。本体では「魔法はフロアが高い」のに精神では逆。理由なし |
| B-11 | 式③が式①の構造コピペ | 精神攻撃という別コンセプトに対し差分型をそのまま流用 |
| B-13 | 余裕ブーストの`L差×MentalHP×0.2`の形 | MentalHPとの積が暴走の主因。L差だけで十分表現可能 |

## 6. 制作者判断が必要な保留事項（C群: 5項目）

再設計の方向性を大きく左右するため、先に制作者の意図確認が必要。

### C-1. SPの「二重人格」は意図的か？

`SP × (ATK - DEF) + SP` で、SPが「倍率」と「最低保証」を兼ねる。高レベルスキルほどDEFの存在意義が構造的に薄まる。

| 回答 | 再設計の方向 |
|------|-------------|
| **意図的** — 高レベルスキルは防御を意味的に突破する | SP兼任を維持。パラメータ数が少ない利点 |
| **偶然** — 1変数で2役割を兼ねてしまった | Floor層を独立分離: `Floor(SP) × Scale(ATK, DEF)` の3層構造へ |

### C-2. 劣勢時に余裕ブースト=0になるのは意図的か？

`max(0, Leisure差)` により、相手より余裕がない場合ブーストが完全にゼロ。精神攻撃が死に手になり、勝ってる側がさらに勝つ雪崩現象。

| 回答 | 再設計の方向 |
|------|-------------|
| **意図的** — 精神攻撃は「追い打ち」の位置づけ。劣勢時は本体攻撃に専念すべき | ゼロのまま維持。ゲームメッセージとして「余裕なき者に精神の刃はない」 |
| **偶然** — max(0,...)を安易に入れた | 最低値フロアを導入（例: `max(0.1, ...)`）。劣勢でも精神攻撃が選択肢に残る |

### C-3. 漏れ項は通常式にも入れるべきか？

現在DEF完全無視の漏れ項は魔法のみ。通常式にも小さな漏れ項を入れれば「物理でもDEF壁を完全には防げない」表現ができるが、A-2の「通常=ハイリスク」を弱める可能性。

| 回答 | 再設計の方向 |
|------|-------------|
| **通常に壁があるのは意図的** — DEFに負ければ最低保証のみ。それが物理のリスク | 漏れ項は魔法専用のまま。通常のDEF壁がA-2を支える |
| **壁は意図していない** — 漏れ項を入れ忘れた | 通常にも小さな漏れ項を追加。ただしA-2の差別化が薄まるリスクあり |

### C-4. 精神ダメージが本体ATK()を使う設計は意図的か？

式③④の精神攻撃力はATK()（物理と同じ値）を使用。物理が強いキャラは精神攻撃もそれなりに効く。

| 回答 | 再設計の方向 |
|------|-------------|
| **意図的** — 精神を攻めるにも力が要る | ATK()をそのまま使用。物理ビルドでも精神攻撃が一定機能する |
| **別の意図があった** — 精神攻撃は別の能力で決まるべき | 十日能力の別の重み付けで「精神ATK」を算出（Leisure, Smiler等に重み） |

### C-5. 4式の不統一は進化の途中か意図的な非対称か？

通常本体=差分型、魔法本体=シグモイドブレンド、通常精神=差分型(+余裕)、魔法精神=純除算。

| 回答 | 再設計の方向 |
|------|-------------|
| **進化の途中** — 統一したい | 2軸直交フレームワーク: 魔法精神にもブレンドを入れて統一 |
| **意図的な非対称** — 精神は本体と本質的に違うべき | 非対称を維持しつつ、なぜ非対称なのかを文書化 |

## 7. 再設計の前提条件

C群の5問への回答が揃えば、以下が自動的に決定される:

- 式の骨格を3層分離するか（C-1）
- 精神攻撃の戦術的位置づけ（C-2）
- 通常/魔法の差別化の深さ（C-3）
- 精神ATKの算出方法（C-4）
- 4式の統一度合い（C-5）

その上で、A群9項目を骨格として保存し、B群13項目を自由に入れ替えることで再構成を行う。
