# パッシブ仕様書

キャラクターに付与される状態効果（バフ/デバフ）の仕組みを定義する仕様書。
パッシブはステータス補正、戦闘コールバック、ライフサイクル管理を統合したシステムである。

関連仕様書:
- `doc/スキル/スキル計算仕様書.md` — ダメージ計算、命中/回避、ケレン行動パーセント（§8.5）
- `doc/総合戦闘仕様書.md` — 戦闘フロー、パワーレベル、前のめり

---

## 1. 概要

### 1.1 用語定義

| 用語 | 意味 |
|---|---|
| **パッシブ** | キャラクターに付与される状態効果。良い効果（バフ）と悪い効果（デバフ）がある |
| **PassivePower** | 重ね掛け回数。同一パッシブを再付与すると+1される |
| **テンプレート** | `PassiveManager`が保持するマスターデータ。付与時にDeepCopyされる |
| **バッファ** | 戦闘中の付与予約リスト。次ターン開始時に一括適用される |
| **LinkedPassive** | パッシブ間のリンク関係（一緒に消える、ダメージ連動など） |
| **VitalLayer** | パッシブにバインドされた追加HP層 |
| **SkillPassive** | スキルに付与される別種のパッシブ（§7参照） |

### 1.2 システム構成

```
PassiveManager (シングルトン / MonoBehaviour)
  └─ _masterList: 全パッシブテンプレート（SerializeReference）
       └─ GetAtID(id) → テンプレート取得

BasePassive (基底クラス)
  ├─ フィールド: 適合条件、ライフサイクル、ステ補正、キャラ限定補正、リンク等
  ├─ virtualメソッド: コールバック + 補正値返却
  └─ 具体パッシブクラスが継承・override

BaseStates
  ├─ PassiveManager部: 付与 / 除去 / バッファ管理
  └─ HasPassives部: コールバック転送 + 補正値集約
```

> 実装: `PassiveManager.cs`, `BasePassive.cs`, `BaseStates.PassiveManager.cs`, `BaseStates.HasPassives.cs`

## 2. パッシブの属性

### 2.1 基本属性

| フィールド | 型 | 意味 |
|---|---|---|
| `ID` | `int` | 一意の識別子 |
| `PassiveName` | `string` | パッシブ名 |
| `SmallPassiveName` | `string` | 省略名（UI表示用） |
| `OkType` | `CharacterType` | 適合するキャラクター種別 |
| `OkImpression` | `SpiritualProperty` | 適合する精神属性 |
| `IsBad` | `bool` | 悪いパッシブか。除去スキルの判定で使用 |
| `IsCantACT` | `bool` | 所持中は行動不可か（1つでもtrueがあれば行動不能） |
| `CanCancel` | `bool` | キャンセル可能か（行動選択画面でこのパッシブを解除できる） |
| `IsEternal` | `bool` | 永続パッシブか（ライフサイクル判定をスキップ） |
| `MaxPassivePower` | `int` | PassivePowerの上限 |

**IsCantACTとCanCancelの連携ルール:**

- `IsCantACT = true` のパッシブが1つでもあれば、キャラクターは通常のスキル行動ができない
- ただし `CanCancel = true` のパッシブが同時に存在する場合、**パッシブキャンセル専用の行動画面**に遷移する。スキル選択はできず、キャンセル可能なパッシブの解除のみが行える
- パッシブによる行動不能は、スキルによるフリーズ行動不能よりもヒエラルキーが低い

> 実装: `BasePassive.cs:132-169`

### 2.2 ライフサイクル属性

パッシブの自動消滅条件。複数を同時に設定可能。

| フィールド | 型 | 消滅条件 |
|---|---|---|
| `DurationTurn` | `int` | 戦闘ターン経過（-1=ターンで消えない） |
| `DurationWalk` | `int` | 歩数経過（-1=戦闘終了時消滅、0=歩行開始時即消滅） |
| `RemoveOnDeath` | `bool` | 所持者が死亡した時 |
| `RemoveOnDamage` | `bool` | 被ダメージ時 |
| `RemoveOnAttack` | `bool` | 攻撃行動時 |
| `RemoveOnAfterAttack` | `bool` | 攻撃後 |
| `RemoveOnAfterAttackToTarget` | `bool` | キャラ単位への攻撃後 |
| `RemoveOnAfterPerfectSingleAttack` | `bool` | 完全単体選択攻撃の直後 |
| `RemoveOnInterruptCounter` | `bool` | 割り込みカウンター発生時 |
| `RemoveOnNotVanguard` | `bool` | 前のめりでなくなった時 |
| `RemoveOnAfterAlliesDamage` | `bool` | 味方が被ダメージ後 |

ターンと歩数はそれぞれカウンター（`DurationTurnCounter` / `DurationWalkCounter`）で管理される。重ね掛け時に `DurationRefill()` でカウンターがリセットされる。

**DurationWalkの詳細:**

| 値 | 動作 |
|---|---|
| -1 | 戦闘終了時に消滅（`OnBattleEnd`） |
| 0 | 戦闘終了後、歩行した瞬間に効果なしで消滅 |
| 1以上 | 歩行効果が発生し、カウント消費後に消滅 |

歩行効果（`WalkEffect()`）は `DurationWalkCounter > 0` のときのみ実行され、生存判定はその後に行われる。

**ターン経過とVitalLayerの優先順位:** 戦闘ターン経過による自動消滅は、VitalLayerの生存条件よりも**優先される**。生存条件のVitalLayerが残っていても、ターンが0になればパッシブは消滅する。

**戦闘終了時の処理:** 戦闘終了時、全パッシブの `DurationTurnCounter` は **-1に強制リセット**される。戦闘後の残存はDurationWalk（歩行ターン）のみで決まる。

> 実装: `BasePassive.cs:171-236`

### 2.3 VitalLayer統合

パッシブは追加HP層（VitalLayer）をバインドできる。

```
PassiveVitalLayerBinding:
  VitalLayerId       — 追加HP層のID
  GrantTiming        — 付与タイミング（OnApply or OnRemove）
  RemoveOnPassiveRemove — パッシブ消滅時にこの層も消すか
  IsSurvivalCondition   — この層がパッシブの生存条件か
```

`IsSurvivalCondition = true` の場合、その層が全て消滅するとパッシブも自動消滅する。

> 実装: `BasePassive.cs:246, 522-560`

### 2.4 LinkedPassive

パッシブ間のリンク関係を定義する。

| フィールド | 意味 |
|---|---|
| `Passive` | リンク先のパッシブインスタンス |
| `RemoveWithMe` | 自身が消えた時にリンク先も消すか |
| `DamageLinkRatio` | ダメージ連動率（0より大きい場合、被ダメの一部をリンク先の所持者に転送） |
| `LayerDamage` | ダメージ転送時にVitalLayerを通すか |

> 実装: `BasePassive.cs:90-112`

### 2.5 その他の属性

| フィールド | 型 | 意味 |
|---|---|---|
| `ExtraPassivesIdOnSkillACT` | `List<ExtraPassiveBinding>` | スキル実行時に追加付与するパッシブIDリスト（味方/敵/両方を指定可） |
| `BlockVanguardByAlly_IfImVanguard` | `bool` | 自分が前のめり時に味方の前のめり交代を阻止するか |
| `_owner` | `BaseStates` | パッシブの所持者 |
| `_grantor` | `BaseStates` | パッシブの付与者 |
| `_maxRecoveryTurnModifier` | `int` | リカバリーターン補正値。正=行動を遅らせる、負=行動を早める。全パッシブの値を加算して適用 |

> 実装: `BasePassive.cs:241-277, 1071`

## 3. 付与・除去・重ね掛け

### 3.1 付与フロー

```
ApplyPassiveByID(id, grantor)
  │
  ├─ PassiveManager.GetAtID(id) → テンプレート取得
  ├─ DeepCopy() → インスタンス生成（共有防止）
  ├─ CanApplyPassive() → OkType, OkImpression の適合チェック
  │    └─ 不適合なら付与中止
  │
  └─ ApplyPassive(passive, grantor)
       ├─ 既に同IDを所持？
       │    ├─ YES → 重ね掛け（§3.3）
       │    └─ NO  → _passiveList.Add(passive)
       │              passive.OnApply(user, grantor) を呼び出し
       │              └─ DurationRefill()
       │              └─ VitalLayer付与（GrantTiming == OnApply）
       └─ 完了
```

> 実装: `BaseStates.PassiveManager.cs:114-184`

### 3.2 戦闘中のバッファ付与

戦闘中の付与は即時適用ではなく**バッファに予約**される。次ターン開始時に一括適用される。

```
ApplyPassiveBufferInBattleByID(id, grantor)
  └─ DeepCopy()してBufferApplyingPassiveListに追加

[次ターン]
ApplyBufferApplyingPassive()
  └─ バッファ内の全パッシブを ApplyPassive() で適用
  └─ バッファをクリア
```

これにより、同ターン内でのパッシブ付与が即座に他の処理に影響することを防ぐ。

**注意: 実質ターン数は設定値-1。** バッファ内のパッシブは付与されたターンの `NextTurn` で適用される。その直後に `UpdateTurnSurvival` が走り1ターン消費されるため、**実効的なターン数は `DurationTurn - 1`** となる。

> 実装: `BaseStates.PassiveManager.cs:76-107`

### 3.3 重ね掛け

同一IDのパッシブを再付与した場合：

1. 既存パッシブの `PassivePower` を保存
2. 既存パッシブを `RemovePassive()` で除去
3. 新しいパッシブを `ApplyPassive()` で付与（再帰呼び出し、ループしない）
4. 保存した `PassivePower` を復元し、+1 加算
5. `DurationRefill()` でターン数・歩数をリセット

`MaxPassivePower` を超えた場合はクランプされる。

> 実装: `BasePassive.cs:300-306`, `BaseStates.PassiveManager.cs:161-183`

### 3.4 除去フロー

```
RemovePassive(passive)
  ├─ _passiveList.Remove(passive)
  └─ passive.OnRemove(user)
       ├─ VitalLayer処理
       │    ├─ GrantTiming == OnRemove → 層を付与
       │    └─ RemoveOnPassiveRemove → 層を除去
       ├─ LinkedPassive処理
       │    └─ RemoveWithMe == true → リンク先も除去
       └─ _owner = null
```

> 実装: `BaseStates.PassiveManager.cs:287-306`, `BasePassive.cs:472-516`

### 3.5 除去スキルの判定

スキルの `SkillType.RemovePassive` によるパッシブ除去は、対象の `IsBad` フラグで判定方式が分かれる。

| IsBad | 判定 |
|---|---|
| `true`（悪いパッシブ） | `SkillHitCalc` のみで解除される |
| `false`（良いパッシブ） | `IsReactHIT` の命中判定を通す（良いパッシブを無理やり外す攻撃的行為） |

> 実装: `BaseStates.ReactionSkill.cs:292, 369`

### 3.6 適合判定と命中判定の独立性

パッシブの適合判定（`OkType`, `OkImpression`）と、付与スキルの命中判定は**独立した処理**である。スキルがHitしても適合しなければパッシブは付与されず、逆に適合しなかったことで命中結果が変わることはない。

プレイヤーはスキルが命中したのにパッシブが付かなかった場合、適合条件を満たさなかったことを推測できる。

### 3.7 派生クラス実装時の注意

`OnApply` をoverrideする際、**base.OnApply は派生処理の後ろに配置すること**。`base.OnApply` 内で `DurationRefill()` が実行されるため、ターン数や歩数の設定値操作は `base.OnApply` の前に行わないとリセットされてしまう。

## 4. ステータス補正

パッシブが四大ステータス（ATK/DEF/EYE/AGI）やその他の値に与える補正の一覧。

### 4.1 四大ステ: 固定値補正

全パッシブの値を**合算**して適用。

| メソッド | デフォルト | 意味 |
|---|---|---|
| `ATKFixedValueEffect()` | `_atkFixedValue` | ATK加算値 |
| `DEFFixedValueEffect()` | `_defFixedValue` | DEF加算値 |
| `EYEFixedValueEffect()` | `_eyeFixedValue` | EYE加算値 |
| `AGIFixedValueEffect()` | `_agiFixedValue` | AGI加算値 |

> 実装: `BasePassive.cs:754-810`

### 4.2 四大ステ: 倍率補正（ブレンド方式）

全パッシブの倍率を**ブレンド方式**で合成する。デフォルト値は `1.0`（補正なし）。

```
blend = product^0.26 × average^0.74
```

- `product`: 全倍率の積
- `average`: 全倍率の算術平均
- 積寄り26%、平均寄り74%で融合

これにより、単純な積だと極端に小さくなる問題と、単純な平均だと複数パッシブの影響が弱まる問題の両方を緩和する。

| メソッド | デフォルト | 意味 |
|---|---|---|
| `ATKPercentageModifier()` | `1.0` | ATK倍率 |
| `DEFPercentageModifier()` | `1.0` | DEF倍率 |
| `EYEPercentageModifier()` | `1.0` | EYE倍率 |
| `AGIPercentageModifier()` | `1.0` | AGI倍率 |

> 実装: `BasePassive.cs:812-864`, `BaseStates.HasPassives.cs:316-328`

### 4.3 攻撃者限定補正

特定の攻撃者に対してのみ発動する補正。同じブレンド方式で合成される。

| 補正 | メソッド | 意味 |
|---|---|---|
| DEF倍率 | `CheckDefencePercentageModifierByAttacker(attacker)` | 特定攻撃者への防御力補正 |
| 回避倍率 | `CheckEvasionPercentageModifierByAttacker(attacker)` | 特定攻撃者への回避補正 |
| HP割合ダメージ制限 | `CheckDontDamageHpMinRatioNormalOrByAttacker(attacker)` | 特定攻撃者からのHP下限保証 |

これらは `CharacterSpecificModifier` リストに `(TargetCharacter, Modifier)` ペアで保存される。`GrowEvasionPercentageModifierByAttacker()` 等で戦闘中に成長させることも可能。

> 実装: `BasePassive.cs:872-1040`

### 4.4 その他の補正

| 補正 | 集約方法 | 詳細 |
|---|---|---|
| **ダメージ減衰率** | 平均（-1除外） | `DamageReductionRateOnDamage`。0〜1の値。-1は計算対象外 |
| **回復率** | 下位65%の平均 | `HealEffectRate()`。複数の悪い補正が全体を支配するのを緩和 |
| **スキル発動率** | 平均 | `SkillActivationRate()`。デフォルト100%。全て100%なら計算省略 |
| **ターゲット確率** | FE式（S字カーブ） | `TargetProbability`。-100〜100。正規化後にFE式で実効化 |
| **ケレン行動%** | デフォ以外からランダム1つ | 攻撃時/防御時で独立。詳細は `スキル計算仕様書.md §8.5` を参照 |
| **HP割合ダメージ制限** | 最大値採用 | `NormalDontDamageHpMinRatio`。MaxHPの割合でHPをクランプ。レイザーダメージは貫通 |

#### FE式（ターゲット確率の実効化）

```
norm = |avgRate| / 100     （0〜1に正規化）
if norm ≤ 0.76:
  eff = 2 × norm²
else:
  eff = 1 - 2 × (1 - norm)²
→ 符号を戻して ±100 にスケール
```

中間値（0.76以下）では緩やかに変化し、極端な値でのみ急激に効果が出るS字カーブ。

> 実装: `BaseStates.HasPassives.cs:187-315`

## 5. コールバック

戦闘イベントに対応するvirtualメソッド。派生クラスでoverrideして固有の効果を実装する。

### 5.1 コールバック一覧

| コールバック | 引数 | タイミング | 呼び出し元 |
|---|---|---|---|
| `OnApply(user, grantor)` | 所持者, 付与者 | パッシブ付与時 | PassiveManager |
| `OnRemove(user)` | 所持者 | パッシブ除去時 | PassiveManager |
| `OnNextTurn()` | なし | 次ターン開始時 | HasPassives |
| `OnBeforeDamage(attacker)` | 攻撃者 | ダメージ計算直前 | HasPassives |
| `OnBeforeDamageActivate(attacker)` | 攻撃者 | ダメージ発動判定（falseで無効化） | HasPassives |
| `OnAfterDamage(attacker, damage)` | 攻撃者, ダメージ値 | ダメージ適用後 | HasPassives |
| `OnAfterAttack()` | なし | 攻撃後（全体） | HasPassives |
| `OnAfterAttackToTargetWitheHitresult(target, hitResult)` | 被攻撃者, 命中段階 | キャラ単位への攻撃後 | HasPassives |
| `OnAfterPerfectSingleAttack(target)` | 被攻撃者 | 完全単体選択攻撃の直後 | HasPassives |
| `OnAfterAlliesDamage(attacker)` | 攻撃者 | 味方が被ダメージ後 | HasPassives |
| `OnBeforeAllAlliesDamage(attacker, ref underActers)` | 攻撃者, 被害者リスト | 全体攻撃前（※発動是非の判定は未対応） | HasPassives |
| `OnInterruptCounter()` | なし | 割り込みカウンター発生時 | HasPassives |
| `WalkEffect()` | なし | 歩行時（DurationWalkCounter > 0のみ） | HasPassives |
| `BattleEffect()` | なし | 戦闘時効果 | HasPassives |
| `ACTEffect()` | なし | 行動時効果 | HasPassives |

### 5.2 コールバック実行時の安全策

全コールバックは `_passiveList.ToArray()` でコピーを取ってから反復する。これは、コールバック内でパッシブが除去される可能性があるため。

### 5.3 OnBeforeDamageActivate の特殊性

複数パッシブのうち**1つでも `false` を返すとスキルが発動しない**。他のコールバックは全て void で、効果の打ち消しはできない。

> 実装: `BaseStates.HasPassives.cs:22-159`

## 7. SkillPassive（スキルパッシブ）

### §7.1 概要と設計思想

パッシブ（Passive）とスキルパッシブ（SkillPassive）は別のシステムである。
スキルパッシブはキャラクターではなく**スキル自体に付く**パッシブであり、
TLOAの思い入れシステム（スキルへの愛着で威力が変動する仕組み）のために設計された。

そこから「スキル自体に付く不透明な影響」という性格が生まれ、
スキルロック（封印）や敵対的なデバフにも使える汎用システムに拡張された。
ゲーム内呼称は「冥医サイコロジー」— 暗くて見えない医学的・心理的な影響というニュアンス。

**設計上の核心: 不透明性。** スキルパッシブは簡単には制御・除去できない設計にすることで、
プレイヤーが効率的に操作しすぎることを防ぎ、精神的・心理的な「かかっているもの」としての存在感を持たせる。

| 項目 | Passive | SkillPassive |
|---|---|---|
| **保有者** | キャラクター（BaseStates） | スキル（BaseSkill） |
| **リスト** | `_passiveList` | `ReactiveSkillPassiveList` |
| **効果範囲** | 四大ステ補正、ダメージ、コールバック等 | SkillPower補正率 + スキルロック |
| **付与スキル種別** | `SkillType.addPassive` | `SkillType.addSkillPassive` |
| **重ね掛け** | PassivePower + 1 | なし |
| **ライフサイクル** | DurationTurn / DurationWalk / RemoveOn系 | 独自の `Update()` / `UpdateWalk()` |
| **管理** | PassiveManager（シングルトン）でテンプレ管理 | Manager不要。クラス直接生成で付与 |
| **永続** | IsEternalで永続可能 | 不可。永続的な変化はスキルレベルで行う |

### §7.2 プロパティ

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| `SkillPowerRate` | float | スキル威力への乗算率（デフォルト1.0） |
| `IsLock` | bool | trueでそのスキルが使用不可（封印） |
| `IsBad` | bool | 悪いスキルパッシブか |
| `DurationTurn` | int | 持続ターン（-1で消えない。0以下で自動除去） |
| `DurationWalkTurn` | int | 持続歩数（-1で消えない） |

SkillPowerRateの効果はSkillPower計算時に累積乗算される:

```
最終SkillPower = SkillPower × Π(1 + passive.SkillPowerRate)
```

> 実装: `BaseSkill.SkillPowe.cs` 内の `SkillPassiveSkillPowerRate()`

### §7.3 付与方式

スキルパッシブの付与時、対象スキルの選び方に3方式がある。
`SkillPassiveTargetSelection` enumで指定。

| 方式 | enum値 | 説明 | 主な用途 |
|------|--------|------|---------|
| **直接選択式** | `Select` | UIでプレイヤーが対象スキルを選ぶ | TLOAの思い入れ等の標準ロジック |
| **ランダム式** | `Random` | 対象のスキルからランダム抽選 | レアケース |
| **反応式** | `Reaction` | キャラ名+スキル名の組み合わせに自動反応 | 特定キャラの特定スキルへの因縁的な効果 |

**SkillFilter（区切り）による絞り込み:**
直接選択式・ランダム式では、`SkillFilter`で対象スキルを事前に絞り込める。
フィルタ条件: スキル印象、動作的雰囲気、精神属性、物理属性、十日能力、攻撃性質、特殊判別性質。

**付与個数:** `SkillPassiveEffectCount`で1回のスキルで何個のスキルに付与するか指定可能。

> 実装: `BaseSkill.SkillPassive.cs` の `SelectSkillPassiveAddTarget()`
> 戦闘中はバッファ経由: `ApplySkillPassiveBufferInBattle()` → 次ターン `ApplyBufferApplyingSkillPassive()`

### §7.4 除去方式

除去は意図的に制限されている。

| 方式 | 利用可否 | 理由 |
|------|---------|------|
| **反応式（個別指定）** | ✅ 唯一の除去手段 | キャラ名+スキル名で一致したスキルのスキルパッシブを**全て一括抹消** |
| 直接選択式 | ❌ | TLOAの思い入れを自由に消せてしまい、システムデメリットを打ち消す効率プレイになる |
| ランダム式 | ❌ | 同上 |
| 区切り（SkillFilter）式 | ❌ | 同じ区切りに属するTLOAの思い入れを巻き込んで消してしまう |

**除去時のisBad扱い:** スキルパッシブ個々のisBadでは場合分けしない。
除去スキル自体にisBadを指定し、スキル攻撃性質レベルで友好的/敵対的の命中計算を分ける。
一回の命中で対象スキルのスキルパッシブが**isBad関係なく全て消える**（浸透的な除去）。

> 実装: `BaseStates.ReactionSkill.cs` の `SkillPassiveRemove()` → `SkillRemoveSkillPassive()`（Clear）

### §7.5 関連SkillType

| SkillType | 用途 | 命中計算 |
|-----------|------|---------|
| `addSkillPassive`（友好的） | 味方のスキルにスキルパッシブ付与 | `SkillHitCalc`のみ |
| `addSkillPassive`（敵対的） | 敵のスキルに悪いスキルパッシブ付与 | `IsReactHIT` → 90%ランダム発動 |
| `removeBadSkillPassive`（友好的） | 味方の悪いスキルパッシブ除去 | `SkillHitCalc`のみ |
| `removeGoodSkillPassive`（敵対的） | 敵の良いスキルパッシブ除去 | `IsReactHIT` → 90%ランダム発動 |

## 8. 戦闘開始時のターンリセット

戦闘開始時、歩行ターンの消耗具合に応じて戦闘ターンカウンターを再調整する。

```
if DurationWalkCounter ≤ DurationWalk - 2:
  rate = DurationWalkCounter / DurationWalk
  DurationTurnCounter = DurationTurn × rate
```

歩行中にかなり消耗したパッシブは、戦闘中のターン数も短縮される。

> 実装: `BaseStates.HasPassives.cs:167-182`

## 9. 計算全体フロー図

パッシブが関与する戦闘中の計算フロー。

```
スキル実行開始
  │
  ├─ ApplyExtraPassivesToSkill() ← パッシブ由来の追加パッシブをスキルに注入
  │
  ├─ IsReactHIT() 命中判定
  │    ├─ CalcMinimumHitChancePer() ← ケレン行動%はパッシブ由来
  │    └─ EvasionRate() ← パッシブの回避補正（攻撃者限定含む）
  │
  ├─ [被害者側] PassivesOnBeforeDamage(attacker)
  ├─ [被害者側] PassivesOnBeforeDamageActivate(attacker)
  │    └─ false → 攻撃無効化（KageronCounter等）
  │
  ├─ DamageOnBattle()
  │    ├─ ATK/DEF計算 ← 固定値補正 + 倍率補正（ブレンド方式）
  │    ├─ DEF補正 ← 攻撃者限定DEF補正（ブレンド方式）
  │    ├─ パッシブ減衰 ← PassivesDamageReductionEffect()（平均）
  │    ├─ 命中段階適用（Critical 1.5倍 / Graze 2〜35%）
  │    └─ DontDamagePassiveEffect() ← HP割合制限（最大値採用）
  │
  ├─ [被害者側] PassivesOnAfterDamage(attacker, damage)
  │    └─ ダメージリンク処理（LinkedPassive）
  │
  ├─ [攻撃者側] PassivesOnAfterAttack()
  ├─ [攻撃者側] PassivesOnAfterAttackToTargetWithHitresult(target, hitResult)
  ├─ [攻撃者側] PassivesOnAfterPerfectSingleAttack(target)
  │
  └─ パッシブ自動消滅判定
       ├─ RemoveOnDamage / RemoveOnAttack / RemoveOnAfterAttack 等
       └─ VitalLayer生存条件チェック

[ターン終了時]
  ├─ PassivesOnNextTurn()
  ├─ UpdateTurnAllPassiveSurvival() ← ターン経過消滅
  ├─ ApplyBufferApplyingPassive() ← バッファ内パッシブを一括適用
  └─ UpdateNotVanguardAllPassiveSurvival()

[歩行時]
  ├─ AllPassiveWalkEffect()
  └─ UpdateWalkAllPassiveSurvival() ← 歩数消滅
```
