# セーブロード仕様書

本プロジェクトのセーブ/ロード機能の現状仕様を記す。

---

## 1. アーキテクチャ概要

セーブ/ロードは**Players（味方データ）** と **GameContext（ウォーク進捗）** を一つのJSONファイルに統合して保存する構造。

```
PlayersBootstrapper (エントリーポイント)
    │
    ├─ Save: PlayersRuntime.CreateSaveData()
    │            └─ PlayersSaveService.Build(roster, composition)
    │                   └─ WalkProgressData.FromContext(GameContextHub.Current)
    │                          └─ PlayersSaveIO.SaveDefault(data, fileName)
    │                                 └─ File.WriteAllText(JSON)
    │
    └─ Load: PlayersSaveIO.TryLoadDefault(out data, fileName)
                 └─ PlayersRuntime.ApplySaveData(data)
                        └─ PlayersSaveService.Apply(data, roster, registry, composition)
                               └─ WalkProgressData.ApplyToContext(gameContext)
```

### クラス責務

| クラス | 責務 |
|--------|------|
| `PlayersBootstrapper` | セーブ/ロードのエントリーポイント（UIやシーンから呼ばれる） |
| `PlayersRuntime` | Playersデータの作成・適用ロジック |
| `PlayersSaveService` | セーブデータの変換・復元処理 |
| `PlayersSaveIO` | JSONファイルI/O |
| `GameContext` | ウォーク進捗状態管理 |
| `GameContextHub` | GameContextのグローバルアクセスポイント |
| `WalkProgressData` | GameContextの永続化形式（セーブ用データクラス） |

---

## 2. セーブデータ形式・保存先

- **形式**: JSON（`JsonUtility`）
- **保存先**: `Application.persistentDataPath/save/{fileName}`
- **デフォルトファイル名**: `players_save.json`
- **複数ファイル対応**: `fileName`パラメータで技術的に対応可能だが、UIは未実装

## 3. セーブデータ構造

### 3.1 PlayersSaveData（トップレベル）

```
PlayersSaveData
├─ UnlockedCharacterIds: List<string>     // 解放済みキャラ一覧
├─ Allies: List<PlayersAllySaveData>      // 全キャラクターのデータ
├─ ActivePartyIds: List<string>           // 現在のパーティ編成
└─ WalkProgress: WalkProgressData         // ウォーク進捗
```

### 3.2 PlayersAllySaveData（キャラクターごと）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| CharacterId | string | キャラ識別子 |
| HP | float | 現在HP |
| MentalHP | float | 現在精神HP |
| P | int | パワー |
| NowPower | PowerLevel | パワーレベル |
| MyImpression | SpiritualProperty | 現在の精神属性 |
| DefaultImpression | SpiritualProperty | デフォルト精神属性 |
| NowResonanceValue | float | 共鳴値 |
| EmotionalAttachmentSkillID | int | 思い入れスキルID |
| EmotionalAttachmentSkillQuantity | float | 思い入れスキル数量 |
| ValidSkillIds | List\<int\> | 有効なスキルID一覧 |
| BaseTenDayValues | List\<TenDayValueSaveData\> | 十日サイクル能力値 |
| AttrPoints | AttrPointSaveState | 属性ポイント（配分マップ＋割当履歴） |
| Skills | List\<PlayerSkillSaveData\> | スキル習得状態（SkillId, RecordDoCount, Proficiency） |
| Passives | List\<PassiveSaveData\> | パッシブ（PassiveId, PassivePower, TurnCounter, WalkCounter） |
| VitalLayers | List\<VitalLayerSaveData\> | ヴァイタルレイヤー（VitalLayerId, LayerHP） |
| AdaptationMemories | List\<AdaptationMemorySaveData\> | スキル適応メモリ（SkillImpression, Value） |

### 3.3 WalkProgressData（ウォーク進捗）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| GlobalSteps | int | 通算歩数 |
| NodeSteps | int | ノード内歩数 |
| TrackProgress | int | トラック進捗 |
| CurrentNodeId | string | 現在ノードID |
| NodeSeed | uint | ノードシード値 |
| VarietyHistoryIndex | int | 多様性履歴インデックス |
| Flags | List\<FlagEntry\> | ゲーム内フラグ（Key-Value） |
| Counters | List\<CounterEntry\> | ゲーム内カウンター（Key-Value） |
| Tags | List\<string\> | ゲーム内タグ |
| EncounterStates | List\<EncounterStateData\> | エンカウント状態（TableId, IsInitialized, Cooldown, Grace, Misses, PityAccumulated） |
| GateStates | List\<GateRuntimeStateData\> | ゲート状態 |
| Anchors | List\<WalkAnchorData\> | アンカーポイント状態 |
| EncounterOverlays | List\<EncounterOverlayData\> | エンカウント修飾子（永続） |
| SideObjectState | SideObjectState | サイドオブジェクト状態 |
| CentralObjectState | CentralObjectState | セントラルオブジェクト状態 |
| EventEntryStates | List\<EventQueueEntryState\> | イベントキュー状態 |
| FriendshipCombos | List\<FriendshipComboSaveData\> | 友情コンビ登録状態（ComboId, MemberGuids, IsKusareEnPath, ReEncountCount） |
| EnemyStates | List\<EnemyPersistenceData\> | 敵個体状態（全敵対象。詳細は§3.4参照） |

## 4. セーブ/ロードトリガー

### 4.1 セーブ

現状、セーブは**明示的な呼び出しのみ**。自動セーブのトリガーはゲームランタイム上に存在しない。

```csharp
PlayersBootstrapper.SaveToJsonDefault(string fileName = "players_save.json")
```

### 4.2 ロード

ロードも**明示的な呼び出しのみ**。ゲーム開始時の自動ロードは未実装。

```csharp
PlayersBootstrapper.LoadFromJsonDefault(string fileName = "players_save.json")
```

### 4.3 エディタAutoSave（`Assets/Editor/AutoSave.cs`）

エディタ専用のシーン/プレハブ保存機能。ゲームデータのセーブとは無関係。

| 機能 | トリガー |
|------|---------|
| PlayMode開始時に保存 | `playModeStateChanged` |
| タイマー保存（60秒間隔） | `EditorApplication.update` |
| 手動保存時バックアップ | `AssetModificationProcessor` |
| 手動バックアップ | Menu: File/Backup/Backup |
| ロールバック | Menu: File/Backup/Rollback |

## 5. ロード時の復元フロー

### 5.1 Players復元

```
PlayersSaveService.Apply()
  ├─ 1. WalkProgress復元（GameContextへ）
  ├─ 2. Rosterクリア
  ├─ 3. UnlockedCharacterIds → Roster再構築（キャラインスタンス生成）
  ├─ 4. ActivePartyIds → パーティ編成復元
  └─ 5. 各キャラクターのステータス復元
       ├─ HP / MentalHP / P / Impression
       ├─ 属性ポイント復元
       ├─ スキル習得状態復元
       ├─ パッシブ再適用（RemovePassive → ApplyPassive）
       ├─ ヴァイタルレイヤー再適用
       └─ 適応メモリ復元
```

### 5.2 GameContext復元

```
WalkProgressData.ApplyToContext(gameContext)
  ├─ カウンター設定（GlobalSteps, NodeSteps, TrackProgress）
  ├─ CurrentNodeId / NodeSeed / VarietyHistoryIndex
  ├─ Flags / Counters / Tags
  ├─ GateResolver スナップショット復元
  ├─ AnchorManager データ復元
  ├─ EncounterStates / EncounterOverlays
  ├─ SideObjectState / CentralObjectState
  ├─ EventEntryStateManager
  ├─ ClearEnemyCache()（敵キャッシュ + 復元用ストレージをクリア。二重ロード対策）
  ├─ EnemyRebornManager.ClearAll()（旧セッションのReborn情報をクリア）
  ├─ GameContext.ImportEnemyStates()（全敵の状態を復元用ストレージに取り込み）
  ├─ FriendshipComboRegistry（コンボ登録復元）
  └─ GetRuntimeEnemies時に敵個体状態を適用
     ├─ DeepCopy → RestoreGuid → RestoreState
     ├─ HP / MentalHP / broken / LastEncounterProgress / 武器適応率 / BondDeltas
     └─ RebornManager.RestoreRebornState（復活カウンター復元）
```

### 5.3 セーブ時のエクスポートフロー

```
GameContext.ExportAllEnemyStates()
  ├─ encounterEnemies（訪問済み）からランタイム最新状態をエクスポート
  ├─ _savedEnemyStates（未訪問）から前回セーブの状態をマージ
  └─ 訪問済みが優先。未訪問分はセーブデータをそのまま引き継ぎ
```

**設計意図**: 遅延実体化パターンのため、ロード後に全エンカウントを訪問しなくても
再セーブで未訪問敵の状態が消失しない。

### 5.4 ノード移動時の敵状態破棄

```
AreaController.TransitionTo()
  ├─ GameContext.ClearEnemyCache()（encounterEnemies + _savedEnemyStates をクリア）
  └─ EnemyRebornManager.ClearAll()
```

ノード移動時は敵キャッシュを全破棄。移動先ノードではテンプレートから新規生成される。

### 5.5 フォールバック挙動

- `UnlockedCharacterIds` が空の場合、`Allies` データからの復元を試みる（レガシー対応）
- `ActivePartyIds` が無効の場合、全キャラをパーティに設定
- Registryに未登録のキャラは復元スキップ（警告ログ出力）

## 6. 永続化対象の一覧

### 6.1 永続化される項目

**味方キャラクター**: HP, MentalHP, パワー, 精神属性, 共鳴値, 思い入れスキル, 有効スキル一覧, 十日能力値, 属性ポイント（配分+履歴）, スキル習得状態（実行回数, 習熟度）, パッシブ（レベル, ターンカウンタ, 歩数カウンタ）, ヴァイタルレイヤー, 適応メモリ

**パーティ**: 解放済みキャラID一覧, 現在のパーティ編成

**ウォーク進捗**: 歩数（通算/ノード内/トラック）, 現在ノード, ランダムシード, フラグ/カウンター/タグ, エンカウント状態, ゲート/アンカー状態, エンカウント修飾子, サイド/セントラルオブジェクト, イベントキュー状態

**敵個体（全敵対象）**: GUID, broken, HP, MentalHP, 最終エンカウント歩数, 武器適応率（3フィールド）, 復活カウンター（残り歩数, 最終更新歩数, 状態）, 相性値変動, 友情コンビ登録状態

### 6.2 永続化されない項目

| 項目 | 備考 |
|------|------|
| 敵パッシブ | 毎エンカウント時にテンプレート→歩数シミュレーション→コンボ蓄積で再導出。明示的保存不要 |
| BattleMemory | 設計意図あり（毎戦闘Clear） |
| NowBattleProtocol | 「どの規格を使っているか」は揮発。ロード時にApplyWeapon(InitWeaponID)で初期規格に戻る。AIの戦術的判断なので再選択すればよい。一方「規格への慣れ（適応率）」はAdaptationRate等で永続化される。適応率は特定の規格と紐づかず、ロード後は初期規格の排他ATKにそのまま適用される。AIがSwitchProtocolで別の規格に切り替えた時点で適応率はリセットされ新しいラグが始まる |

## 8. 現時点の制約

| 制約 | 詳細 |
|------|------|
| 自動セーブなし | ゲームランタイム上にAutoSaveトリガーが存在しない |
| セーブスロットUI未実装 | ファイル名パラメータは存在するがUI側がない |
| バージョン管理なし | セーブデータにバージョン番号がなく、互換性チェック不可 |
| タイムスタンプなし | セーブ日時・プレイ時間の記録がない |
| エラーリカバリなし | セーブ失敗・破損時の復旧処理がない |
