# 挑発属性仕様書

## 1. 概要

「挑発属性」はスキルの性質の一つ。挑発属性を持つスキルで攻撃すると、被攻撃者に「イラつき」を付与する。イラつきは対象の行動を歪め、冷静さを失わせる。

```
A（挑発者）──挑発属性スキルで攻撃──→ B（被挑発者）
                                       │
                                       ▼
                                 B に「イラつき→A」が蓄積
```

### 付与の2経路

| 経路 | 仕組み | 特徴 |
|------|--------|------|
| **直接付与（スキル属性）** | 挑発属性を持つスキルが対象にヒット（CompleteEvade以外）でイラつき加算 | 全SkillTypeで機能（攻撃/回復/パッシブ付与等） |
| **持続付与（パッシブ経由）** | 挑発パッシブを付与 → パッシブの `OnNextTurn()` で毎ターンイラつき加算 | じわじわ溜まる持続型 |

直接付与は全SkillType共通の `HitResult` を使うため、スキル種別によるパイプラインの違いを意識する必要はない。CompleteEvade以外なら一律で挑発が付く。

## 2. イラつきの基本仕様

### 2.1 蓄積構造

- イラつきは **「誰に対してどれだけイラついているか」の整数カウンター**
- 1キャラが複数の相手に対するイラつきを同時に持てる
- 付与量はスキルごとに設定可能（`ProvokeAmountOnHit`: 整数値）

```
例: Bの状態（スキルAの付与量=2、スキルCの付与量=1の場合）
  イラつき→A: 4
  イラつき→C: 1
  合計イラつき: 5
```

### 2.2 付与制限

| 条件 | 結果 |
|------|------|
| 自分自身への挑発 | 付与されない |
| 対象が機械種別（`CharacterType.Machine`） | 付与されない |
| 精神属性による補正 | 付与量が精神属性ごとに乗算補正される（セクション5.2） |
| 十日能力の強さ比較 | 挑発者が被挑発者より3倍以上弱い場合、付与量が低下する（セクション5.4） |
| パッシブ耐性 | 付与量軽減率パッシブにより乗算軽減される（セクション5.5） |

#### 最終付与量の計算式

```
最終付与量 = スキル付与量 × 精神属性補正(5.2) × 強さ補正(5.4) × パッシブ付与量軽減(5.5)
           → Mathf.Max(1, Mathf.RoundToInt(...))
```

`Mathf.Max(1, ...)` により完全に0にはならない。どれだけ弱い相手でも最低1は付与される。

### 2.3 減衰

- **3ターンごとに判定**。判定時にイラつきカウントが減少する
- 減衰量は精神属性ごとに異なる（セクション5.1）。デフォルト: 1。パッシブで加算可能（セクション5.5）
- 精神属性の減衰量が0の場合はパッシブでも覆せない（不動の固有性質）
- 0以下になったイラつきは消滅する
- 判定タイミングは3ターンごとで固定（精神属性が変わっても変わらない）

## 3. イラつきの効果

### 3.1 イラつき攻撃

イラつきを持つキャラのターン開始時に**確率判定**が行われる。この判定は `TargetProbability` とは独立した専用ロジック。

#### パターン1: 確率ヒット → 強制攻撃

```
ターン開始 → イラつき確率判定 → ヒット
  → ターゲットがイラつき最大の相手に強制固定
  → スキルは攻撃系のみに制限
  → 強制前のめり + 命中低下免除なし
```

プレイヤーも敵も統一。殴る相手は選べないが、何で殴るかは選べる（攻撃系限定）。

#### パターン2: 確率ミス → 自発的なイラつき攻撃

```
ターン開始 → イラつき確率判定 → ミス → 通常コマンド選択
  → 攻撃系スキルでイラつき対象を攻撃先に選んだ場合
    → イラつき攻撃として処理（強制前のめり + 命中低下免除なし）
  → 非攻撃系スキルでイラつき対象を選んだ場合
    → 通常処理（イラつき攻撃にはならない）
```

#### まとめ

| 状況 | 対象選択 | スキル選択 | 前のめり | 命中低下免除 |
|------|---------|----------|---------|---------|
| 確率ヒット | 最大対象に強制 | 攻撃系のみ | 強制 | なし |
| 確率ミス + 攻撃系でイラつき対象を選択 | 自分で選択 | 攻撃系（条件） | 強制 | なし |
| 確率ミス + 非攻撃系でイラつき対象を選択 | 自分で選択 | 制限なし | 通常 | 通常 |
| 確率ミス + イラつき対象以外を選択 | 自分で選択 | 制限なし | 通常 | 通常 |

#### 範囲スキルとの関係

イラつき攻撃パターン1（確率ヒット）では、スキルの `SkillZoneTrait`（範囲性質）に関係なくターゲットが単体に強制される（`DirectedWill.One` に上書き）。全体攻撃スキルや範囲攻撃スキルを選んでも、実行時にはイラつき最大の相手1人だけを攻撃する。

- スキル選択のUI制限は `SkillType.Attack` のみ。`SkillZoneTrait` は制限しない
- 範囲スキルが単体に絞られた場合、分散割合（PowerSpread）は1人目の値（通常1.0）が適用されるため、威力の損失はない
- 「全体攻撃を選んだのに1人に飛ぶ」のはイラつきによる制御不能の演出として意図的な仕様

#### 命中低下免除について

通常の前のめりでは後衛ペナルティ（相手のAGI×0.2の命中低下）が免除される。しかしイラつき攻撃の前のめりは怒りで雑に突っ込んでいるため、この免除が適用されない。

リカバリーターン2倍速は物理的位置関係の効果なので、イラつき攻撃でもキャンセルされない。

#### 精神属性による例外

| 精神属性 | 例外 |
|---------|------|
| Psycho | イラつき攻撃時でも命中低下免除が通常通り適用される |

#### 発動確率

```
発動値 = 最大相手のカウント + (合計 - 最大相手のカウント) / 4
発動確率 = 発動値 × Demeanor別係数%（セクション5.3） × パッシブ発動確率軽減（セクション5.5）
```

| 最大相手 | 残り | 発動値 | 確率(6%) |
|---------|------|--------|------|
| 1 | 0 | 1.0 | 6% |
| 3 | 0 | 3.0 | 18% |
| 5 | 2 | 5.5 | 33% |
| 8 | 0 | 8.0 | 48% |

やり場のないイラつきは発動確率計算には含めない（対象不在のため）。

### 3.2 暴走（イライラ状態）

イラつきの**合計カウント**が暴走閾値以上になると暴走する。

- **暴走閾値**: キャラごとに設定可能（整数値、デフォルト: **8**）
- 合計には「やり場のないイラつき」も含まれる

| 効果 | 詳細 |
|------|------|
| **火力上昇** | 全攻撃のATKに倍率補正 |
| **全攻撃で強制前のめり** | イラつき対象以外を狙う場合も前のめりになる |

#### 火力倍率テーブル

| 超過量 | 倍率 | 例（閾値8） |
|--------|------|-----------|
| 0（ちょうど閾値） | 1.25倍 | 合計8 |
| 1〜2 | 1.40倍 | 合計9〜10 |
| 3〜4 | 1.55倍 | 合計11〜12 |
| 5以上 | 1.75倍 | 合計13以上 |

※ 仮値。プレイ感で調整する。

#### 暴走とイラつき攻撃の差別化

| | イラつき攻撃（3.1） | 暴走（3.2） |
|--|--|--|
| 前のめり | 強制 | 強制 |
| 命中低下免除なし | あり | なし（通常通り免除） |
| 火力上昇 | なし | あり |
| ターゲット強制 | 最大対象に固定 | なし（自由） |

暴走は「強くなったが前に出ざるを得ない」リスクリワード。イラつき攻撃の「制御不能感」とは異なる。

#### 設計意図: 双方向リスク

- **挑発された側**: 行動を強制される不自由さ + 暴走で被弾リスク増加
- **挑発した側**: 暴走で相手の火力が上昇 → 追い込むほど自分が危険
- 挑発は一方的な有利を生まず、加減するか一気に畳みかけるかの判断が生まれる

### 3.3 イラつき対象の死亡時処理

#### 基本: 対象が死んだらそのイラつきは消える

殺したのが誰かに関係なく、対象の死亡でそのカウンターは消える。

```
B: イラつき→A: 5, イラつき→C: 2
Aが死亡 → イラつき→A: 消滅
B: イラつき→C: 2
```

#### やり場のないイラつきへの変換

以下を**全て**満たす場合、「やり場のないイラつき」に変換される:

1. その対象への個別イラつき値が **暴走閾値以上**（対象単体の値で判定、合計ではない）
2. **自身の攻撃・パッシブで倒したのではない**（他の味方が倒した等）

```
例: 暴走閾値 = 8
B: イラつき→A: 9

ケース①: 味方DがAを倒した
  → Aへのイラつき9 ≧ 閾値8、かつB自身が倒してない
  → やり場のないイラつき→(A): 9 に変換

ケース②: B自身の攻撃でAを倒した
  → B自身が倒した → 変換なし、すっきり消滅

ケース③: Bのパッシブ効果でAが死亡した
  → B自身のパッシブで倒した → 変換なし、すっきり消滅

ケース④: Aへのイラつきが6（閾値8未満）で味方DがAを倒した
  → 閾値未満 → 変換なし、普通に消滅
```

#### やり場のないイラつきの性質

| 性質 | 詳細 |
|------|------|
| 暴走の合計値に加算 | 通常イラつきと合算して暴走判定に影響 |
| イラつき攻撃は発生しない | 対象不在のため発動確率に含めない |
| 対象情報を保持 | 復活等での再登場に備える |
| 通常の減衰は適用されない | 対象不在のため3ターン減衰対象外 |

#### やり場のないイラつきの解消条件

**条件1: 元の対象に攻撃でダメージを与える**
- 復活等で再登場した対象に、自身の攻撃（パッシブは対象外）で最大HPの3%以上のダメージを**一度に**与えると全消滅

**条件2: 同じ対象から再度挑発される**
- やり場のないイラつきが通常カウンターに即座に全加算されて消滅

```
例: B: やり場のないイラつき→(A): 6
Aが復活して挑発属性スキル（付与量2）でBを攻撃 →
  通常付与: 2 + やり場のない分: 6 = イラつき→A: 8
  やり場のないイラつき→(A): 消滅
```

## 4. 関連システムとの境界

### 挑発者（A）側の効果

不要。Bの「イラつき→A」が攻撃偏向を生むことで、Aが狙われやすくなる効果は自然に実現される。

### 機械系の特殊スキル

機械はイラつきが付かない代わりに、スキル効果として強制ターゲット吸引を持つ。挑発属性とは独立した仕組み。

### TargetProbabilityとの関係

`TargetProbability` は「自分が狙われる確率」の制御であり、「特定の相手を狙いに行く」制御には不向き。イラつきのターゲット偏向は専用ロジックで独立して動作する。

### 賢い敵AI

確率判定ミスの場合、思慮レベルの高い敵AIは「自分がイラついている」ことを認識し、あえてイラつき対象を避ける戦術も取れる。ただし確率ヒット時は思慮レベルに関係なく強制。

### 5.3 Demeanor別 発動確率係数テーブル

基本式の6%係数をDemeanorごとに差し替える。`最終確率 = 発動値 × 係数%`。

| 人間状況 | 係数 | メモ |
|---------|------|------|
| Painful | 2% | 辛いから反応しづらい |
| Optimistic | 6% | イラつきそうでありながら余裕で流せる。プラマイゼロ |
| Elated | 3% | 高揚してるから見えにくい |
| Resolved | 6% | 覚悟と挑発に乗るのは別レイヤ |
| Angry | 6% | 怒りで見えにくい反面イラつきやすい。プラマイゼロ |
| Doubtful | 7% | 疑念をいだいてると挑発に乗りやすい |
| Confused | 6% | 混乱してるから別レイヤ |
| Normal | 6% | |

### 5.4 十日能力による強さ補正

挑発者と被挑発者の十日能力総量を比較し、被挑発者から見て3倍以上弱い相手からの挑発は付与量が低下する。「こんな雑魚にイラつかない」というフレーバー。

ゆりかごシステムの `TendayAdvantage`（`doc/ゆりかごシステム仕様書.md` セクション①）と同じ十日能力総量比較パターンを踏襲。

```
strengthRatio = source.TenDayValuesSumForSkill() / target.TenDayValuesSumBase()

ratio > 0.33  → 補正 1.0（同格〜3倍弱い程度、普通にイラつく）
ratio <= 0.33 → 補正 = ratio / 0.33（3倍以上弱いと線形に低下）
```

| 攻撃者の強さ（対被害者比） | ratio | 補正 |
|---|---|---|
| 同格以上 | 1.0+ | 1.0 |
| 半分 | 0.5 | 1.0 |
| 3分の1（境界） | 0.33 | 1.0 |
| 6分の1 | 0.166 | 約0.5 |
| 10分の1 | 0.1 | 約0.3 |
| 雑魚 | 0.01 | 約0.03 |

ゼロ除算防止: 被挑発者の十日能力総量が0以下の場合、補正は1.0。

### 5.5 パッシブ耐性プロパティ

BasePassiveの `[SerializeField]` フィールドとして3つの耐性プロパティを持つ。パッシブを通じて挑発への戦術的な備えを組み込める。

**設計判断: キャラ固有ステータスには耐性を持たせない。** 機械免疫（`CharacterType.Machine`）が完全免疫の唯一の手段であり、有機キャラはイラつきから完全に逃れることはできない。パッシブはパッシブ枠を消費するコストと引き換えに軽減を提供する「補完要素」。

| プロパティ | 型 | デフォルト | 集約 | 効果 |
|-----------|-----|----------|------|------|
| `IrritationApplicationResistRate` | `float` | 1.0 | 乗算（Π） | 付与量に乗算。0.5で半減 |
| `IrritationDecayBonus` | `int` | 0 | 合算（Σ） | 減衰量に加算。精神属性の減衰0時は不動（適用されない） |
| `IrritationTriggerResistRate` | `float` | 1.0 | 乗算（Π） | 発動確率に乗算。0.5で確率半減 |

## 6. 表示

イラつき情報は3つの経路で伝達する。

### 6.1 エフェクト表示（敵味方共通）

バトルアイコン上にKFXループエフェクトを再生。**重ね掛け方式**（暴走はイラつきの上に重なる）。

| ファイル | 状態 | 演出 |
|---------|------|------|
| `irritation_aura.json` | イラつき存在 | 控えめな赤い脈動オーラ + 少数の上昇パーティクル |
| `rage_aura.json` | 暴走中 | 深紅のグロウ + 拡散リング + 攻撃的パーティクル |

| タイミング | 処理 |
|-----------|------|
| イラつき初発生 | `irritation_aura` ループ開始 |
| 暴走突入 | `rage_aura` ループ開始（irritation_aura継続） |
| 暴走解除（イラつき残存） | `rage_aura` 停止 |
| イラつき全消滅 | 両方停止 |

更新タイミング: イラつき加算・減衰・死亡消滅の各処理後。

### 6.2 Kズーム表示（敵味方共通）

アイコンクリック時のKズーム内にイラつきトークンを表示。パッシブ表示とは別の専用領域。

| トークン | 条件 |
|---------|------|
| `<イラつき→(キャラ名)>` | イラつき最大の対象 |
| `<イライラ>` | 暴走中 |

やり場のないイラつき・減衰情報はKズームでは省略。

### 6.3 Characonfig表示（味方専用）

StatesBannerの専用ページにイラつき全情報を表示。

| 要素 | 表示例 |
|------|--------|
| 通常イラつき（対象別） | `<敵A→5>  敵B→2`（最大対象を`<>`で強調） |
| やり場のないイラつき | `やり場なし: 敵C→4` |
| 暴走関連 | `イライラ状態! 合計11/8 (超過3 → 1.55倍)` |
| 減衰情報 | `減衰: あと1ターン (−2)` |
| 減衰情報（不動時） | `減衰なし（不動）`（精神属性の減衰量が0の場合） |
