# 思えの値（Resonance Value）仕様書

**ステータス: 値システムのみ実装済み / 思え鳥化は未実装**

精神的な正気の残量を表すゲージ。設計上は0になると**思え鳥化**（発狂）するが、**現在は値の計算・ダメージ・回復のみ実装されており、0到達時の思え鳥化トリガーや状態変化は未実装**。`NowResonanceValue` が0になっても0にクランプされるだけで何も起きない。

正体は精霊であり、**属性攻撃**で削られる。知能が低いほど最大値が大きく、馬鹿ほど発狂しにくい。
基本的に**めったに思え鳥にはならない**ことを前提とした設計。

## 1. 設計思想

思えの値の根底にあるのは**「馬鹿ほど狂わない」**という原則。

知能が低いキャラはゲージが巨大で、圧倒的な格上に殴られても正気を保てる。賢く経験豊富なキャラほどゲージが小さく、恐怖を理解してしまうが故に脆い。ただし精神属性の多様性（精神ポテンシャル）がダメージ軽減として機能するため、**多様な経験を積んだ賢者は知性の脆さを経験の厚みで補える**。

スキル枠に対するアンチシステムでもある。スキルの数を増やすほど思えの最大値が微減し、思え鳥のリスクが上がる。ただし精神属性の種類数が増えれば思えDEFも上がるため、「量だけ」ではなく「質（多様性）」を伴えばカバーできる設計。

## 2. 最大値の計算

> 実装: `BaseStates.StatesNumber.cs:1295-1307`

### 2.1 計算式

```
ResonanceValue = baseValue + Baka × 1.3

baseValue の算出:
  ① 十日能力総和 × 0.56           … カタログスペックの56%が基盤
  ② ①をスキル数で微小スケーリング  … 頭のほぐされ具合
  ③ ②を思慮係数でスケーリング      … キャラ固有の知能レベル
```

### 2.2 3段階のスケーリング

#### ① 基礎値（十日能力ベース）

```
baseValue = TenDayValuesSumBase() × 0.56
```

キャラの能力が高いほど深精神耐性がある。鍛錬は正気の基盤になる。

#### ② スキル数による微小スケーリング

> 実装: `BaseStates.StatesNumber.cs:1380-1384`

```
baseValue = baseValue × (1 - (skillCount - 1) × 0.04)
```

| スキル数 | 係数 | 意味 |
|---------|------|------|
| 1 | 1.00× | 変化なし |
| 3 | 0.92× | 8%減 |
| 6 | 0.80× | 20%減 |
| 10 | 0.64× | 36%減 |

スキルが多い＝頭が柔軟＝知覚が鋭い＝恐怖を感じやすい。「努力と開花した才能の指数」であり、生物としてのカタログ知能に対する**頭のほぐされ具合**。

#### ③ 思慮係数による知能スケーリング

> 実装: `BaseStates.StatesNumber.cs:1390-1394`

```
scale = Lerp(scaleMax, 1.0, _thinkingFactor / 100.0)
baseValue = baseValue × scale
```

| 思慮係数 | scale | 効果 |
|---------|-------|------|
| 1（最低・馬鹿）| ≈ 11.0× | 最大値が11倍に膨張 → 思え鳥になりにくい |
| 50（中間）| ≈ 6.0× | 中程度の膨張 |
| 100（最高・賢い）| 1.0× | 変化なし → 思え鳥になりやすい |

キャラごとにインスペクタで `_thinkingFactor`（1〜100）を手動設定する。知能が低いキャラは特殊な超スケーリングアップにより馬鹿でかいゲージを持つ。`scaleMax` は現在 `11f` で呼び出されている。

#### ④ 馬鹿能力の直接加算

```
+ TenDayValuesBase().GetValueOrZero(TenDayAbility.Baka) × 1.3
```

十日能力「馬鹿」の値を1.3倍で最大値に直接加算。スケーリングとは独立した鈍感さのボーナス。

### 2.3 想定レンジ

> 参照: `doc/十日能力・四大ステ成長レンジ想定.md`

思慮係数・馬鹿能力・スキル数を除いた基礎部分（`十日能力合計 × 0.56`）のみの目安：

| フェーズ | 十日能力合計 | 基礎思えの値 |
|---------|------------|------------|
| 序盤 | 40〜120 | 22〜67 |
| 中盤 | 150〜300 | 84〜168 |
| 終盤 | 300〜600 | 168〜336 |
| やりこみ | 600〜1000+ | 336〜560+ |

実際の最大値はここに思慮係数（最大11倍）と馬鹿ボーナスが乗る。

## 4. 思えダメージ

> 実装: `BaseStates.Damage.cs:1068-1188`

通常ダメージ・精神ダメージを食らう**前に**判定される。追加HPでは防げない。

### 4.1 発動条件

```
powerRatio = 攻撃者の十日能力合計 / 被害者の十日能力合計
powerRatio >= 1.4 の場合のみ発動
```

**圧倒的な格上だけが正気を揺さぶれる。** スキルの十日能力補正はこの比率計算に関与しない（素の能力値同士の比較）。

### 4.2 計算式

```
① 基礎ダメージ = dmg.TenDayValuesSum × 0.06 + 人間状況別ボーナス
② 危険度       = min(0.8, 1.0 - (HP + MentalHP) / 2 / MaxHP)
③ 強さ倍率     = powerRatio - 0.4
④ 最終ダメージ  = ① × ② × ③ × (1 - 精神ポテンシャル × 0.06)
⑤ スキル数分散  = ④をランダマイズ（スキル7個以上の場合）
```

### 4.3 人間状況別ボーナス

**その人間状況のとき、どの属性と能力で正気が削られ発狂に至るか**を定義する。

| 人間状況 | 参照する十日能力成分 | 精神属性ボーナス(×1.29) |
|---------|-------------------|----------------------|
| **辛い** | 未消光の道 + 燃え火吹きウィフ | Devil, LiminalWhiteTile |
| **楽観的** | 夜暗黒 + 星テルシ | Doremis |
| **高揚** | ドクマムシ + 泉水 | なし（どの精神属性も効かない） |
| **覚悟** | テント空洞 + ヴォンド | Psycho |
| **怒り** | 陽炎 + 雨 + 冷酷冷静 | LiminalWhiteTile |
| **疑念** | 人殺し人 + ペルソナ乖離 + エノクナギ + 刃物 | Doremis, Psycho |
| **混乱** | サイレント練度 + ミザ + レインコート | Doremis, Sacrifaith |
| **普調** | — | **食らわない。自動回復する** |

精神属性ボーナスは、攻撃者の精神属性（`MyImpression`）とスキルの精神属性（`SkillSpiritual`）がそれぞれ一致した場合に×1.29倍ずつ乗算される（最大で2回適用）。

### 4.4 危険度（瀕死補正）

```
危険度 = 1.0 - (現在HP + 現在精神HP) / 2 / 最大HP
危険度 = min(0.8, 危険度)
```

瀕死に近いほど1に近づく（最大0.8）。体力・精神の両方が削れているほど正気も揺らぎやすい。

### 4.5 思えDEF（精神ポテンシャル）

> 実装: `BaseStates.StatesState.cs:53-70`

```
精神ポテンシャル = デフォルト精神属性 + 全スキルの精神属性（重複除外）の種類数
最終ダメージ × (1 - 精神ポテンシャル × 0.06)
```

スキルの精神属性が多様なほど思えダメージが軽減される。**スキル枠を増やすリスク（最大値の微小縮小）に対するカウンター**。

量を増やすだけではゲージが縮むだけだが、精神属性の種類数（質）を伴えば防御力が上がる。

### 4.6 スキル数によるダメージ分散

> 実装: `BaseStates.Damage.cs:1043-1063`

```
スキル数 <= 6: ランダマイズなし（固定ダメージ）
スキル数 > 6:
  ランダマイズ回数 = スキル数 - 6
  各回: 上振れ +1〜1.5%、下振れ -1〜3%
  → 累積した範囲内でランダム乗算
```

下振れ比重が大きい設計（上1〜1.5% vs 下1〜3%）。スキルが多い経験豊富なキャラは平均的にはダメージが軽くなる傾向があるが、上振れで一気に削られるリスクも生まれる。

## 5. 回復

### 5.1 戦闘中回復（普調時のみ）

> 実装: `BaseStates.Damage.cs:1193-1201`

```
人間状況が普調のとき、行動するたびに:
  回復量 = ResonanceValue × Random(0.03, 0.11)
```

最大値の3〜11%をランダム回復。普調以外の人間状況では回復しない。

### 5.2 歩行時回復

> 実装: `BaseStates.StatesNumber.cs:1364-1367`

```
回復量 = 1.0 + SpringNap × 1.5
```

戦闘外の歩行時に固定値＋春仮眠（SpringNap）補正で回復。

### 5.3 死亡時リセット

> 実装: `BaseStates.CallBack.cs:60`

死亡すると `ResetResonanceValue()` で最大値にリセットされる。

## 6. ライフサイクル

```
戦闘開始 → InitializeNowResonanceValue()   最大値の60〜100%でランダム開始
    │
    ├─ 攻撃を受ける → ResonanceDamage()    格上からのみ発動、人間状況で分岐
    ├─ 行動する     → ResonanceHealingOnBattle()  普調なら3〜11%回復
    ├─ 0到達       → 思え鳥化（※未実装。現状は0クランプのみ）
    └─ 死亡        → ResetResonanceValue()  最大値に戻る

戦闘外
    └─ 歩行        → ResonanceHealingOnWalking()  固定値 + 春仮眠補正
```

## 8. 人間状況との関連

思えの値は**人間状況（Demeanor）がある存在のみに機能する**。

- 人間状況がどの属性攻撃に対して思えが削られやすいかを決定する
- 普調は完全免疫＋回復フェーズ
- 戦闘中の心理変動が思えの脆弱性を動的に変化させる
- 人間状況自体はストーリー（主人公）や地区の影響（鬱病地区で鬱を移される等）で変わりうる
